## 1. 프로젝트 개요

### 1.1 프로젝트명

**PubliSync** (가칭) — 공공기관 소셜 미디어 통합 관리 플랫폼

### 1.2 배경

한국의 대다수 공공기관 및 단체는 국민과의 소통, 신뢰 구축, 정책 전달 등을 목적으로 YouTube, Instagram, Facebook, X(구 트위터), 네이버 블로그 등 다수의 소셜 미디어 플랫폼을 운영하고 있다. 이러한 계정의 실질적 운영(콘텐츠 제작·편집·게시, 댓글 관리, 모니터링, 위기 대응 등)은 대부분 전문 수탁업체에 위탁되어 수행된다.

### 1.3 문제 정의

| 문제 영역 | 상세 내용 |
|---|---|
| **반복 업무 과중** | 동일한 콘텐츠를 플랫폼마다 개별 접속하여 반복 게시·관리해야 한다. |
| **멀티 클라이언트 부담** | 수탁업체 한 곳이 다수의 위탁기관을 동시에 관리하므로, 반복 업무가 기관 수에 비례하여 증가한다. |
| **위기 대응 지연** | 악성 댓글, 여론 이슈 발생 시 플랫폼별로 확인해야 하므로 즉각적인 대응이 어렵다. |
| **수작업 편집 공수** | 영상 자막, 효과음, 이모지 삽입 등 편집 작업과 메타데이터(제목, 해시태그, 설명문) 작성에 과도한 시간이 소요된다. |
| **보고 업무 비효율** | 위탁기관에 제출하는 주간·월간 운영 리포트를 수작업으로 작성하며, 데이터 취합에 많은 시간이 투입된다. |
| **협업 체계 미흡** | 위탁기관 담당자와 수탁업체 실무자 간 검수·승인 프로세스가 체계화되어 있지 않다. |
| **자산 관리 부재** | 콘텐츠 원본, 편집본, 썸네일 등 디지털 자산이 개인 PC나 공유 드라이브에 산재하여, 재활용과 이력 추적이 어렵다. |

### 1.4 프로젝트 목적

> 하나의 웹 플랫폼에서 다수 위탁기관의 다수 플랫폼 계정을 통합 관리하되, AI(LLM, VLM 등)를 적극 활용하여 반복 업무를 자동화하고, 콘텐츠 품질과 위기 대응 역량을 동시에 끌어올려, 수탁업체와 위탁기관 관계자 모두에게 효율적이고 신뢰할 수 있는 협업 환경을 제공한다.

## 2. 용어 정의

| 용어 | 정의 |
|---|---|
| **위탁기관** | 소셜 미디어 계정 운영을 외부 업체에 위탁하는 공공기관 또는 단체 |
| **수탁업체** | 위탁기관으로부터 소셜 미디어 운영을 위탁받아 대행하는 전문 업체 |
| **플랫폼** | YouTube, Instagram, Facebook, X, 네이버 블로그 등 소셜 미디어 서비스 |
| **콘텐츠** | 플랫폼에 게시되는 영상, 이미지, 텍스트 등 모든 유형의 게시물 |
| **채널** | 특정 위탁기관이 특정 플랫폼에서 운영하는 개별 계정 (예: ○○부 YouTube 채널) |
| **멀티테넌트** | 하나의 시스템에서 여러 위탁기관의 데이터를 논리적으로 분리하여 관리하는 구조 |
| **감성 분석** | 댓글·반응의 감정(긍정/부정/중립/위험)을 AI가 자동으로 분류하는 것 |
| **숏폼** | 30초~60초 이내의 짧은 영상 콘텐츠 (YouTube Shorts, Instagram Reels 등) |
| **RBAC** | Role-Based Access Control, 역할 기반 접근 제어 |
| **PoC** | Proof of Concept, 핵심 기술의 실현 가능성을 사전 검증하는 소규모 시제품 *(v1.2 추가)* |
| **Fallback** | AI 자동 처리가 실패하거나 품질 기준 미달 시 대체 수행되는 수동/반자동 처리 경로 *(v1.2 추가)* |
| **Rate Limit** | 플랫폼 API가 일정 시간 내 허용하는 최대 호출 횟수 제한 *(v1.2 추가)* |
| **백프레셔(Backpressure)** | 시스템 과부하 시 요청 유입 속도를 자동으로 조절하는 메커니즘 *(v1.2 추가)* |

## 3. 개발 범위

### 3.1 초기 개발 대상

- **웹(Web) 버전만 개발**한다.
- 반응형 웹 디자인을 적용하여 모바일 브라우저에서도 핵심 기능(알림 확인, 승인 처리 등)을 사용할 수 있도록 한다.
- UI 언어는 **한국어를 기본**으로 하며, 다국어 UI는 초기 개발 범위에서 제외한다. *(v1.2 추가)*
- 네이티브 모바일 앱은 초기 개발 범위에서 제외하며, 향후 확장 단계에서 검토한다.

### 3.2 대상 플랫폼 (연동 대상)

| 플랫폼 | 콘텐츠 유형 | API 연동 방식 | 알려진 제약사항 | 비고 |
|---|---|---|---|---|
| YouTube | 영상, 숏츠 | YouTube Data API v3 (OAuth 2.0) | 일일 쿼터 10,000 단위, 업로드 6건/일 기본 | 영상 업로드, 댓글 관리, 분석 |
| Instagram | 이미지, 릴스, 스토리 | Instagram Graph API (OAuth 2.0) | 스토리는 게시 후 24시간 후 소멸, 스토리 인사이트 수집은 소멸 전에 완료해야 함 | 게시, 댓글 관리, 인사이트 |
| Facebook | 이미지, 영상, 텍스트 | Facebook Graph API (OAuth 2.0) | 페이지 토큰 관리 필요, API 버전 Deprecation 주기 약 2년 | 페이지 관리, 게시, 댓글 |
| X (구 트위터) | 텍스트, 이미지 | X API v2 (OAuth 2.0) | 무료 플랜 제한적, Basic 이상 필요 가능성 | 게시, 댓글/멘션 관리 |
| 네이버 블로그 | 텍스트, 이미지 | 네이버 블로그 API / 검색 API | **게시 API만 공식 지원, 댓글 관리·수정·삭제 API 미제공 가능성 높음** | 포스팅 위주, 댓글 기능 제한적 |

> **플랫폼 API 사전 검증 필수:** 각 플랫폼의 공식 API를 기반으로 연동하며, 개발 착수 전 **PoC를 통해 API 가용 범위를 실증 확인**한다. API 지원이 확인되지 않는 기능은 해당 플랫폼에서 제외하거나 대안을 모색한다. *(v1.2 보완)*

> **네이버 블로그 특기사항:** 네이버 블로그는 공식 API 범위가 타 플랫폼 대비 제한적이므로, Phase 1-A 개발 전 반드시 API PoC를 수행하고, 결과에 따라 지원 범위를 조정한다. 댓글 관리 등 API 미지원 기능은 웹 스크래핑 방식 검토 또는 기능 범위 축소를 결정한다. *(v1.2 추가)*

> **Instagram 스토리 수집 주의:** 스토리 콘텐츠 및 인사이트 데이터는 게시 후 24시간 이내에 수집을 완료해야 한다. 크롤링 스케줄러 설계 시 스토리 소멸 시점을 고려한 **우선 수집 로직**을 포함한다. *(v1.2 추가)*

## 5. 기능 정의

### 5.1 핵심 기능 (Core Features)

#### F01. 통합 멀티 플랫폼 게시

- 하나의 콘텐츠(영상, 이미지, 텍스트)를 작성·편집한 뒤, 연동된 모든 플랫폼에 **동시 게시**한다.
- 플랫폼별 게시 포맷(이미지 비율, 글자 수 제한, 해시태그 규칙 등)을 **자동 최적화**하여 적용한다.
- 예약 게시 기능을 지원한다.
- 게시 이력을 기록하고 조회할 수 있다.
- 게시 실패 시 **재시도 및 오류 알림** 기능을 제공한다.
- 다수 기관의 동시 게시 요청이 발생할 경우, **비동기 큐 기반**으로 순차 처리하여 API Rate Limit 초과를 방지한다. *(v1.2 추가)*

#### F02. AI 기반 콘텐츠 메타데이터 자동 생성 (LLM)

- 업로드된 콘텐츠를 분석하여 **제목, 해시태그, 설명문(캡션)**을 자동 생성한다.
- 플랫폼별 특성에 맞는 톤앤매너로 변환된 버전을 함께 제공한다.
- 담당자가 수정·확정할 수 있는 편집 인터페이스를 포함한다.
- **Fallback:** AI 생성 결과가 만족스럽지 않을 경우, 담당자가 직접 작성하거나 기존 템플릿에서 선택할 수 있다. AI 생성은 항상 **"제안"**이며, 최종 결정은 사람이 한다. *(v1.2 추가)*

#### F03. AI 기반 영상 편집 보조 (VLM + LLM + 오디오 AI)

- 영상 콘텐츠에 **자막을 자동 생성**한다. (STT 기반)
- 영상의 장면 전환, 감정 흐름 등을 VLM이 분석하여 **효과음 및 배경음악을 자동 인식·삽입**한다.
- 특정 장면(강조 포인트, 감정 변화 지점 등)에 **이모지 및 시각 효과를 자동 배치**한다.
- 편집 결과를 미리보기하고 수정할 수 있는 인터페이스를 제공한다.
- **기술 리스크 (v1.2 추가):**
  - 본 기능은 VLM 기반 장면 분석 + 오디오 AI 효과음 삽입 등 **기술적 난이도가 높은 기능**을 다수 포함한다.
  - 개발 착수 전 반드시 **PoC를 수행**하여, (1) VLM의 장면 감정 분석 정확도, (2) 효과음 자동 삽입의 품질 수준, (3) 처리 시간 및 비용을 검증한다.
  - PoC 결과에 따라 기능 범위를 **단계적으로 축소 또는 조정**할 수 있다.
  - **최소 보장 범위:** STT 기반 자막 자동 생성은 기술 성숙도가 높으므로 반드시 포함한다. 효과음·이모지 자동 삽입은 PoC 결과에 따라 결정한다.
- **Fallback:** AI가 생성한 편집 결과는 반드시 미리보기 후 수동 조정 가능해야 하며, AI 편집 없이 수동 편집만으로도 워크플로우가 완결될 수 있어야 한다.

#### F04. 통합 댓글 관리

- 연동된 모든 플랫폼의 댓글을 **각 플랫폼 공식 API를 통해 통합 수집**하고 하나의 화면에서 조회한다. *(v1.2 표현 수정: "크롤링" → "공식 API 수집" 명확화)*
- 댓글에 대한 **감성 분석**(긍정 / 부정 / 중립 / 위험)을 자동 수행한다.
- 위험 등급으로 분류된 댓글에 대해 **웹 알림 및 텔레그램 알림**을 발송한다.
- 알림을 받은 담당자가 **웹(모바일 브라우저 포함)에서 승인 처리**를 하면, 해당 댓글에 대해 다음 조치가 자동 실행된다.
  - **삭제**: 해당 플랫폼에서 댓글을 삭제한다.
  - **자동 답글**: LLM이 생성한 답글 초안을 게시한다.
  - **숨김 처리**: 삭제 대신 플랫폼에서 숨김 처리한다. (플랫폼이 지원하는 경우)
- **삭제·숨김 처리된 댓글은 DB에 아카이빙**하여, 이후 이력 조회 및 증빙으로 활용할 수 있다.
- 아카이빙된 댓글은 **처리 사유, 처리자, 처리 일시**를 함께 기록한다.
- **플랫폼별 제약 (v1.2 추가):** 댓글 삭제·숨김·답글 기능은 각 플랫폼 API의 지원 범위에 따라 가용성이 다르다. API가 미지원하는 조치는 해당 플랫폼에서 비활성화하고, UI에 사유를 표시한다.

#### F05. AI 답글 초안 자동 생성 (LLM)

- 댓글에 대한 답글을 기관의 **톤과 정책 기조에 맞춰 초안을 자동 작성**한다.
- 반복되는 민원성 댓글에 대해서는 **FAQ 기반 자동 응답 템플릿**과 연동한다.
- 담당자는 초안을 확인·수정한 뒤 승인만으로 게시할 수 있다.
- **Fallback:** AI 초안의 품질이 부적절하다고 판단될 경우, 담당자가 직접 답글을 작성하거나 기존 템플릿에서 선택할 수 있다. *(v1.2 추가)*

#### F06. 통합 모니터링 대시보드

- 플랫폼별 접속자 수, 조회 수, 좋아요, 공유 수, 팔로워 변화 등 **핵심 KPI를 한 화면에서 통합 표시**한다.
- 위탁기관별, 플랫폼별, 기간별 필터링 및 비교 기능을 제공한다.
- **실시간 갱신** 또는 설정 가능한 주기별 자동 갱신을 지원한다.
- 각 플랫폼 API의 데이터 제공 주기·지연 시간이 다르므로, **데이터 최신성(freshness)을 플랫폼별로 표시**한다. *(v1.2 추가)*

#### F07. 텔레그램 긴급 알림 연동

- 위험 댓글 감지, 부정 여론 급증, 시스템 이상 등 **긴급 상황 발생 시 텔레그램 메시지를 즉시 발송**한다.
- 알림 수신 대상 및 알림 조건을 설정할 수 있다.
- 알림 유형별(위험 댓글, 여론 급변, 게시 실패, 시스템 장애 등) **채널 또는 그룹을 분리 설정**할 수 있다.

#### F08. 사용자 및 권한 관리

- **역할 기반 접근 제어(RBAC)**를 적용한다.
  - 시스템 관리자: 시스템 전체 관리 권한
  - 위탁기관 담당자: 검수·승인 권한
  - 수탁업체 관리자: 계정 관리·팀원 관리 권한
  - 수탁업체 실무자: 편집·게시·모니터링 권한
- 수탁업체 한 곳이 **다수의 위탁기관을 멀티테넌트 구조로 관리**할 수 있다.
- 위탁기관 간 데이터는 **논리적으로 완전히 격리**되어, 타 기관의 데이터에 접근할 수 없다.

#### F09. 콘텐츠 검수·승인 워크플로우 *(v1.1 추가)*

- 콘텐츠 게시 전, 정의된 **승인 절차를 반드시 거치도록** 워크플로우를 제공한다.
- 기본 워크플로우: **작성(수탁 실무자) → 검토 요청 → 검수(위탁기관 담당자) → 승인/반려 → 게시**
- 반려 시 **반려 사유를 기록**하고, 수정 후 재검토 요청이 가능하다.
- 각 단계의 **처리 일시, 처리자, 코멘트를 이력으로 보관**한다.
- 위탁기관별로 승인 절차의 단계 수와 승인자를 **커스터마이징**할 수 있다.
- 긴급 게시가 필요한 경우를 위한 **긴급 승인 경로(Fast Track)**를 설정할 수 있다.

#### F10. 콘텐츠 캘린더 *(v1.1 추가)*

- 모든 위탁기관의 게시 예정·완료 콘텐츠를 **캘린더 뷰로 통합 표시**한다.
- 월간/주간/일간 보기를 지원하며, 위탁기관별·플랫폼별 필터링이 가능하다.
- 캘린더에서 직접 **게시 예약 등록·수정·삭제**를 할 수 있다.
- 동일 시간대에 게시가 집중될 경우 **충돌 경고**를 표시한다.
- 공휴일, 기념일, 정책 일정 등 **주요 일정을 함께 표시**하여 콘텐츠 기획에 활용할 수 있다.

#### F11. 디지털 자산 관리 (미디어 라이브러리) *(v1.1 추가)*

- 업로드된 영상, 이미지, 문서, 템플릿 등 모든 디지털 자산을 **중앙 저장소에서 체계적으로 관리**한다.
- 위탁기관별, 콘텐츠 유형별, 태그별 **분류 및 검색** 기능을 제공한다.
- 자산의 **버전 관리**(원본, 편집본, 최종본)를 지원한다.
- 자산을 콘텐츠 제작 시 바로 불러와 **재활용**할 수 있다.
- 저장 용량 및 보관 기간 정책을 위탁기관별로 설정할 수 있다.

#### F12. 플랫폼 계정 연동 관리 *(v1.1 추가)*

- 각 위탁기관의 플랫폼 계정(YouTube, Instagram, Facebook, X, 네이버 블로그)을 PubliSync에 **OAuth 또는 API 키 방식으로 연동**한다.
- 연동된 계정의 **인증 상태를 상시 모니터링**하고, 토큰 만료 임박 시 **갱신 알림**을 발송한다.
- 계정별 **연동 이력(연결·해제·갱신 일시)**을 기록한다.
- 플랫폼 API 호출 빈도를 관리하여 **Rate Limit 초과를 방지**한다.
- **Rate Limit 관리 전략 (v1.2 추가):**
  - 모든 API 호출은 **중앙 큐(Queue)**를 통해 관리한다.
  - 플랫폼별 **남은 쿼터를 실시간 추적**하고, 임계치(80%) 도달 시 호출 속도를 자동 조절한다 (백프레셔).
  - 다수 기관의 요청이 동시에 발생할 경우, **우선순위 정책**을 적용한다: 긴급 게시 > 예약 게시 > 데이터 수집 > 분석 요청.
  - Rate Limit 초과 시 **지수 백오프(Exponential Backoff)**로 재시도하며, 최종 실패 시 담당자에게 알림을 발송한다.
  - 플랫폼별 Rate Limit 현황을 **시스템 관리자 대시보드에서 모니터링**할 수 있다.

#### F13. 알림 센터 *(v1.1 추가)*

- 텔레그램 긴급 알림(F07)과 별도로, **웹 플랫폼 내부에 통합 알림 센터**를 제공한다.
- 알림 유형: 위험 댓글 감지, 승인 요청, 승인/반려 결과, 게시 완료/실패, 토큰 만료 임박, 시스템 공지 등
- 알림별 **읽음/미읽음 상태 관리** 및 **알림 필터링** 기능을 제공한다.
- 브라우저 **웹 푸시 알림(Web Push Notification)**을 지원하여, 브라우저가 열려 있지 않아도 알림을 수신할 수 있다.
- 사용자별로 **수신할 알림 유형을 선택**할 수 있다.

#### F14. 활동 이력 관리 (감사 로그) *(v1.1 추가)*

- 시스템 내 모든 주요 행위를 기록하고 조회할 수 있는 **감사 로그** 기능을 제공한다.
- 기록 대상: 콘텐츠 작성·수정·삭제·게시, 댓글 삭제·숨김·답글, 승인·반려, 계정 연동·해제, 사용자 로그인·권한 변경 등
- 기록 항목: **행위자, 행위 유형, 대상, 일시, IP 주소, 변경 전후 값**
- 위탁기관별로 격리하여 **해당 기관의 이력만 조회** 가능하다.
- 감사 로그는 **수정·삭제가 불가능한 불변(Immutable) 구조**로 저장한다.
- 공공기관 감사 및 정보공개 요청에 대비한 **로그 내보내기(CSV/PDF)** 기능을 제공한다.

### 5.2 AI 고도화 기능 (AI-Enhanced Features)

#### F15. AI 숏폼 자동 생성 (VLM + LLM)

- 긴 영상(정책 브리핑, 행사 영상 등)을 VLM이 분석하여 **핵심 장면을 자동 추출**하고, 30초~60초 분량의 숏폼 콘텐츠를 생성한다.
- 추출된 장면에 자막, 캡션, 강조 효과를 자동 적용한다.
- 공공기관 영상의 숏폼 전환을 통해 **채널 도달률을 극대화**한다.
- **기술 리스크 (v1.2 추가):** VLM 기반 핵심 장면 추출의 정확도는 영상 유형(브리핑, 행사, 인터뷰 등)에 따라 편차가 클 수 있다. 공공기관 영상 특성에 맞는 **프롬프트 튜닝 및 PoC**가 선행되어야 한다.
- **Fallback:** AI가 추출한 구간을 담당자가 확인·수정할 수 있는 **타임라인 편집 인터페이스**를 제공한다. AI 추출 없이 수동으로 구간을 지정하여 숏폼을 만들 수도 있다.

#### F16. AI 썸네일 자동 생성 (VLM + 이미지 생성)

- 영상에서 **가장 시선을 끄는 프레임을 VLM이 선별**한다.
- 제목 텍스트 오버레이를 포함한 썸네일을 자동 생성한다.
- 플랫폼별 권장 비율(16:9, 1:1, 9:16)에 맞춰 **자동 리사이징 및 리컴포지션**한다.
- **Fallback:** AI가 선별한 후보 프레임 3~5개를 제시하고, 담당자가 최종 선택 후 텍스트 오버레이를 수동 편집할 수 있다. *(v1.2 추가)*

#### F17. 플랫폼별 톤앤매너 자동 변환 (LLM)

- 하나의 원문을 플랫폼 특성에 맞게 **문체를 자동 변환**한다.
  - YouTube → 정보 전달형 설명문
  - Instagram → 감성적·시각 중심 캡션
  - X → 핵심 압축형 문장
  - 블로그 → 상세 서술형 포스트
- 기관별 **톤 가이드를 학습**시켜 일관성을 유지한다.
- **Fallback:** 변환 결과를 플랫폼별로 나란히 미리보기하고, 개별적으로 수정할 수 있다. *(v1.2 추가)*

#### F18. 여론 동향 예측 및 이슈 선제 감지 (LLM + 트렌드 분석)

- 댓글·반응 패턴의 **시계열 변화를 추적**하여 부정 여론의 징후를 조기에 감지한다.
- 특정 키워드 출현 빈도 상승, 공유 수 급증 등 **선행 지표를 분석**한다.
- "24시간 내 이슈화 가능성" 등 **위기 확률 예측 리포트**를 제공한다.
- **주의사항 (v1.2 추가):** 예측 정확도에 대한 과도한 기대를 방지하기 위해, 리포트에 **신뢰도 수준(높음/보통/낮음)**을 함께 표시한다. 이 기능은 의사결정 "참고 자료"이며, 최종 판단은 담당자가 한다.

#### F19. AI 자동 운영 리포트 생성 (LLM)

- 주간·월간 운영 리포트를 **자동 생성**한다.
- 플랫폼별 KPI 종합 요약, 차트, 개선 제안을 포함한다.
- 공공기관 보고서 양식에 맞춘 **템플릿 기반 출력**(PDF 등)을 지원한다.
- 수탁업체가 위탁기관에 제출하는 보고서 작성 시간을 획기적으로 단축한다.
- **Fallback:** AI가 생성한 리포트 초안을 담당자가 편집·보완한 뒤 최종 확정한다. 리포트 내 데이터(수치, 차트)는 원본 데이터에서 직접 추출하며, AI 해석이 포함된 텍스트 영역만 AI가 작성한다. *(v1.2 추가)*

#### F20. 콘텐츠 성과 예측 (LLM + 데이터 분석)

- 게시 전에 콘텐츠의 **예상 도달률, 반응률을 AI가 예측**한다.
- 과거 유사 콘텐츠의 성과 데이터를 학습하여 **제목·설명문 A/B 테스트를 제안**한다.
- **최적 게시 시간대**를 추천한다.
- **주의사항 (v1.2 추가):** 초기에는 학습 데이터가 부족하여 예측 정확도가 낮을 수 있다. 최소 **3~6개월 이상의 운영 데이터 축적** 후에 의미 있는 예측이 가능하며, 데이터 축적 기간에는 기능을 "베타" 상태로 제공한다.

#### F21. 공공기관 표현 가이드 자동 검수 (LLM)

- 게시 전에 콘텐츠를 자동 스캔하여 다음 항목을 검출한다.
  - 부적절한 표현 및 차별적 문구
  - 개인정보 노출 (이름, 전화번호, 주민등록번호 등)
  - 저작권 위반 소지가 있는 이미지·문구
- 웹 접근성 지침(장애인접근성) 충족 여부를 체크한다.
- 검수 결과를 **경고 리포트로 표시**하여 게시 전 확인할 수 있도록 한다.
- 콘텐츠 승인 워크플로우(F09)와 연동하여, 검수 미통과 시 **자동 반려 또는 경고 첨부** 처리한다.
- **Fallback:** AI 검수는 보조 수단이며, 최종 검수 책임은 승인 워크플로우의 검수자에게 있다. AI가 검출하지 못한 항목에 대한 면책 범위를 명확히 한다. *(v1.2 추가)*

#### F22. 다국어 자동 번역 및 자막 생성 (LLM)

- 외국인 주민·관광객 대상 콘텐츠에 **다국어 자막 및 번역문을 자동 생성**한다.
- 공공기관의 다문화 소통 의무에 부합하며, 글로벌 노출을 확대한다.
- **지원 언어 (v1.2 추가):** 초기 지원 대상은 영어, 중국어(간체), 일본어, 베트남어로 한정하며, 수요에 따라 확장한다.

#### F23. 경쟁·벤치마크 분석 (LLM + 크롤링)

- 유사 기관 또는 타 기관의 SNS 운영 현황을 **자동 수집·비교 분석**한다.
- "A기관은 이번 주 릴스 조회수가 300% 증가, 사용된 포맷은 ○○" 등 **인사이트를 자동 리포트로 제공**한다.
- 수탁업체가 위탁기관에 **데이터 기반 전략 제안**을 할 수 있도록 지원한다.
- **제약사항 (v1.2 추가):** 타 기관 데이터 수집은 각 플랫폼의 공개 데이터(조회수, 좋아요 등)에 한정하며, 비공개 데이터(인사이트, 도달률 등)는 수집 불가하다. 플랫폼 이용약관을 준수하는 범위에서 수행한다.

### 5.3 AI 기능 공통 정책 *(v1.2 추가)*

모든 AI 기능(F02, F03, F05, F15~F23)에 대해 다음 공통 정책을 적용한다.

| 정책 항목 | 내용 |
|---|---|
| **Human-in-the-Loop 원칙** | AI의 모든 산출물은 **"제안(Suggestion)"**이며, 최종 결정·게시 권한은 사람에게 있다. AI가 자동으로 외부에 콘텐츠를 게시하는 것은 허용하지 않는다. |
| **Fallback 필수** | 모든 AI 기능은 AI 없이도 **수동으로 동일 작업을 수행할 수 있는 경로**를 반드시 제공한다. AI 장애 시에도 핵심 업무(게시, 댓글 관리, 승인)가 중단되지 않아야 한다. |
| **품질 피드백 루프** | 사용자가 AI 결과를 채택/수정/거부한 이력을 수집하여, **프롬프트 개선 및 품질 향상**의 기초 데이터로 활용한다. |
| **비용 투명성** | AI 호출로 인한 비용을 **기관별·기능별로 추적**할 수 있도록 하며, 시스템 관리자가 모니터링할 수 있는 대시보드를 제공한다. |
| **AI 라벨링** | AI가 생성하거나 보조한 콘텐츠에는 내부적으로 **AI 기여 이력을 기록**한다. 외부 게시물에 AI 생성 라벨을 표시할지 여부는 위탁기관 정책에 따른다. |

## 9. 비용 구조 고려사항 *(v1.2 추가)*

본 프로젝트는 AI 서비스와 클라우드 인프라를 활용하므로, 운영 비용이 사용량에 비례하여 증가한다. 아키텍처 설계 및 사업 모델 수립 시 다음 비용 요소를 고려해야 한다.

| 비용 영역 | 비용 발생 요인 | 고려사항 |
|---|---|---|
| **AI/LLM 호출** | 메타데이터 생성, 감성 분석, 답글 초안, 리포트 생성 등 | 기관 수 × 콘텐츠/댓글 수에 비례 증가. 기관별 월간 AI 호출 한도 설정 검토 |
| **VLM/영상 처리** | 영상 분석, 숏폼 생성, 자막 추출, 썸네일 선별 | 영상 길이·해상도에 따라 비용 급증. 영상당 처리 비용 상한 설정 필요 |
| **스토리지** | 미디어 라이브러리, 영상 원본·편집본, 감사 로그 | 기관별 저장 용량 할당 및 보관 기간별 자동 아카이빙/삭제 정책 필요 |
| **인프라 (서버/DB)** | 멀티테넌트 환경의 컴퓨팅·DB 리소스 | 기관 수 증가에 따른 수평 확장 비용 |
| **플랫폼 API** | X API 유료 플랜 등 | 일부 플랫폼은 API 사용에 유료 플랜 가입 필요 |
| **영상 인코딩** | 숏폼 생성, 포맷 변환, 해상도 변환 | 트랜스코딩 서비스 비용 (예: AWS MediaConvert 등) |

> **과금 모델 방향:** 위탁기관별 또는 수탁업체별 사용량 기반 과금 모델을 수립하되, 기관별 **월간 사용량 한도(AI 호출 수, 저장 용량, 영상 처리 시간)**를 설정하여 비용을 예측 가능하게 관리한다. 구체적인 과금 체계는 사업 모델 설계 단계에서 확정한다.

## 12. 핵심 가치 요약

```
반복 업무 자동화             →  운영 공수 절감

AI 기반 콘텐츠 보조          →  편집 품질 향상 + 속도 개선
실시간 감성 분석 + 알림      →  위기 대응 속도 향상
자동 리포트 + 성과 예측      →  데이터 기반 의사결정 지원
표현 가이드 자동 검수        →  공공기관 리스크 사전 차단
검수·승인 워크플로우         →  위탁-수탁 간 책임 명확화

디지털 자산 중앙 관리        →  콘텐츠 재활용 및 이력 추적
Human-in-the-Loop AI        →  AI 리스크 최소화 + 신뢰성 확보  ← (v1.2 추가)
```
