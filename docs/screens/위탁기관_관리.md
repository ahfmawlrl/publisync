# 위탁기관 관리 (OrganizationsPage)

> **경로:** `/settings/organizations`
> **소스:** `frontend/src/features/settings/pages/OrganizationsPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F08 권한관리

---

## 1. 화면 개요

시스템 관리자(SA) 또는 수탁업체 관리자(AM)가 위탁기관(공공기관)을 등록, 수정, 삭제하는 화면이다.
위탁기관은 멀티테넌트 아키텍처의 핵심 단위로, 각 기관별로 플랜(basic/standard/premium)을 할당하고
상태(활성/비활성/정지)를 관리한다. 기관 슬러그는 URL 식별자로 사용되며 생성 시에만 설정 가능하다.

### 핵심 원칙
- **멀티테넌트 격리:** 각 위탁기관은 독립된 데이터 공간을 가지며, PostgreSQL RLS로 격리
- **플랜 기반 관리:** 3단계 플랜으로 기관별 기능 범위 차등 적용
- **슬러그 불변성:** 슬러그는 생성 시 설정 후 수정 불가 (URL 식별자 안정성 보장)

## 4. 위탁기관 테이블

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Table` (rowKey="id") |
| **데이터 소스** | `data?.data || []` |
| **API** | `GET /admin/organizations` |
| **페이지 크기** | 20건 |

### 4.1 테이블 컬럼

| 컬럼 | dataIndex | 렌더링 |
|------|-----------|--------|
| **기관명** | `name` | 텍스트 |
| **슬러그** | `slug` | 텍스트 |
| **플랜** | `plan` | `Tag` (레이블 변환) |
| **상태** | `status` | `Tag` (색상 매핑) |
| **생성일** | `created_at` | `toLocaleDateString('ko-KR')` |
| **관리** | - | 수정(EditOutlined) + 삭제(DeleteOutlined) 버튼 |

### 4.2 플랜 옵션

| 값 | 한글 레이블 |
|----|-----------|
| `basic` | 베이직 |
| `standard` | 스탠다드 |
| `premium` | 프리미엄 |

### 4.3 상태 색상 매핑

| 상태 코드 | 한글명 | Tag 색상 |
|-----------|--------|----------|
| `ACTIVE` | 활성 | `green` |
| `INACTIVE` | 비활성 | `default` |
| `SUSPENDED` | 정지 | `red` |

### 4.4 페이지네이션
| 항목 | 내용 |
|------|------|
| **현재 페이지** | `page` 상태 |
| **전체 건수** | `data?.meta?.total || 0` |
| **페이지 크기** | 20 |
| **총 건수 표시** | `총 ${total}개` |

### 4.5 행 액션 버튼

| 버튼 | 아이콘 | 동작 |
|------|--------|------|
| **수정** | `EditOutlined` | `handleEdit(record)` -- 기관 수정 모달 열기 |
| **삭제** | `DeleteOutlined` (danger) | `Popconfirm` 확인 후 `deleteMutation.mutate(record.id)` |

## 6. 연결 흐름 (Connected Flows)

### 6.1 위탁기관 CRUD 흐름

```
[위탁기관 관리 화면]
       │
       ├──(기관 추가)──→ [추가 모달] ──→ POST /admin/organizations ──→ 목록 갱신
       │                    │
       │                    └── name + slug + plan 입력
       │
       ├──(기관 수정)──→ [수정 모달] ──→ PUT /admin/organizations/:id ──→ 목록 갱신
       │                    │
       │                    └── name + plan + status 변경
       │
       └──(기관 삭제)──→ [Popconfirm] ──→ DELETE /admin/organizations/:id ──→ 목록 갱신
```

### 6.2 기관과 사용자 관계

```
[위탁기관 관리]                [사용자 관리]
      │                            │
      └── organization_id ────────→│ (사용자 소속 기관 지정)
                                   │
                                   └── 기관별 사용자 필터링
```

### 6.3 기관과 채널 관계

```
[위탁기관 생성]
       │
       ↓
[기관 활성화] ──→ [채널 연동 (/channels)]
                       │
                       └── 기관별 SNS 채널 OAuth 연결
                            (YouTube, Instagram, Facebook, X, 네이버)
```

### 6.4 기관과 워크스페이스

```
[위탁기관 생성] ──→ 워크스페이스 자동 생성
                       │
                       ↓
                  [상단바 워크스페이스 전환]
                       │
                       ↓
                  SET LOCAL app.current_org_id = '{org_id}'
                       │
                       ↓
                  [RLS 자동 데이터 격리]
```

## 8. 데이터 타입

### 8.1 위탁기관 레코드 (OrgRecord)

```typescript
interface OrgRecord {
  id: string;          // 기관 고유 ID (UUID)
  name: string;        // 기관명 (예: "서울특별시")
  slug: string;        // URL 식별자 (예: "seoul-city")
  plan: string;        // 플랜 (basic | standard | premium)
  status: string;      // 상태 (ACTIVE | INACTIVE | SUSPENDED)
  created_at: string;  // 생성일시 (ISO 8601)
}
```

### 8.2 기관 생성 요청

```typescript
interface OrgCreateData {
  name: string;   // 기관명 (필수)
  slug: string;   // 슬러그 (필수, /^[a-z0-9-]+$/)
  plan: string;   // 플랜 (필수, basic | standard | premium)
}
```

### 8.3 기관 수정 요청

```typescript
interface OrgUpdateData {
  id: string;       // 대상 기관 ID (필수)
  name?: string;    // 기관명 (선택)
  plan?: string;    // 플랜 (선택)
  status?: string;  // 상태 (선택)
}
```

### 8.4 API 응답 타입

```typescript
// 목록 조회 응답
PaginatedResponse<OrgRecord> {
  data: OrgRecord[];
  meta: {
    total: number;
    page: number;
    limit: number;
  };
}

// 단건 응답
ApiResponse<OrgRecord> {
  success: boolean;
  data: OrgRecord;
}
```

## 11. 사용 시나리오

### 시나리오 1: 신규 위탁기관 등록
1. [기관 추가] 버튼 클릭
2. 기관명: "서울특별시", 슬러그: "seoul-city", 플랜: "스탠다드" 입력
3. [추가] 클릭 -- POST /admin/organizations 호출
4. 성공 시 "위탁기관이 생성되었습니다" 메시지 + 모달 닫힘 + 목록 갱신
5. 워크스페이스 전환 메뉴에 "서울특별시" 추가됨

### 시나리오 2: 기관 플랜 업그레이드
1. 대상 기관 행의 수정(연필) 아이콘 클릭
2. 플랜을 "베이직"에서 "프리미엄"으로 변경
3. [저장] 클릭 -- PUT /admin/organizations/:id 호출
4. 성공 시 "위탁기관 정보가 수정되었습니다" 메시지 + 플랜 Tag 갱신

### 시나리오 3: 기관 일시 정지
1. 대상 기관 행의 수정(연필) 아이콘 클릭
2. 상태를 "활성"에서 "정지"로 변경
3. [저장] 클릭 -- PUT /admin/organizations/:id 호출
4. 성공 시 상태 Tag가 빨간색 "정지"로 변경
5. 해당 기관 사용자의 접근이 제한됨

### 시나리오 4: 위탁기관 삭제
1. 대상 기관 행의 삭제(휴지통) 아이콘 클릭
2. "위탁기관을 삭제하시겠습니까?" 확인 팝업 표시
3. [삭제] 클릭 -- DELETE /admin/organizations/:id 호출
4. 성공 시 "위탁기관이 삭제되었습니다" 메시지 + 목록에서 제거
