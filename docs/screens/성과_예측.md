# 성과 예측 (PredictionPage)

> **경로:** `/analytics/prediction`
> **소스:** `frontend/src/features/analytics/pages/PredictionPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 3
> **관련 기능:** F20 성과예측

---

## 1. 화면 개요

축적된 성과 데이터를 기반으로 AI가 콘텐츠의 예상 성과(조회수, 좋아요)를 플랫폼별로 예측하고,
최적 게시 시간과 A/B 테스트 제안을 제공하는 화면이다. BETA 기능으로, 최소 3개월 이상의
데이터가 축적되어야 정확한 예측이 가능하다. 데이터 부족 시 안내 화면을 표시하며,
별도의 AI 최적 게시 시간 추천 도구를 제공한다.

### 핵심 원칙
- **Human-in-the-Loop:** 모든 예측 결과는 "참고 자료"이며 최종 판단은 담당자가 한다
- **Fallback 필수:** 예측 데이터 부족 시에도 수동 AI 최적 시간 추천 도구 사용 가능
- **워크플로우 내장 AI:** 예측과 A/B 테스트 제안이 분석 화면 안에서 인라인으로 제공됨

## 10. 연결 흐름 (Connected Flows)

### 10.1 성과 예측 진입 흐름

```
[사이드바] ──→ [성과 분석] ──→ [성과 예측 /analytics/prediction]
                                     │
                                     ├──(데이터 충분)──→ 예측 결과 표시
                                     │
                                     └──(데이터 부족)──→ 안내 화면 표시
```

### 10.2 AI 최적 시간 추천 흐름

```
[콘텐츠 내용 입력]
       │
       ├──→ [플랫폼 선택]
       │
       └──→ [추천받기] ──→ POST /ai/optimal-time
                              │
                              ├──(성공)──→ AI 추천 결과 카드 표시
                              │              (요일/시간대/근거 + 신뢰도/모델)
                              │
                              └──(실패)──→ Mutation 에러 처리
```

### 10.3 예측 결과 활용 흐름

```
[예측 결과 확인]
       │
       ├──→ 플랫폼별 예상 조회수/좋아요 확인
       │
       ├──→ 최적 게시 시간 확인 → 콘텐츠 예약 게시에 반영
       │
       └──→ A/B 테스트 제안 → 콘텐츠 작성 시 적용
```

## 12. 데이터 타입

### 12.1 예측 데이터 (PredictionData)

```typescript
interface PredictionData {
  is_beta: boolean;                            // BETA 여부
  content_id: string | null;                   // 콘텐츠 ID
  platform_predictions: PredictionPlatformItem[]; // 플랫폼별 예측
  optimal_publish_times: OptimalTimeSlot[];     // 최적 게시 시간
  ab_test_suggestions: AbTestSuggestion[];     // A/B 테스트 제안
  confidence: number;                          // 전체 신뢰도 (0~1)
  data_months: number;                         // 학습 데이터 기간 (월)
}
```

### 12.2 플랫폼 예측 아이템 (PredictionPlatformItem)

```typescript
interface PredictionPlatformItem {
  platform: string;                             // 플랫폼 코드
  estimated_views: PredictionRangeItem | null;  // 예상 조회수 (min~max)
  estimated_likes: PredictionRangeItem | null;  // 예상 좋아요 (min~max)
}

interface PredictionRangeItem {
  min: number;  // 최소값
  max: number;  // 최대값
}
```

### 12.3 최적 시간 슬롯 (OptimalTimeSlot)

```typescript
interface OptimalTimeSlot {
  day_of_week: string;  // 요일 (월, 화, ...)
  time_range: string;   // 시간대 ("09:00~10:00")
  reason: string;       // 근거 설명
}
```

### 12.4 A/B 테스트 제안 (AbTestSuggestion)

```typescript
interface AbTestSuggestion {
  field: string;       // 항목 (제목, 시간, 해시태그 등)
  option_a: string;    // 옵션 A
  option_b: string;    // 옵션 B
  prediction: string;  // 예측 결과
  reason: string;      // 근거
}
```

### 12.5 AI 최적 시간 응답

```typescript
interface OptimalTimeResponse {
  isAiGenerated: boolean;
  optimal_times: Array<{
    day_of_week: string;
    time_range: string;
    reason: string;
    confidence: number;
  }>;
  confidence: number;
  model: string;
}
```

| 영역 | 도구 | 설명 |
|------|------|------|

## 14. 오류 처리 및 Fallback

| 상황 | 처리 |
|------|------|
| **데이터 로딩 중** | 전체 화면 `Spin` (size="large", h-64 중앙) |
| **데이터 부족 (< 3개월)** | `Empty` + 안내 메시지 (남은 기간 표시) |
| **플랫폼 예측 없음** | `Empty` (description="예측 데이터가 없습니다") |
| **최적 시간 없음** | `Empty` (description="최적 시간 데이터가 없습니다") |
| **A/B 테스트 제안 없음** | 해당 섹션 숨김 |
| **AI 최적 시간 요청 실패** | Mutation 에러 핸들러 (comment: "error handled by mutation") |
| **콘텐츠 미입력 시 추천** | 추천받기 버튼 `disabled` 상태 |

1. 사이드바 > 성과 분석 > 성과 예측 클릭
2. BETA 태그 확인, 예측 신뢰도 바 확인
3. 플랫폼별 예상 성과 테이블에서 예상 조회수/좋아요 범위 확인
4. 최적 게시 시간 테이블에서 요일/시간대/근거 확인
5. A/B 테스트 제안에서 제목/시간 등의 실험 아이디어 참고

### 시나리오 2: AI 최적 게시 시간 추천 받기

1. 페이지 하단 "AI 최적 게시 시간 추천" 카드 확인
2. 게시 예정 콘텐츠 내용 입력
3. 대상 플랫폼 선택 (YouTube, Instagram)
4. [추천받기] 버튼 클릭
5. AI 추천 결과 카드에서 요일/시간대/근거 확인
6. 신뢰도와 사용 모델 확인
7. 콘텐츠 작성 화면에서 예약 게시 시간 설정에 반영

### 시나리오 3: 데이터 부족 시

1. 성과 예측 화면 접속
2. "데이터가 부족합니다" 안내 표시
3. "현재 N개월의 데이터가 축적되어 있습니다" 확인
4. 기존 콘텐츠 게시를 계속하며 3개월 데이터 축적 대기
