# 알림 센터 (NotificationsPage)

> **경로:** `/notifications`
> **소스:** `frontend/src/features/notifications/pages/NotificationsPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 1-B (MVP 확장)
> **관련 기능:** F13 알림센터

---

## 1. 화면 개요

모든 역할의 사용자가 자신에게 발송된 알림을 실시간으로 확인하고 관리하는 화면이다.
게시 완료/실패, 승인 요청/결과, 위험 댓글 감지, 토큰 만료 등 8가지 유형의 알림을
유형별 아이콘과 색상으로 구분하여 표시하며, 읽지 않은 알림 개수를 배지로 보여준다.
알림 클릭 시 해당 리소스 화면으로 자동 이동하는 네비게이션 기능을 제공한다.

### 핵심 원칙
- **실시간 알림:** SSE + Redis Pub/Sub 기반 실시간 알림 수신
- **유형별 시각 구분:** 8가지 알림 유형별 고유 아이콘, 색상, 레이블 제공
- **빠른 액션:** 알림 클릭으로 관련 화면 직접 이동

## 6. 연결 흐름 (Connected Flows)

### 6.1 알림 수신 흐름

```
[서버 이벤트 발생]
       │
       ├── 게시 완료/실패 ──→ Celery publish 큐
       ├── 승인 요청/결과 ──→ 승인 서비스
       ├── 위험 댓글 감지 ──→ Celery ai 큐
       ├── 새 댓글 수신 ──→ Celery comment 큐
       ├── 토큰 만료 임박 ──→ Celery system 큐
       └── 시스템 공지 ──→ 관리자 직접 발송
              │
              ↓
       [Redis Pub/Sub] ──→ [SSE 실시간 전달] ──→ [알림 센터 목록 갱신]
                                                       │
                                                       ↓
                                                 [미읽음 배지 갱신]
                                                 (30초 폴링)
```

### 6.2 알림 클릭 네비게이션

```
[알림 클릭]
       │
       ├── PUBLISH_COMPLETE ──→ /contents/:id (콘텐츠 상세)
       ├── PUBLISH_FAILED ──→ /contents/:id (콘텐츠 상세)
       ├── APPROVAL_REQUEST ──→ /approvals/:id (승인 상세)
       ├── APPROVAL_RESULT ──→ /contents/:id (콘텐츠 상세)
       ├── DANGEROUS_COMMENT ──→ /comments (댓글 관리)
       ├── COMMENT_NEW ──→ /comments (댓글 관리)
       ├── TOKEN_EXPIRING ──→ /channels (채널 연동)
       └── SYSTEM ──→ action_url 또는 없음
```

### 6.3 알림 설정과의 연결

```
[알림 센터] ←──── 수신 대상 결정 ────→ [알림 설정 (/settings/notifications)]
                                            │
                                            ├── 웹 알림 ON/OFF
                                            ├── 텔레그램 알림 ON/OFF
                                            └── 웹 푸시 알림 ON/OFF
```

## 7. API 엔드포인트 연결

| 동작 | HTTP | 엔드포인트 | Hook | Phase |
|------|------|-----------|------|-------|
| 알림 목록 조회 | GET | `/notifications` | `useNotifications()` | 1-B |
| 미읽음 개수 조회 | GET | `/notifications/unread-count` | `useUnreadCount()` | 1-B |
| 개별 읽음 처리 | PATCH | `/notifications/:id/read` | `useMarkRead()` | 1-B |
| 전체 읽음 처리 | POST | `/notifications/mark-all-read` | `useMarkAllRead()` | 1-B |

### 쿼리 파라미터 (GET /notifications)

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `page` | number | 현재 페이지 (기본: 1) |
| `limit` | number | 페이지당 건수 (기본: 20) |
| `type` | string | 알림 유형 필터 (선택) |

### 미읽음 개수 폴링
| 항목 | 내용 |
|------|------|
| **주기** | 30초 (`refetchInterval: 30_000`) |
| **Hook** | `useUnreadCount()` |
| **용도** | 상단바 알림 배지 + 알림 센터 헤더 배지 실시간 갱신 |

## 8. 데이터 타입

### 8.1 알림 레코드 (NotificationRecord)

```typescript
interface NotificationRecord {
  id: string;                            // 알림 고유 ID (UUID)
  organization_id: string;               // 소속 기관 ID
  user_id: string;                       // 수신 사용자 ID
  type: NotificationType;                // 알림 유형 (8가지)
  channel: string;                       // 발송 채널 (web/email/telegram/webpush)
  title: string;                         // 알림 제목
  message: string;                       // 알림 본문 메시지
  payload: Record<string, unknown> | null; // 추가 데이터 (JSON)
  is_read: boolean;                      // 읽음 여부
  read_at: string | null;               // 읽음 시각 (ISO 8601)
  action_url: string | null;            // 클릭 시 이동 URL
  created_at: string;                    // 생성일시 (ISO 8601)
}
```

### 8.2 알림 유형 (NotificationType)

```typescript
type NotificationType =
  | 'PUBLISH_COMPLETE'    // 게시 완료
  | 'PUBLISH_FAILED'      // 게시 실패
  | 'APPROVAL_REQUEST'    // 승인 요청
  | 'APPROVAL_RESULT'     // 승인 결과
  | 'DANGEROUS_COMMENT'   // 위험 댓글
  | 'COMMENT_NEW'         // 새 댓글
  | 'TOKEN_EXPIRING'      // 토큰 만료
  | 'SYSTEM';             // 시스템
```

### 8.3 미읽음 개수 (UnreadCountData)

```typescript
interface UnreadCountData {
  count: number;  // 미읽음 알림 개수
}
```

### 8.4 전체 읽음 처리 응답

```typescript
interface MarkAllReadResponse {
  affected: number;  // 읽음 처리된 알림 개수
}
```

## 10. 오류 처리

| 상황 | 처리 |
|------|------|
| **알림 목록 로딩 실패** | 리스트 로딩 상태 유지, TanStack Query 자동 재시도 |

| **전체 읽음 처리 실패** | "일괄 읽음 처리에 실패했습니다" error toast |
| **미읽음 개수 조회 실패** | 배지 숨김 (0으로 처리), 30초 후 재시도 |
| **action_url 없는 알림 클릭** | 읽음 처리만 수행, 네비게이션 없음 |

## 11. 사용 시나리오

### 시나리오 1: 미읽은 알림 확인
1. 상단바 알림 아이콘의 배지 확인 (42)
2. 알림 센터 페이지 이동
3. 파란색 배경의 미읽은 알림 목록 확인
4. 관심 있는 알림 클릭 -- 자동 읽음 처리 + 관련 화면 이동

### 시나리오 2: 승인 요청 알림 확인
1. 알림 유형 필터에서 "승인 요청" 선택
2. 승인 요청 알림만 필터링되어 표시
3. 알림 클릭 -- 승인 상세 화면(`/approvals/:id`)으로 이동
4. 콘텐츠 검토 후 승인/반려 처리

### 시나리오 3: 위험 댓글 긴급 대응
1. 빨간색 "위험 댓글" 알림 확인
2. 알림 클릭 -- 댓글 관리 화면(`/comments`)으로 이동
3. 위험 댓글에 대한 답변 작성 또는 숨김/삭제 처리

### 시나리오 4: 전체 읽음 처리
1. [모두 읽음 처리] 버튼 클릭
2. "42개 알림을 읽음 처리했습니다" 성공 메시지
3. 모든 알림의 파란색 배경 제거 + 배지 0으로 갱신

### 시나리오 5: 토큰 만료 대응
1. "토큰 만료" 유형 알림 확인
2. 알림 클릭 -- 채널 연동 화면(`/channels`)으로 이동
3. 만료 임박 채널의 OAuth 토큰 재연결 수행
