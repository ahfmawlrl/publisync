# 콘텐츠 수정 (ContentEditPage)

> **경로:** `/contents/:id/edit`
> **소스:** `frontend/src/features/contents/pages/ContentEditPage.tsx` (983행)
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자)
> **Phase:** 1-A (기본 폼) + 1-B (AI·미리보기·미디어)
> **관련 기능:** F01 통합게시, F02 AI메타, F09 승인워크플로우, F16 AI썸네일, F22 번역

---

## 1. 화면 개요

기존 콘텐츠를 수정하는 화면이다. 새 콘텐츠 작성(ContentCreatePage)과 **동일한 2컬럼 레이아웃**을 사용하며,
좌측에 콘텐츠 편집 폼, 우측에 플랫폼별 미리보기 + AI 어시스턴트 패널을 배치한다.

`DRAFT`(초안) 또는 `REJECTED`(반려) 상태의 콘텐츠만 수정 가능하며,
기존 콘텐츠 데이터를 미리 채워(pre-fill) 표시한다.

### 핵심 원칙
- **기존 데이터 보존:** 수정 화면 진입 시 서버에서 최신 데이터를 조회하여 폼에 반영
- **상태 기반 접근:** DRAFT/REJECTED 상태만 수정 가능, 그 외 상태는 접근 차단
- **수정 후 확인:** 저장 성공 시 상세 페이지로 이동하여 변경 내용 즉시 확인
- **AI 워크플로우 내장:** 7개 AI 기능이 수정 화면 내에서 직접 호출 가능
- **Human-in-the-Loop:** AI 산출물은 항상 "제안" — 최종 결정은 사람이 한다

### 4.1 데이터 프리필 (Pre-fill)

| 항목 | 내용 |

| 폼 필드 | 데이터 소스 | 변환 |
|---------|-----------|------|
