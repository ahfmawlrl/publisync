# 자막 편집기 (SubtitleEditorPage)

> **경로:** `/ai/subtitle-editor/:assetId`
> **소스:** `frontend/src/features/ai/pages/SubtitleEditorPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자)
> **Phase:** 2
> **관련 기능:** F03 자막 자동 생성

---

## 1. 화면 개요

수탁업체 실무자(AO) 또는 관리자(AM)가 영상 콘텐츠의 자막을 AI로 자동 생성하고 수동으로 편집하는 화면이다.
좌측에 동영상 미리보기와 오디오 파형(Waveform)을, 우측에 자막 목록과 편집 도구를 배치한 2컬럼 레이아웃으로
구성된다. AI가 생성한 자막을 검토/수정한 뒤 SRT 파일로 내보낼 수 있다.

### 핵심 원칙
- **Human-in-the-Loop:** AI가 자막을 자동 생성하지만, 최종 편집과 확인은 사용자가 수행
- **Fallback 필수:** AI 자막 생성 실패 시에도 수동 추가로 작업 가능
- **워크플로우 내장 AI:** 자막 편집 화면 안에서 AI 생성을 직접 호출

### 3.4 자막 합성 영상 버튼

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Button` (icon=VideoCameraOutlined) |
| **레이블** | "자막 합성 영상" |
| **loading** | `burninMutation.isPending` (요청 중) |
| **disabled** | `subtitles.length === 0 \|\| isJobRunning` (자막 없거나 작업 진행 중) |
| **동작** | ① 자막 자동 저장 (`PUT /media/{assetId}/subtitles`) → ② `POST /ai/subtitle-burnin` → 202 Accepted (jobId 반환) → 폴링 시작 |

#### 자막 합성 처리 과정

1. 버튼 클릭 시 현재 편집 중인 자막을 먼저 서버에 자동 저장
2. 저장 성공 후 `POST /ai/subtitle-burnin` 요청 → Celery `ai` 큐에 비동기 작업 등록
3. 백엔드에서 ffmpeg `subtitles` 필터로 SRT 자막을 영상에 하드코딩(burn-in)
4. 완료 시 새 MediaAsset 생성 (parent_asset_id = 원본 에셋 ID)

#### 출력 파일 정보

| 항목 | 내용 |
|------|------|
| **저장 위치** | MinIO/S3: `{org_id}/media/subtitle-burnin/{uuid}_{원본파일명}_subtitled.mp4` |
| **로컬 개발** | `storage/media/{org_id}/subtitle-burnin/{uuid}_{원본파일명}_subtitled.mp4` |
| **미디어 라이브러리** | 합성 완료 후 미디어 라이브러리에 자동 등록 (parent_asset_id로 원본 추적 가능) |
| **재생 방법** | 미디어 라이브러리에서 해당 파일 선택 → VideoPlayer로 재생, 또는 다운로드 후 로컬 재생 |
| **코덱** | H.264 (libx264), 원본 해상도 유지 |

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `AiJobProgress` (자체 컴포넌트) |
| **표시 조건** | `isJobRunning` (jobId가 있을 때) |

| **폴링** | 3초 간격 GET `/ai/jobs/{jobId}` (useJobStatus) |

#### 작업 상태 표시

| 상태 | Tag 색상 | 아이콘 | 라벨 |
|------|---------|--------|------|
| PENDING | default | Clock | 대기 중 |
| PROCESSING | processing | Loader | 처리 중 |
| COMPLETED | success | CheckCircle | 완료 |
| FAILED | error | XCircle | 실패 |

#### 완료 시 동작 (handleJobComplete)

| 항목 | 내용 |
|------|------|
| **결과 파싱** | `result.subtitles` 배열에서 `SubtitleEntry[]` 변환 |
| **매핑** | `start` (문자열), `end` (문자열), `text` (문자열) 추출 |
| **성공** | "{N}개 자막이 생성되었습니다" toast |
| **파싱 실패** | "자막 결과를 파싱할 수 없습니다. 수동으로 입력해주세요." warning toast (Fallback) |
| **중복 방지** | `resultConsumedRef`로 결과가 한 번만 처리되도록 보장 |

## 6. SRT 형식 처리

### 6.1 SRT 시간 형식

| 방향 | 함수 | 설명 |
|------|------|------|
| SRT → 초 | `srtTimeToSeconds("HH:MM:SS,mmm")` | "00:01:30,500" → 90.5 |
| 초 → SRT | `secondsToSrtTime(totalSeconds)` | 90.5 → "00:01:30,500" |

### 6.2 SRT 파일 생성 (generateSrt)

```
1
00:00:01,000 --> 00:00:03,000
첫 번째 자막

2
00:00:03,500 --> 00:00:06,000
두 번째 자막
```

### 6.3 파일 다운로드 (downloadAsFile)

| 항목 | 내용 |
|------|------|
| **MIME 타입** | `text/plain;charset=utf-8` |
| **방식** | Blob → URL.createObjectURL → 임시 앵커 클릭 → revokeObjectURL |
| **파일명** | `subtitles_{assetId}.srt` |

## 7. 연결 흐름 (Connected Flows)

### 7.1 AI 자막 생성 흐름

```
[AI 자막 생성 버튼 클릭]
       │
       ↓
POST /ai/generate-subtitles (media_asset_id) → 202 Accepted
       │
       ↓
[jobId 반환] → AiJobProgress 표시
       │
       ↓
[3초 간격 폴링] GET /ai/jobs/:jobId
       │
       ├──(PENDING)──→ 대기 중 표시
       ├──(PROCESSING)──→ 처리 중 + Progress 바
       ├──(FAILED)──→ 오류 메시지 Alert
       └──(COMPLETED)──→ result.subtitles 파싱
                              │
                        ┌─────┴──────┐
                        ↓            ↓
                  [파싱 성공]    [파싱 실패]
                  N개 자막 목록   "수동으로 입력해주세요" (Fallback)
                  에 로드
```

### 7.2 자막 편집 흐름

```
[자막 목록]
       │
       ├──(시간 수정)──→ 시작/종료 Input 값 변경 → handleUpdateEntry
       │
       ├──(텍스트 수정)──→ TextArea 값 변경 → handleUpdateEntry
       │
       ├──(항목 추가)──→ handleAddEntry
       │                   │
       │                   ├──(기존 자막 있음)──→ 마지막 항목의 end 시간 기준 +2초 구간 생성
       │                   └──(자막 없음)──→ 00:00:00,000 ~ 00:00:01,000 빈 항목 생성
       │
       └──(항목 삭제)──→ handleRemoveEntry → 나머지 항목 재인덱싱 (1부터)
```

### 7.3 자막 합성 영상 흐름

```
[자막 합성 영상 버튼 클릭]
       │
       ├──(자막 없음)──→ "합성할 자막이 없습니다" warning toast
       │
       └──(자막 있음)──→ ① PUT /media/{assetId}/subtitles (자동 저장)
                              │
                        ┌─────┴──────┐
                        ↓            ↓
                  [저장 성공]    [저장 실패]
                        │       "자막 저장에 실패하여
                        │        합성을 진행할 수 없습니다" error toast
                        ↓
              ② POST /ai/subtitle-burnin → 202 Accepted
                        │
                        ↓
              [burninJobId 반환] → AiJobProgress 표시
                        │
                        ↓
              [3초 간격 폴링] GET /ai/jobs/:jobId
                        │
                        ├──(PROCESSING)──→ ffmpeg 진행률 표시
                        ├──(FAILED)──→ 오류 메시지 Alert
                        └──(COMPLETED)──→ 새 MediaAsset 생성 완료
                                           │
                                           ↓
                                   "자막이 합성된 영상이 생성되었습니다.
                                    미디어 라이브러리에서 확인하세요."
                                    success toast
```

### 7.4 SRT 내보내기 흐름

```
[SRT 내보내기 버튼 클릭]
       │
       ├──(자막 없음)──→ "내보낼 자막이 없습니다" warning toast
       │
       └──(자막 있음)──→ generateSrt() → downloadAsFile()
                              │
                              ↓
                        "SRT 파일을 다운로드했습니다" success toast
                        (subtitles_{assetId}.srt 파일 저장)
```

## 9. 데이터 타입

### 9.1 자막 항목 (SubtitleEntry, 컴포넌트 내부)

```typescript
interface SubtitleEntry {
  index: number;           // 자막 번호 (1부터 시작)
  start: string;           // SRT 시작 시간 "HH:MM:SS,mmm"
  end: string;             // SRT 종료 시간 "HH:MM:SS,mmm"
  text: string;            // 자막 텍스트
}
```

### 9.2 자막 생성 요청 (SubtitleRequest)

```typescript
interface SubtitleRequest {
  media_asset_id: string;       // 대상 미디어 에셋 ID
  language?: string;             // 언어 (선택)
  include_timestamps?: boolean;  // 타임스탬프 포함 여부
}
```

### 9.3 AI 작업 생성 응답 (AiJobCreateResponse)

```typescript
interface AiJobCreateResponse {
  job_id: string;          // 작업 ID (폴링 키)
  job_type: string;        // 작업 유형 (SUBTITLE)
  status: string;          // 초기 상태
  message: string;         // 안내 메시지
}
```

### 9.4 AI 작업 상태 (AiJobStatus)

```typescript
interface AiJobStatus {
  job_id: string;
  job_type: string;
  status: 'PENDING' | 'PROCESSING' | 'COMPLETED' | 'FAILED';
  progress: number;                       // 0~100
  result: Record<string, unknown> | null; // 완료 시 { subtitles: [...] }
  error_message: string | null;           // 실패 시 오류 메시지
  started_at: string | null;
  completed_at: string | null;
  created_at: string;
}
```

## 10. 상태 관리

| 영역 | 도구 | 설명 |
|------|------|------|

| **작업 상태 폴링** | TanStack Query `useQuery` | useJobStatus (3초 간격, refetchInterval) |

| **AI 자막 생성 요청 실패** | "AI 자막 생성 요청에 실패했습니다" error toast |
| **AI 작업 실패 (FAILED)** | AiJobProgress 내 Alert (error) — error_message 표시 |
| **작업 결과 파싱 실패** | "자막 결과를 파싱할 수 없습니다. 수동으로 입력해주세요." warning toast (Fallback) |
| **SRT 내보내기 시 자막 없음** | "내보낼 자막이 없습니다" warning toast |
| **작업 정보 조회 불가** | "작업 정보를 불러올 수 없습니다." Alert (warning) |

## 12. 사용 시나리오

### 시나리오 1: AI 자막 자동 생성
1. 미디어 라이브러리에서 동영상 에셋 선택 → 자막 편집기 진입
2. 좌측에서 동영상 미리보기 + 오디오 파형 확인
3. [AI 자막 생성] 클릭 → "AI 자막 생성 작업을 시작했습니다" toast
4. AiJobProgress에서 진행률 확인 (대기 중 → 처리 중 → 완료)
5. 완료 시 "15개 자막이 생성되었습니다" toast → 우측 자막 목록에 자동 로드

### 시나리오 2: 자막 수동 편집
1. AI 생성된 자막 목록에서 시작/종료 시간 수정 (SRT 형식 직접 입력)
2. 자막 텍스트 수정 (오타 교정, 표현 변경)
3. 불필요한 자막 항목 [삭제] → 나머지 항목 자동 재인덱싱
4. [+추가] 버튼으로 빈 자막 항목 추가 (이전 항목 종료 시간 기준)

### 시나리오 3: 자막 합성 영상 생성
1. AI 자막 생성 또는 수동 편집으로 자막 완성
2. [서버에 저장] 클릭하여 자막 데이터 저장 (또는 합성 시 자동 저장)
3. [자막 합성 영상] 버튼 클릭 → "자막 합성 영상 생성을 시작했습니다" toast
4. AiJobProgress에서 ffmpeg 진행률 확인 (0% → 50% → 100%)
5. 완료 시 "자막이 합성된 영상이 생성되었습니다. 미디어 라이브러리에서 확인하세요." toast
6. **결과 파일 확인:** 미디어 라이브러리 → 새로 생성된 `{원본파일명}_subtitled.mp4` 선택 → 재생

#### 출력 파일 저장 위치 및 재생 방법

| 항목 | 설명 |
|------|------|
| **저장 위치** | 프로덕션: MinIO `{org_id}/media/subtitle-burnin/` 하위 |
| | 로컬 개발: `storage/media/{org_id}/subtitle-burnin/` 하위 |
| **파일명** | `{uuid}_{원본파일명}_subtitled.mp4` |
| **재생 방법** | ① 미디어 라이브러리에서 해당 파일 선택 → 내장 VideoPlayer로 재생 |
| | ② 미디어 라이브러리에서 다운로드 → 로컬 미디어 플레이어(VLC 등)로 재생 |
| **원본 추적** | `parent_asset_id`로 원본 영상과 연결 (미디어 라이브러리에서 원본 참조 가능) |

### 시나리오 4: SRT 파일 내보내기
1. 자막 편집 완료 후 [SRT 내보내기] 클릭
2. `subtitles_{assetId}.srt` 파일 자동 다운로드
3. "SRT 파일을 다운로드했습니다" toast 확인
4. 다운로드된 SRT 파일을 영상 편집 도구에서 활용

### 시나리오 5: AI 실패 시 Fallback
1. [AI 자막 생성] 클릭 → 작업 시작
2. 서버 오류로 작업 FAILED → AiJobProgress에 오류 메시지 표시
3. [닫기] 버튼으로 진행 상황 카드 닫기
4. [+추가] 버튼으로 수동 자막 항목 추가하여 작업 계속
