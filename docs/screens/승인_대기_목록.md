# 승인 대기 목록 (ApprovalsListPage)

> **경로:** `/approvals`
> **소스:** `frontend/src/features/approvals/pages/ApprovalsListPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F09 승인워크플로우

---

## 1. 화면 개요

수탁업체 실무자(AO)가 작성한 콘텐츠의 검토 요청을 위탁기관 담당자(CD)가 승인 또는 반려하는 화면이다.
탭 기반 필터링으로 "내가 검수할 항목", "내가 요청한 항목", "전체"를 분류하며,
카드 목록에서 콘텐츠 제목, 긴급도 뱃지, 상태 태그를 한눈에 확인할 수 있다.
카드를 클릭하면 상세 검수 패널로 전환되어 콘텐츠 미리보기, AI 검수 결과, 승인 이력을 확인하고
승인/반려 액션을 수행할 수 있다.

### 핵심 원칙
- **Human-in-the-Loop:** AI 검수 결과는 참고 자료이며, 최종 승인/반려 결정은 사람이 한다
- **Fallback 필수:** AI 검수 실패 시에도 수동 검토로 승인/반려 가능
- **워크플로우 내장 AI:** AI 검수 결과가 승인 화면 안에서 인라인으로 표시됨

| 키 | 라벨 | API 필터 | 용도 |
|----|------|---------|------|

| `all` | 전체 | 필터 없음 (전체) | 모든 승인 요청 전체 조회 |

#### 첫째 줄: 뱃지 + 제목 + 상태 + 플랫폼 태그

| 요소 | 컴포넌트 | 설명 |

| 상태 | 색상 | 텍스트 |

| `REJECTED` | red | 반려됨 |

| 플랫폼 | 색상 | 라벨 |

## 8. 모달 대화상자

### 8.1 반려 사유 모달

| 항목 | 내용 |
|------|------|
| **트리거** | 상세 검수 패널 → "반려" 버튼 클릭 |
| **컴포넌트** | `Modal` (Ant Design) |
| **제목** | "반려 사유" |
| **상태 관리** | `useState<string | null>(null)` — rejectModalId |
| **열림 조건** | `!!rejectModalId` |

#### 모달 내부 구성

| 요소 | 설명 |
|------|------|
| **반려 사유 입력** | `TextArea` (rows=4, placeholder="반려 사유를 입력하세요") |
| **상태 관리** | `rejectComment` 공유 (검수 의견 필드와 동일) |

#### 모달 버튼

| 버튼 | 동작 | 로딩 |
|------|------|------|
| **반려** (okText) | `handleReject()` — 반려 API 호출 | `rejectMutation.isPending` |
| **취소** (cancelText) | `setRejectModalId(null)` + `setRejectComment('')` | - |

#### handleReject 처리 흐름

1. `rejectMutation.mutate({ id: rejectModalId, comment: rejectComment })`
2. 성공 시: `APPROVAL_MESSAGES.REJECT_SUCCESS` ("반려가 완료되었습니다") → 모달 닫기 → 검수 의견 초기화 → 목록으로 복귀
3. 실패 시: `APPROVAL_MESSAGES.REJECT_ERROR` ("반려 처리에 실패했습니다") 에러 메시지

#### handleApprove 처리 흐름

1. `approveMutation.mutate({ id })`
2. 성공 시: `APPROVAL_MESSAGES.APPROVE_SUCCESS` ("승인이 완료되었습니다") → 목록으로 복귀
3. 실패 시: `APPROVAL_MESSAGES.APPROVE_ERROR` ("승인 처리에 실패했습니다") 에러 메시지

## 9. 연결 흐름 (Connected Flows)

### 9.1 콘텐츠 작성 → 검토 요청 → 승인/반려 흐름

```
[새 콘텐츠 작성] (/contents/create)
       │
       └──(검토 요청)──→ [승인 대기 목록] (/approvals)
                              │
                        ┌─────┴──────┐
                        ↓            ↓
                   [CD: 승인]   [CD: 반려 + 사유]
                        │            │
                        ↓            ↓
                 [자동 게시]    [AO/AM: 반려 사유 확인]
                 (예약/즉시)         │
                                    └──→ [콘텐츠 수정] (/contents/:id)
                                               │
                                               └──→ [재검토 요청] → [승인 대기 목록]
```

### 9.2 탭 필터링 흐름

```
[승인 대기 목록]
       │
       ├──[내가 검수할 항목]──→ status=PENDING_REVIEW 필터
       │                         (CD가 검수 대기 중인 항목)
       │
       ├──[내가 요청한 항목]──→ requested_by=currentUserId 필터
       │                         (AO/AM이 자기가 요청한 항목만 조회)
       │
       └──[전체]──→ 전체 조회
                     (모든 승인 요청 이력)
```

### 9.3 상세 검수 흐름

```
[카드 클릭] ──→ [상세 검수 패널]
                     │
                     ├──→ [콘텐츠 내용 확인] (좌측)
                     │       └──→ 미디어 미리보기, 콘텐츠 ID, 상태, 긴급 여부
                     │
                     ├──→ [AI 검수 결과 확인] (우측 상단)
                     │       └──→ 표현 적합성, 개인정보, 이미지 대체텍스트
                     │
                     ├──→ [승인 이력 확인] (우측 중단)
                     │       └──→ 요청 일자, 요청자, 코멘트
                     │
                     └──→ [검수 의견 + 액션] (우측 하단)
                             │
                             ├──→ [승인 후 게시] ──→ POST /approvals/:id/approve
                             │                         │
                             │                         └──→ 콘텐츠 상태 → APPROVED → 자동 게시
                             │
                             └──→ [반려] ──→ [반려 사유 모달] ──→ POST /approvals/:id/reject
                                                                    │
                                                                    └──→ 콘텐츠 상태 → REJECTED
```

## 14. 사용 시나리오

### 시나리오 1: CD가 긴급 콘텐츠 승인

1. CD가 사이드바 > 승인 클릭 → `/approvals` 진입
2. "내가 검수할 항목" 탭에서 빨간 좌측 경계의 [긴급] 카드 확인
3. [검수하기] 클릭 → 상세 검수 패널 전환
4. 좌측: 콘텐츠 미리보기, 콘텐츠 ID, 긴급 여부 확인
5. 우측: AI 검수 결과 확인 (표현 적합성 통과, 개인정보 미검출)
6. 검수 의견에 "확인 완료" 입력
7. [승인 후 게시] 클릭 → "승인되었습니다" 메시지 → 목록으로 복귀
8. 콘텐츠가 자동 게시됨 (예약 시간 또는 즉시)

### 시나리오 2: CD가 콘텐츠 반려

1. "내가 검수할 항목" 탭에서 콘텐츠 카드 클릭
2. 상세 검수 패널에서 AI 검수 결과 확인 — 이미지 대체텍스트 미입력 발견
3. [반려] 클릭 → 반려 사유 모달 열림
4. "이미지 대체텍스트 누락. 접근성 기준 미충족으로 수정 요청합니다." 입력
5. [반려] 클릭 → "반려되었습니다" 메시지 → 목록으로 복귀
6. AO/AM에게 반려 알림 발송, 수정 후 재검토 요청 가능

### 시나리오 3: AO가 자신의 요청 상태 확인

1. AO가 승인 대기 목록 진입
2. "내가 요청한 항목" 탭 클릭
3. 자신이 요청한 콘텐츠들의 현재 상태 확인 (검토 대기 / 검토 중 / 승인됨 / 반려됨)
4. 반려된 항목 카드 클릭 → 반려 사유 확인
5. [← 목록으로] → 콘텐츠 수정 화면으로 이동하여 수정 후 재요청

### 시나리오 4: 전체 승인 이력 조회

1. AM이 "전체" 탭 클릭
2. 모든 승인 요청 이력을 시간순으로 확인
3. 필요 시 페이지네이션으로 이전 이력 탐색
4. 승인됨/반려됨 상태 태그로 처리 현황 파악
