# 위험 댓글 관리 (DangerousCommentsPage)

> **경로:** `/comments/dangerous`
> **소스:** `frontend/src/features/comments/pages/DangerousCommentsPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 1-B (MVP 확장)
> **관련 기능:** F04 댓글관리

---

## 1. 화면 개요

AI 감성 분석에서 "위험(DANGEROUS)"으로 분류된 댓글만 모아 보여주는 전용 관리 화면이다.
신뢰도 80% 이상의 고위험 댓글은 "긴급" 태그로 강조 표시되며,
미처리/처리완료/아카이브 탭으로 처리 상태를 구분하여 빠른 대응을 지원한다.
각 댓글에 대해 AI 답글 생성, 숨김 처리, 삭제 승인 요청, 무시(위험 해제) 등
4가지 빠른 조치를 즉시 수행할 수 있다.

### 핵심 원칙
- **Human-in-the-Loop:** AI 감성 분석은 "분류 제안"이며, 무시(위험 해제) 기능으로 사용자가 최종 판단
- **Fallback 필수:** AI 답글 실패 시에도 수동 답글 작성 가능
- **워크플로우 내장 AI:** AI 답글 기능이 위험 댓글 카드 내에서 직접 호출됨

## 5. 위험 댓글 카드

각 위험 댓글은 개별 카드 형태로 표시된다.

### 5.1 카드 스타일
| 항목 | 내용 |
|------|------|
| **컴포넌트** | `div` (rounded-lg, shadow-sm, p-4, 좌측 4px 테두리) |
| **좌측 테두리** | 긴급(80%+): #ff4d4f (빨강), 주의(80% 미만): #faad14 (주황) |
| **간격** | mb-3 (카드 간 12px) |

### 5.2 긴급도 판별 기준

| 신뢰도 | 분류 | Tag | 좌측 테두리 |
|--------|------|-----|-----------|
| 80% 이상 | 긴급 | `Tag` color="red", 텍스트 "긴급" | #ff4d4f |
| 80% 미만 | 주의 | `Tag` color="gold", 텍스트 "주의" | #faad14 |

### 5.3 카드 구성 요소

| 요소 | 설명 |
|------|------|
| **긴급도 태그** | [긴급] 또는 [주의] Tag |
| **메타 정보** | "{상대시간} · {플랫폼명} · 콘텐츠 {content_id 앞 8자}" (text-xs, type="secondary") |
| **댓글 내용** | "{작성자명}: \"{댓글 텍스트}\"" (text-sm, 따옴표로 감싸 표시) |
| **감성 정보** | "감성: 위험 ({신뢰도}%) · 키워드: {키워드1}, {키워드2}" (text-xs, text-gray-500) |
| **액션 버튼** | 4개 빠른 조치 버튼 (flex-wrap 가로 배치) |

### 5.4 시간 표시 로직 (formatTime)

| 조건 | 표시 |
|------|------|
| 60분 미만 | `{분}분 전` |
| 24시간 미만 | `{시간}시간 전` |
| 24시간 이상 | `ko-KR` 로케일 날짜 |

우선순위: `platform_created_at` > `created_at`

### 5.5 플랫폼 표시 (PLATFORM_SHORT)

| 값 | 표시명 |
|----|--------|
| `YOUTUBE` | YouTube |
| `INSTAGRAM` | Instagram |
| `FACEBOOK` | Facebook |
| `X` | X |
| `NAVER_BLOG` | 네이버 블로그 |

## 9. 연결 흐름 (Connected Flows)

### 9.1 위험 댓글 대응 흐름

```
[위험 댓글 관리 진입] (/comments/dangerous)
       │
       ↓
[미처리 탭] ── 위험 댓글 목록 표시
       │
       ├──→ [AI 답글 생성] → [답변 모달] → 답글 입력 → [답변 등록]
       │         │                                        │
       │         ↓                                        ↓
       │    POST /comments/:id/reply            목록 갱신 + 처리 완료 탭 이동
       │
       ├──→ [숨김 처리] → Popconfirm → POST /comments/:id/hide
       │                                    │
       │                                    ↓
       │                              아카이브 탭으로 이동
       │
       ├──→ [삭제 승인 요청] → Popconfirm → POST /comments/:id/delete-approve
       │                                          │
       │                                          ↓
       │                                    삭제 승인 대기 상태
       │
       └──→ [무시] → Popconfirm → POST /comments/:id/ignore
                                        │
                                        ↓
                                  위험 표시 해제 (목록에서 제거)
```

### 9.2 통합 댓글함 연계

```
[통합 댓글함] (/comments) ──(🔴 위험 탭)──→ 위험 감성 필터링
       │
       └──→ [위험 댓글 관리] (/comments/dangerous) ← 전용 관리 화면
                    │
                    └──→ 처리 완료 후 [통합 댓글함]에서도 상태 반영
```

### 9.3 처리 상태 전이

```
미처리 (UNPROCESSED)
    │
    ├──(답글 게시)──→ 처리 완료 (PUBLISHED)
    │
    ├──(숨김 처리)──→ 아카이브 (HIDDEN)
    │
    ├──(삭제 승인 요청)──→ 삭제 대기 (PENDING_DELETE)
    │
    └──(무시)──→ 위험 해제 (목록에서 제거)
```

## 10. API 엔드포인트 연결

| 동작 | HTTP | 엔드포인트 | Hook | Phase |

## 11. 데이터 타입

### 11.1 댓글 레코드 (CommentRecord)

```typescript
interface CommentRecord {
  id: string;                          // 댓글 고유 ID
  organization_id: string;             // 기관 ID (RLS)
  content_id: string | null;           // 원본 콘텐츠 ID
  channel_id: string;                  // 채널 ID
  platform: string;                    // 플랫폼 (YOUTUBE/INSTAGRAM/FACEBOOK/X/NAVER_BLOG)
  external_id: string;                 // 플랫폼 외부 ID
  text: string;                        // 댓글 본문
  author_name: string;                 // 작성자 이름
  author_profile_url: string | null;   // 작성자 프로필 URL
  parent_comment_id: string | null;    // 부모 댓글 ID (대댓글인 경우)
  sentiment: string | null;            // 감성 분석 결과 (위험 댓글은 항상 DANGEROUS)
  sentiment_confidence: number | null;  // 감성 분석 신뢰도 (0~1)
  dangerous_level: string | null;      // 위험 수준
  keywords: string[] | null;           // 추출 키워드 배열
  status: string;                      // 상태 (UNPROCESSED/PUBLISHED/HIDDEN/PENDING_DELETE/DELETED)
  reply_text: string | null;           // 게시된 답글 텍스트
  reply_draft: string | null;          // 답글 임시 저장 텍스트
  replied_at: string | null;           // 답글 게시 일시
  hidden_reason: string | null;        // 숨김 사유
  delete_reason: string | null;        // 삭제 사유
  processed_by: string | null;         // 처리자 ID
  platform_created_at: string | null;  // 플랫폼 원본 작성 일시
  created_at: string;                  // 시스템 생성 일시
  updated_at: string;                  // 시스템 수정 일시
}
```

## 12. 상태 관리

| 영역 | 도구 | 상태 변수 | 설명 |
|------|------|----------|------|
| **페이지** | React `useState` | `page` | 현재 페이지 번호 |
| **처리 상태 탭** | React `useState` | `activeTab` | 선택된 탭 (unprocessed/processed/archived) |
| **답변 대상** | React `useState` | `replyComment` | 답변할 댓글 객체 (CommentRecord | null) |
| **답변 모달** | React `useState` | `replyOpen` | 답변 모달 열림 상태 |
| **답변 텍스트** | React `useState` | `replyText` | 답변 입력 내용 |
| **서버 상태** | TanStack Query | `useDangerousComments()` | 위험 댓글 목록 쿼리 (queryKey: ['comments', 'dangerous', params]) |
| **답글 게시** | TanStack Query | `useReplyComment()` | 답글 게시 뮤테이션 |
| **숨김 처리** | TanStack Query | `useHideComment()` | 댓글 숨김 뮤테이션 |
| **삭제 승인** | TanStack Query | `useApproveDelete()` | 삭제 승인 요청 뮤테이션 |
| **위험 해제** | TanStack Query | `useIgnoreDangerous()` | 위험 표시 해제 뮤테이션 |

### 캐시 무효화 전략

모든 댓글 관련 뮤테이션(reply, hide, approveDelete, ignoreDangerous)은 성공 시 `queryKey: ['comments']`를 무효화하여 위험 댓글 목록과 통합 댓글함을 동시에 갱신한다.

## 13. 오류 처리 및 Fallback

| 상황 | 처리 |

| **숨김 처리 성공** | "댓글이 숨김 처리되었습니다" success toast |

| **삭제 요청 성공** | "삭제 요청이 등록되었습니다" success toast |
| **삭제 요청 실패** | "삭제 요청에 실패했습니다" error toast |
| **위험 해제 성공** | "위험 표시가 해제되었습니다" success toast |

## 14. 사용 시나리오

### 시나리오 1: 긴급 위험 댓글 즉시 대응
1. 사이드바 > 댓글 > 위험 댓글 관리 진입
2. 상단 "🔴 3건 대기" 배지로 미처리 건수 확인
3. 미처리 탭에서 [긴급] 태그(신뢰도 95%)가 붙은 댓글 확인
4. [숨김 처리] 클릭 → Popconfirm "숨기기" → 즉시 숨김
5. 해당 댓글이 아카이브 탭으로 이동

### 시나리오 2: AI 답글로 위험 댓글 대응
1. 위험 댓글 카드에서 [AI 답글 생성] 클릭
2. 답변 모달 열림, 원본 댓글이 빨간 배경으로 인용 표시
3. 답변 내용을 직접 입력 (또는 통합 댓글함의 AI 제안 활용)
4. [답변 등록] 클릭 → 답글 게시 → 처리 완료 탭으로 이동

### 시나리오 3: 위험 오탐(False Positive) 해제
1. [주의] 태그(신뢰도 65%)가 붙은 댓글 내용 확인
2. 실제로는 위험하지 않다고 판단
3. [무시] 클릭 → Popconfirm "해제" → 위험 표시 해제
4. 해당 댓글이 위험 댓글 목록에서 제거됨

### 시나리오 4: 삭제 승인 요청
1. 명백한 욕설/비방 댓글 확인 (신뢰도 98%)
2. [삭제 승인 요청] 클릭 → Popconfirm "요청"
3. 삭제 요청이 등록되어 관리자 승인 대기
4. 승인 후 해당 플랫폼에서 댓글 삭제 처리

### 시나리오 5: 처리 이력 확인
1. [처리 완료] 탭 클릭 → 답글 게시 완료된 위험 댓글 목록
2. [아카이브] 탭 클릭 → 숨김 처리된 위험 댓글 목록
3. 과거 대응 이력을 참조하여 일관된 대응 유지
