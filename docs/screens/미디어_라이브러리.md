# 미디어 라이브러리 (MediaLibraryPage)

> **경로:** `/media`
> **소스:** `frontend/src/features/media/pages/MediaLibraryPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 2
> **관련 기능:** F11 미디어 라이브러리

---

## 1. 화면 개요

수탁업체 실무자(AO), 관리자(AM), 위탁기관 담당자(CD)가 콘텐츠에 사용할 이미지, 동영상, 오디오, 문서 파일을
통합 관리하는 화면이다. 폴더 트리 구조로 파일을 분류하고, 그리드/리스트 뷰 전환, 미디어 유형/태그/검색 필터,
드래그 앤 드롭 업로드, 파일 상세 미리보기(이미지/동영상/오디오), 메타데이터 수정 기능을 제공한다.

### 핵심 원칙
- **폴더 기반 분류:** 트리 구조 폴더로 미디어 자산을 체계적으로 정리
- **다중 뷰 모드:** 그리드(썸네일) / 리스트(상세) 뷰 전환으로 작업 효율 극대화
- **Presigned URL 업로드:** MinIO에 직접 업로드하여 서버 부하 최소화

## 8. 연결 흐름 (Connected Flows)

### 8.1 파일 업로드 흐름

```
[업로드 버튼 클릭] → [업로드 모달 열기]
       │
       ↓
[드래그 앤 드롭 / 클릭] → [파일 유효성 검사 (형식, 크기 100MB)]
       │
       ├──(실패)──→ 오류 toast (파일명 + 오류 메시지)
       │
       └──(성공)──→ POST /media/presigned-upload → Presigned URL 발급
                          │
                          ↓
                    PUT (MinIO XHR 직접 업로드) → Progress 바 표시
                          │
                          ↓
                    POST /media/upload → 에셋 레코드 생성
                          │
                          ↓
                    [업로드 완료 toast] → 1.5초 후 진행 항목 제거
```

### 8.2 파일 관리 흐름

```
[그리드/리스트 뷰 파일 카드]
       │
       ├──(👁 상세)──→ [상세 모달] → 미리보기(이미지/동영상/오디오) + 메타데이터
       │
       ├──(✏ 수정)──→ [수정 모달] → 파일명/태그/폴더 수정 → PUT /media/:id
       │
       └──(🗑 삭제)──→ [Popconfirm "이 미디어를 삭제하시겠습니까?"]
                          │
                          ├──(삭제)──→ DELETE /media/:id → 성공 toast
                          └──(취소)──→ 닫기
```

### 8.3 폴더 관리 흐름

```
[폴더 사이드바]
       │
       ├──(폴더 선택)──→ selectedFolderId 변경 → 해당 폴더 파일 조회 (page=1)
       │
       ├──("전체 파일" 선택)──→ selectedFolderId=undefined → 전체 파일 조회
       │
       └──(📁+ 새 폴더)──→ [폴더 생성 모달] → POST /media/folders → 성공 toast
```

## 9. API 엔드포인트 연결

| 동작 | HTTP | 엔드포인트 | Hook | Phase |
|------|------|-----------|------|-------|
| 미디어 목록 조회 | GET | `/media` | `useMediaList()` | 2 |
| 미디어 상세 조회 | GET | `/media/:id` | `useMediaAsset()` | 2 |
| 미디어 썸네일 | GET | `/media/:id/thumbnail` | (직접 URL) | 2 |
| 미디어 다운로드 | GET | `/media/:id/download` | (직접 URL) | 2 |
| Presigned URL 발급 | POST | `/media/presigned-upload` | `usePresignedUpload()` | 2 |
| 미디어 에셋 등록 | POST | `/media/upload` | `useUploadMedia()` | 2 |
| 미디어 정보 수정 | PUT | `/media/:id` | `useUpdateMedia()` | 2 |
| 미디어 삭제 | DELETE | `/media/:id` | `useDeleteMedia()` | 2 |
| 폴더 목록 조회 | GET | `/media/folders` | `useMediaFolders()` | 2 |

## 13. 사용 시나리오

### 시나리오 1: 파일 업로드
1. 사이드바 > 미디어 라이브러리 클릭
2. [업로드] 버튼 클릭 → 업로드 모달 열림
3. 파일을 드래그 앤 드롭 (또는 클릭하여 선택)
4. 업로드 진행률 확인 (파일별 Progress 바)
5. "업로드 완료" toast 확인
6. 모달 닫기 → 목록에 새 파일 표시

### 시나리오 2: 폴더 기반 파일 정리
1. 좌측 폴더 사이드바에서 [+] 버튼 클릭 → 폴더 생성 모달
2. 폴더명 "2024년 3월 캠페인" 입력, 상위 폴더 선택
3. [생성] 클릭 → 폴더 트리에 새 폴더 추가
4. 기존 파일의 [수정] 버튼 클릭 → 폴더 변경
5. 폴더 트리에서 해당 폴더 클릭 → 해당 폴더 파일만 표시

### 시나리오 3: 미디어 검색 및 필터링
1. 뷰 모드를 "리스트"로 전환
2. 미디어 유형 필터에서 "동영상" 선택
3. 태그 필터에 "캠페인" 입력
4. 검색창에 "서울시" 입력 → 조건에 맞는 파일만 표시
5. 파일 카드의 [상세] 아이콘 클릭 → 상세 모달에서 동영상 미리보기

### 시나리오 4: 미디어 메타데이터 수정 및 삭제
1. 그리드 뷰에서 파일 카드의 [수정] 아이콘 클릭
2. 파일명 변경, 태그 추가, 폴더 이동
3. [저장] 클릭 → "미디어 정보가 수정되었습니다" toast
4. 불필요한 파일의 [삭제] 아이콘 클릭 → 확인 팝업 → "미디어가 삭제되었습니다"
