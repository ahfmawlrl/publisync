# 알림 설정 (NotificationSettingsPage)

> **경로:** `/settings/notifications`
> **소스:** `frontend/src/features/notifications/pages/NotificationSettingsPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 1-B (MVP 확장)
> **관련 기능:** F13 알림센터

---

## 1. 화면 개요

모든 역할의 사용자가 알림 수신 채널(웹 알림, 텔레그램, 웹 푸시)별로 수신 여부를 설정하는 화면이다.
각 채널별로 7가지 알림 유형(위험 댓글, 승인 요청, 승인 결과, 게시 완료, 게시 실패, 토큰 만료, 시스템 공지)에 대한
개별 토글 스위치를 제공하며, 텔레그램 연동을 위한 Chat ID 입력 및 테스트 발송 기능을 포함한다.

### 핵심 원칙
- **채널별 세분화:** 웹/텔레그램/웹 푸시 3개 채널별 독립 설정
- **유형별 제어:** 알림 유형별 개별 ON/OFF 토글
- **테스트 발송:** 텔레그램 설정 시 테스트 알림으로 연결 검증

## 5. 알림 유형 목록 (7가지)

모든 탭에서 공통으로 사용되는 알림 유형 정의:

| key | 한글 레이블 | 설명 |
|-----|-----------|------|
| `dangerous_comment` | 위험 댓글 감지 | 위험 수준의 댓글이 감지되면 알림 |
| `approval_request` | 승인 요청 | 내가 검수자일 때 승인 요청 알림 |
| `approval_result` | 승인/반려 결과 | 내가 요청한 콘텐츠의 승인/반려 결과 |
| `publish_complete` | 게시 완료 | 콘텐츠가 성공적으로 게시되면 알림 |
| `publish_failed` | 게시 실패 | 콘텐츠 게시에 실패하면 알림 |
| `token_expiring` | 토큰 만료 임박 | 채널 토큰 만료가 임박하면 알림 |
| `system` | 시스템 공지 | 시스템 점검, 업데이트 등의 공지 |

## 6. 연결 흐름 (Connected Flows)

### 6.1 설정 로드 및 저장 흐름

```
[알림 설정 화면 진입]
       │
       ↓
GET /notification-settings ──→ 기존 설정 로드
       │
       ↓
[channels 상태 초기화 + telegram_chat_id 초기화]
       │
       ↓
[사용자가 탭별 토글/Chat ID 변경]
       │
       ↓
[저장] 클릭
       │
       ↓
PUT /notification-settings ──→ 설정 저장
       │
       ├──(성공)──→ "알림 설정이 저장되었습니다" + 캐시 무효화
       └──(실패)──→ "알림 설정 저장에 실패했습니다"
```

### 6.2 텔레그램 연동 흐름

```
[텔레그램 탭 선택]
       │
       ↓
[전체 토글 ON]
       │
       ↓
[Chat ID 입력]
       │
       ├──→ [테스트 알림 발송] ──→ POST /notification-settings/telegram/test
       │         │
       │         ├──(성공)──→ "테스트 알림이 발송되었습니다"
       │         └──(실패)──→ 오류 메시지 표시
       │
       └──→ [저장] ──→ Chat ID + channels 설정 저장
```

### 6.3 웹 푸시 권한 흐름

```
[웹 푸시 탭 선택]
       │
       ↓
[전체 토글 ON]
       │
       ↓
[브라우저 Notification.permission 확인]
       │
       ├── 'granted' ──→ "알림 허용됨" 표시
       ├── 'denied'  ──→ "알림 미허용" 표시
       └── 'default' ──→ "알림 미허용" 표시
                            │
                            └──→ 사용자가 브라우저 설정에서 직접 허용 필요
```

### 6.4 알림 센터와의 연결

```
[알림 설정]                      [알림 센터 (/notifications)]
      │                                │
      └── channels 설정 ──────────────→│ 수신 채널 결정
      └── 유형별 ON/OFF ─────────────→│ 수신 유형 필터링
      └── telegram_chat_id ──────────→│ 텔레그램 발송 대상
```

## 8. 데이터 타입

### 8.1 알림 설정 레코드 (NotificationSettingRecord)

```typescript
interface NotificationSettingRecord {
  id: string | null;                        // 설정 ID (신규 시 null)
  organization_id: string;                  // 소속 기관 ID
  user_id: string;                          // 사용자 ID
  channels: NotificationChannelConfig;      // 채널별 설정
  push_subscription: Record<string, unknown> | null;  // 웹 푸시 구독 정보
  telegram_chat_id: string | null;          // 텔레그램 Chat ID
}
```

### 8.2 채널 설정 (NotificationChannelConfig)

```typescript
interface NotificationChannelConfig {
  web: { enabled: boolean };       // 웹 알림 활성화 여부
  email: { enabled: boolean };     // 이메일 알림 활성화 여부
  telegram: { enabled: boolean };  // 텔레그램 알림 활성화 여부
  webPush: { enabled: boolean };   // 웹 푸시 알림 활성화 여부
}
```

### 8.3 설정 저장 요청 (NotificationSettingUpdateData)

```typescript
interface NotificationSettingUpdateData {
  channels?: NotificationChannelConfig;  // 채널별 설정 (선택)
  telegram_chat_id?: string;             // 텔레그램 Chat ID (선택)
}
```

### 8.4 텔레그램 테스트 응답

```typescript
interface TelegramTestResponse {
  sent: boolean;     // 발송 성공 여부
  message: string;   // 결과 메시지
}
```

## 9. 상태 관리

| 영역 | 도구 | 설명 |
|------|------|------|
| **채널 설정** | React `useState` (`channels`) | 4개 채널별 enabled 상태 |

| **서버 상태 (테스트)** | TanStack Query `useMutation` | 텔레그램 테스트 mutation |

## 11. 사용 시나리오

### 시나리오 1: 웹 알림 유형별 설정
1. 알림 설정 화면 진입 -- 기존 설정 자동 로드
2. "웹 알림" 탭에서 전체 토글 ON 확인
3. "시스템 공지" 유형의 수신 스위치 OFF로 변경
4. [저장] 클릭 -- "알림 설정이 저장되었습니다"
5. 이후 시스템 공지 알림은 웹으로 수신되지 않음

### 시나리오 2: 텔레그램 알림 최초 연동
1. "텔레그램" 탭 선택
2. 전체 토글 ON으로 전환
3. Chat ID 영역이 나타남
4. Telegram에서 @PubliSync_Bot을 검색하여 /start 실행
5. 봇이 안내한 Chat ID를 입력 필드에 입력
6. [테스트 알림 발송] 클릭
7. Telegram에서 테스트 메시지 수신 확인
8. "테스트 알림이 발송되었습니다" 성공 메시지
9. [저장] 클릭 -- Chat ID + 채널 설정 저장

### 시나리오 3: 웹 푸시 알림 활성화
1. "웹 푸시" 탭 선택
2. 전체 토글 ON으로 전환
3. "브라우저 알림 상태: 알림 미허용" 표시 확인
4. 브라우저 설정에서 알림 허용으로 변경
5. 페이지 새로고침 후 "브라우저 알림 상태: 알림 허용됨" 확인
6. 유형별 수신 스위치 설정
7. [저장] 클릭

### 시나리오 4: 텔레그램 연동 해제
1. "텔레그램" 탭 선택
2. 전체 토글 OFF로 전환
3. Chat ID 입력 영역 및 유형별 테이블 숨김
