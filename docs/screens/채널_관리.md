# 채널 관리 (ChannelsPage)

> **경로:** `/channels`
> **소스:** `frontend/src/features/channels/pages/ChannelsPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F12 채널(계정) 연동

---

## 1. 화면 개요

SA 또는 AM이 위탁기관의 SNS 플랫폼 채널(YouTube, Instagram, Facebook, X, 네이버 블로그)을
연동·관리하는 화면이다. OAuth 인증을 통해 새 채널을 연결하고, 연동된 채널의 토큰 상태를
모니터링하며, 토큰 갱신·연동 해제를 수행한다. 또한 각 플랫폼의 API Rate Limit 사용 현황을
대시보드 형태로 확인할 수 있다.

### 핵심 원칙
- **멀티테넌트 격리:** 현재 워크스페이스(organization_id)에 속한 채널만 조회·관리 가능
- **보안:** OAuth 토큰은 Fernet 암호화 저장, 토큰 만료 전 자동 갱신 지원
- **Rate Limit 중앙 관리:** 플랫폼별 API 사용량을 실시간 모니터링하여 초과 방지

| 버튼 | 텍스트 | 동작 |
|------|--------|------|

### 5.3 OAuth 연동 흐름 (handleConnect)

```
1. POST /channels/connect/initiate
   - body: { platform, redirect_uri }
   - redirect_uri: `${window.location.origin}/channels/callback`

2. 응답: { auth_url, state }

3. window.open(auth_url, '_blank', 'width=600,height=700')
   - 팝업 창으로 플랫폼 OAuth 인증 페이지 열기

4. 모달 닫기 + info 메시지: "OAuth 인증 창이 열렸습니다. 인증을 완료하세요."

5. (별도 콜백 처리)
   - /channels/callback 페이지에서 code, state 수신
   - POST /channels/connect/callback
   - 성공 시 채널 목록 자동 갱신 (queryClient.invalidateQueries)
```

## 7. 연결 흐름 (Connected Flows)

### 7.1 채널 연동 흐름

```
[채널 관리] ──(채널 연동 클릭)──→ [연동 모달]
                                    │
                                    ├── 플랫폼 선택 (YouTube/Instagram/Facebook/X/네이버)
                                    │
                                    └──(연동 시작)──→ POST /channels/connect/initiate
                                                          │
                                                          ↓
                                                    { auth_url, state }
                                                          │
                                                          ↓
                                              window.open(auth_url) ──→ [플랫폼 OAuth 인증 팝업]
                                                                              │
                                                                        (인증 완료)
                                                                              │
                                                                              ↓
                                                                    /channels/callback
                                                                              │
                                                                              ↓
                                                              POST /channels/connect/callback
                                                              { platform, code, state, redirect_uri }
                                                                              │
                                                                              ↓
                                                                    채널 목록 자동 갱신
```

### 7.2 토큰 갱신 흐름

```
[채널 관리] ──(🔄 토큰 갱신 클릭)──→ POST /channels/:id/refresh-token
                                          │
                                    ┌─────┴──────┐
                                    ↓            ↓
                              (성공)           (실패)
                                │                │
                                ↓                ↓
                      success 메시지        error 메시지
                      "토큰이 갱신되었습니다"  "토큰 갱신에 실패했습니다"
                                │
                                ↓
                      채널 목록 자동 갱신
```

### 7.3 연동 해제 흐름

```
[채널 관리] ──(🗑️ 연동 해제 클릭)──→ [Popconfirm: "채널 연동을 해제하시겠습니까?"]

                                    ┌─────┴──────┐
                                    ↓            ↓
                              (해제 확인)     (취소)
                                    │            └──→ 아무 동작 없음
                                    ↓
                          DELETE /channels/:id
                                    │
                              ┌─────┴──────┐
                              ↓            ↓
                          (성공)        (실패)
                            │              │
                            ↓              ↓
                      success 메시지  error 메시지
                      "채널이 해제    "채널 해제에
                      되었습니다"      실패했습니다"
                            │
                            ↓
                      채널 목록 자동 갱신
```

## 9. 데이터 타입

### 9.1 채널 레코드 (ChannelRecord)

```typescript
interface ChannelRecord {
  id: string;                              // 채널 고유 ID (UUID)
  organization_id: string;                 // 소속 기관 ID
  platform: string;                        // 플랫폼 코드 (YOUTUBE | INSTAGRAM | FACEBOOK | X | NAVER_BLOG)
  platform_account_id: string;             // 플랫폼 내 계정 ID
  name: string;                            // 채널 표시명
  status: string;                          // 연동 상태 (ACTIVE | EXPIRING | EXPIRED | DISCONNECTED)
  token_expires_at: string | null;         // 토큰 만료 일시 (ISO 8601)
  metadata: Record<string, unknown> | null; // 추가 메타데이터 (JSONB)
  created_at: string;                      // 생성 일시 (ISO 8601)
}
```

### 9.2 채널 이력 레코드 (ChannelHistoryRecord)

```typescript
interface ChannelHistoryRecord {
  id: string;                                // 이력 고유 ID (UUID)
  channel_id: string;                        // 대상 채널 ID
  event_type: string;                        // 이벤트 유형 (CONNECTED | DISCONNECTED | TOKEN_REFRESHED | TOKEN_EXPIRED | STATUS_CHANGED)
  details: Record<string, unknown> | null;   // 이벤트 상세 정보 (JSONB)
  actor_id: string | null;                   // 처리자 사용자 ID
  created_at: string;                        // 발생 일시 (ISO 8601)
}
```

### 9.3 연동 시작 응답 (ConnectInitiateResponse)

```typescript
interface ConnectInitiateResponse {
  auth_url: string;   // 플랫폼 OAuth 인증 URL
  state: string;      // CSRF 방지용 state 토큰
}
```

### 9.4 API 상태 레코드 (ApiStatusRecord)

```typescript
interface ApiStatusRecord {
  platform: string;          // 플랫폼 코드
  requests_used: number;     // 현재 사용된 요청 수
  requests_limit: number;    // 최대 요청 한도
  window: string;            // Rate Limit 윈도우 (예: "일간", "시간")
  percentage_used: number;   // 사용 백분율 (0~100)
}
```

## 10. 상태 관리

| 영역 | 도구 | 설명 |
|------|------|------|
| **채널 목록** | TanStack Query `useQuery` | `['channels', { page }]` 키로 서버 상태 캐싱 |
| **API 상태** | TanStack Query `useQuery` | `['channels', 'api-status']` 키, 60초 자동 갱신 |
| **채널 이력** | TanStack Query `useQuery` | `['channels', channelId, 'history']` 키, `enabled: !!channelId` |
| **연동/해제/갱신** | TanStack Query `useMutation` | 성공 시 `queryClient.invalidateQueries(['channels'])` |
| **페이지 번호** | React `useState<number>` | `page` -- 테이블 페이지네이션 |
| **연동 모달** | React `useState<boolean>` | `connectOpen` -- 모달 열림/닫힘 |
| **플랫폼 선택** | React `useState<string>` | `selectedPlatform` -- 연동 시 플랫폼 선택 (기본: 'YOUTUBE') |
| **이력 채널 ID** | React `useState<string \| null>` | `historyChannelId` -- Drawer 열림 제어 |

## 12. 사용 시나리오

### 시나리오 1: 새 채널 연동 (OAuth)
1. 사이드바 > 채널 관리 클릭
2. 우측 상단 [채널 연동] 클릭 -- 모달 열림
3. 플랫폼 선택 (예: Instagram)
4. [연동 시작] 클릭
5. OAuth 인증 팝업 창이 열림 (600x700)
6. 플랫폼에서 계정 인증 및 권한 승인
7. 콜백 처리 후 채널 목록에 새 채널 자동 추가
8. 상태: ACTIVE, 토큰 만료일 표시

### 시나리오 2: 만료 임박 토큰 갱신
1. 채널 목록에서 상태가 "만료 임박" (주황 태그)인 채널 확인
2. 해당 행의 🔄 (토큰 갱신) 버튼 클릭
3. 성공 메시지: "토큰이 갱신되었습니다"
4. 목록 자동 갱신 -- 상태가 "연동됨"으로 변경, 만료일 갱신

### 시나리오 3: 채널 연동 해제
1. 더 이상 사용하지 않는 채널의 🗑️ (연동 해제) 버튼 클릭
2. Popconfirm: "채널 연동을 해제하시겠습니까?"
3. [해제] 클릭으로 확인
4. 성공 메시지: "채널이 해제되었습니다"
5. 목록에서 해당 채널 제거 (또는 상태 DISCONNECTED로 변경)

### 시나리오 4: API Rate Limit 모니터링
1. [API 상태] 탭 클릭
2. 플랫폼별 API 사용 현황 확인
3. YouTube 사용률 90% 초과 -- Progress 바 빨간색 표시
4. 해당 플랫폼 게시 작업을 나중으로 미루거나, Rate Limit 윈도우 경과 대기

### 시나리오 5: 채널 이력 확인
1. 특정 채널의 📋 (이력) 버튼 클릭
2. 우측 Drawer가 열리며 타임라인 표시
3. "토큰 갱신" -- 2026-03-07 14:30
4. "채널 연동" -- 2026-02-15 10:00
5. 이벤트 발생 시점과 유형을 시간순으로 확인
