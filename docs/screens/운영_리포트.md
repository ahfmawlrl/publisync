# 운영 리포트 (ReportsPage)

> **경로:** `/reports`
> **소스:** `frontend/src/features/reports/pages/ReportsPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 3
> **관련 기능:** F19 운영 리포트

---

## 1. 화면 개요

AI가 기관의 SNS 운영 성과 데이터를 분석하여 주간/월간/분기별 운영 리포트를 자동 생성하는 화면이다.
리포트 목록 테이블, 유형/상태 필터, AI 리포트 생성 모달, 리포트 상세 Drawer를 제공한다.
생성된 리포트는 초안(DRAFT) → 확정(FINALIZED) 워크플로우를 거치며,
확정된 리포트는 PDF로 다운로드할 수 있다. AI 생성은 비동기(Celery) 처리되며
Job 폴링으로 완료를 확인한다.

### 핵심 원칙
- **Human-in-the-Loop:** AI가 생성한 리포트는 "초안" 상태이며, 담당자가 검토 후 "확정"해야 한다
- **Fallback 필수:** AI 리포트 생성 실패 시에도 성과 분석 데이터로 수동 작성 가능
- **워크플로우 내장 AI:** 리포트 생성 요청이 목록 화면 안에서 모달로 호출됨

## 6. AI 리포트 생성 모달

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Modal` |
| **제목** | "AI 리포트 생성" |
| **확인 버튼** | "AI 생성 시작" |
| **로딩** | `generateMutation.isPending` |

### 6.1 리포트 유형 선택

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Radio.Group` + `Radio.Button` |
| **라벨** | "리포트 유형" (`Text` strong) |
| **옵션** | 주간 (`WEEKLY`), 월간 (`MONTHLY`), 분기 (`QUARTERLY`) |
| **기본값** | `MONTHLY` |

### 6.2 분석 기간 선택

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `DatePicker.RangePicker` (className="w-full") |
| **라벨** | "분석 기간" (`Text` strong) |
| **유효성 검사** | 미선택 시 "기간을 선택해주세요" warning toast |

### 6.3 안내 텍스트

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Paragraph` (type="secondary") |
| **내용** | "선택한 기간의 성과 데이터를 AI가 분석하여 리포트를 자동 생성합니다. 생성에 1~3분 정도 소요됩니다." |

### 6.4 생성 처리 흐름

```
[AI 생성 시작] 클릭
       │
       ├──→ 기간 미선택 → warning toast "기간을 선택해주세요"
       │
       └──→ POST /reports/generate
                  │
                  ├──(성공)──→ 모달 닫기
                  │            info toast "리포트 생성이 시작되었습니다"
                  │            job_id가 있으면 → Job 폴링 시작
                  │
                  └──(실패)──→ error toast "리포트 생성 요청에 실패했습니다"
```

### 6.5 요청 본문 (GenerateReportRequest)

```typescript
{
  period: string;          // "WEEKLY" | "MONTHLY" | "QUARTERLY"
  period_start: string;    // "YYYY-MM-DD"
  period_end: string;      // "YYYY-MM-DD"
}
```

## 7. Job 폴링 메커니즘

| 항목 | 내용 |
|------|------|
| **트리거** | 리포트 생성 응답에 `job_id`가 포함될 때 |
| **간격** | 3초마다 폴링 |
| **최대 시도** | 60회 (= 최대 3분) |
| **API** | `GET /ai/jobs/{jobId}` |

### 7.1 Job 상태별 처리

| Job 상태 | 처리 |
|----------|------|
| `COMPLETED` | success toast "리포트가 생성되었습니다" + 폴링 종료 |
| `FAILED` | error toast "리포트 생성에 실패했습니다" + 폴링 종료 |
| 기타 (진행중) | 폴링 계속 |
| 최대 시도 초과 | warning toast "리포트 생성이 지연되고 있습니다. 목록에서 확인해주세요" + 폴링 종료 |
| API 오류 | 무시하고 폴링 계속 |

### 7.2 폴링 상태 표시

| 항목 | 내용 |
|------|------|
| **UI** | 헤더 영역에 `Spin` (size="small") 표시 |
| **상태** | `polling` state에 jobId 저장, null이면 숨김 |

| 섹션 키 | 제목 |
|---------|------|
| `summary` | 요약 |
| `platformPerformance` | 플랫폼별 성과 |
| `topContents` | 주요 콘텐츠 |
| `commentAnalysis` | 댓글/여론 분석 |
| `aiSuggestions` | 개선 제안 |

### 8.4 섹션 타입별 렌더링

| type | 렌더링 방식 |
|------|------------|
| `AI_TEXT` | `Card` > `Paragraph` — 텍스트 내용 표시 |
| `CHART_DATA` | `Card` > `Paragraph` + `<pre>` — 텍스트 + JSON 데이터 표시 |
| `TABLE_DATA` | `Card` > `Paragraph` + `Table` — 텍스트 + 동적 테이블 (data 배열의 키로 컬럼 자동 생성) |

### 8.5 빈 상태

| 상태 | 표시 |
|------|------|
| **로딩** | `Spin` (size="large", h-64 중앙) |

## 11. 연결 흐름 (Connected Flows)

### 11.1 리포트 생성 → 확정 → 다운로드 흐름

```
[새 리포트 생성] ──(모달)──→ [유형/기간 선택] ──→ [AI 생성 시작]
                                                       │
                                                       ↓
                                            POST /reports/generate
                                                       │
                                                       ↓
                                              Job 폴링 (3초 간격)
                                                       │
                                           ┌───────────┼───────────┐
                                           ↓           ↓           ↓
                                      [COMPLETED]  [FAILED]   [TIMEOUT]
                                           │           │           │
                                           ↓           ↓           ↓
                                      목록에 DRAFT    에러 토스트  경고 토스트
                                      상태로 표시                  목록 확인 안내
                                           │
                                           ↓
                                    [상세 Drawer 열기]
                                           │
                                    [검토 후 확정 클릭]
                                           │
                                           ↓
                                    POST /reports/{id}/finalize
                                           │
                                           ↓
                                    status: FINALIZED
                                           │
                                           ↓
                                    [PDF 다운로드 가능]
```

### 11.2 성과 분석 → 리포트 연결

```
[성과 분석 /analytics] ──→ 성과 데이터 확인
       │
       └──→ [운영 리포트 /reports] ← 성과 데이터 기반 AI 리포트 생성
```

## 12. API 엔드포인트 연결

| 동작 | HTTP | 엔드포인트 | Hook | Phase |
|------|------|-----------|------|-------|
| 리포트 목록 조회 | GET | `/reports` | `useReportsList()` | 3 |
| 리포트 상세 조회 | GET | `/reports/{id}` | `useReport()` | 3 |
| AI 리포트 생성 | POST | `/reports/generate` | `useGenerateReport()` | 3 |
| 리포트 수정 | PUT | `/reports/{id}` | `useUpdateReport()` | 3 |
| 리포트 확정 | POST | `/reports/{id}/finalize` | `useFinalizeReport()` | 3 |
| PDF 다운로드 | GET | `/reports/{id}/download` | (window.open) | 3 |
| Job 상태 조회 | GET | `/ai/jobs/{jobId}` | (apiClient.get) | 3 |

## 15. 오류 처리 및 Fallback

| 상황 | 처리 |
|------|------|
| **목록 로딩 중** | 테이블 `loading` 속성 표시 |
| **목록 비어있음** | `Empty` (description="리포트가 없습니다") |
| **상세 로딩 중** | Drawer 내 `Spin` (size="large", h-64 중앙) |
| **상세 데이터 없음** | `Empty` (description="리포트를 찾을 수 없습니다") |
| **리포트 내용 없음** | `Empty` (description="리포트 내용이 없습니다") |
| **기간 미선택 시 생성** | warning toast "기간을 선택해주세요" |
| **리포트 생성 실패** | error toast "리포트 생성 요청에 실패했습니다" |
| **리포트 확정 실패** | error toast "리포트 확정에 실패했습니다" |
| **Job 폴링 실패** | 무시하고 폴링 계속 |
| **Job 폴링 타임아웃** | warning toast "리포트 생성이 지연되고 있습니다. 목록에서 확인해주세요" |
| **Job FAILED** | error toast "리포트 생성에 실패했습니다" |

## 16. 사용 시나리오

### 시나리오 1: 월간 리포트 생성

1. 사이드바 > 운영 리포트 클릭
2. [새 리포트 생성] 버튼 클릭
3. 리포트 유형 "월간" 선택
4. 분석 기간 "2026-03-01 ~ 2026-03-31" 선택
5. [AI 생성 시작] 클릭
6. "리포트 생성이 시작되었습니다" 토스트 확인
7. 헤더 Spin 아이콘으로 폴링 진행 확인
8. "리포트가 생성되었습니다" 성공 토스트 확인
9. 목록에서 새 리포트 확인 (상태: 초안)

### 시나리오 2: 리포트 검토 및 확정

1. 목록에서 "초안" 상태의 리포트 제목 클릭
2. 상세 Drawer 열림
3. 메타 정보 (유형, 기간, AI 모델) 확인
4. 요약, 플랫폼별 성과, 주요 콘텐츠, 댓글/여론 분석, 개선 제안 순서로 검토
5. 내용이 적절하면 [확정] 버튼 클릭
6. "리포트가 확정되었습니다" 토스트 확인
7. 상태가 "확정"으로 변경됨

### 시나리오 3: 확정 리포트 PDF 다운로드

1. 목록에서 "확정" 상태의 리포트 확인
2. 우측 다운로드 아이콘 클릭
3. 새 탭에서 PDF 다운로드 시작
4. 또는 상세 Drawer에서 [PDF 다운로드] 버튼 클릭

### 시나리오 4: 필터를 활용한 리포트 검색

1. 유형 필터에서 "월간" 선택
2. 상태 필터에서 "확정" 선택
3. 확정된 월간 리포트만 목록에 표시
4. 필요한 리포트 상세 확인

### 시나리오 5: AI 리포트 생성 실패 대응

1. [AI 생성 시작] → 생성 요청 실패
2. "리포트 생성 요청에 실패했습니다" 에러 토스트
3. 네트워크/서버 상태 확인 후 재시도
4. 또는 성과 분석 화면 데이터를 참고하여 수동 작성
