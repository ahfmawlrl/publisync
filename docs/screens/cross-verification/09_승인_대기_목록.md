# 교차 검증 결과: #9 승인 대기 목록 (ApprovalsListPage)

> **검증 일자:** 2026-03-11
> **검증 대상:** 설계서 `승인_대기_목록.md` ↔ FE `ApprovalsListPage.tsx` ↔ BE `approvals.py` + `content_service.py`
> **브라우저:** localhost:5173/approvals (1440×900 데스크톱)

## 1. 발견 버그 및 수정 사항

### 🔴 심각 (Blocker)

| # | 구분 | 문제 | 수정 |
|---|------|------|------|
| B1 | BE 전체 | **flush() 후 refresh() 누락 → MissingGreenlet 500 에러** | 10개 리포지토리, 21개 인스턴스에 `await self._db.refresh()` 추가 |
| B2 | BE | `request_review`가 `ApprovalRequest` 레코드를 생성하지 않음 → 승인 목록 항상 비어있음 | `content_service.py`에서 `ApprovalRequest` + `ApprovalHistory(SUBMIT)` 생성 로직 추가 |
| B3 | BE | `status` 쿼리 필터가 소문자(draft) 시 PostgreSQL enum 불일치로 500 에러 | `content_repository.py`, `approval_repository.py`에 `status.upper()` 정규화 추가 |

### 🟡 경미 (Minor)

| # | 구분 | 문제 | 영향 |
|---|------|------|------|
| M1 | FE 타입 | `ApprovalRequestRecord`에 `histories` 필드 미포함 | BE는 `histories[]` 반환하지만 FE에서 사용하지 않고 `created_at`으로 이력 구성 |
| M2 | FE | `useApproveRequest`에서 `comment`를 보내지만 실제 UI에서는 `rejectComment` 상태 변수를 공유 | 승인 시 의견이 비어있어 의도된 동작이지만, 승인 의견 입력 기능 부재 |

## 2. FE↔BE 쿼리 검증

### 2.1 Hooks → API 매핑

| FE Hook | HTTP 메서드 | BE Endpoint | BE Service Method | 검증 |
|---------|-----------|-------------|-------------------|------|
| `useApprovals({page, status, requested_by})` | GET | `/approvals?page=&status=&requested_by=` | `list_approvals()` | ✅ 정상 (status 대문자 정규화 후) |
| `useApproval(id)` | GET | `/approvals/:id` | `get_approval()` | ✅ 정상 |
| `useApproveRequest({id, comment})` | POST | `/approvals/:id/approve` | `approve()` | ✅ 정상 (CD/AM 역할 필요) |
| `useRejectRequest({id, comment})` | POST | `/approvals/:id/reject` | `reject()` | ✅ 정상 (CD/AM 역할 필요) |
| `useWorkflows()` | GET | `/approvals/workflows` | `get_workflows()` | ✅ 정상 (설정 화면에서 사용) |
| `useUpdateWorkflow(data)` | PUT | `/approvals/workflows` | `update_workflow()` | ✅ 정상 (AM 역할 필요) |
| 인라인 `useQuery('content', contentId)` | GET | `/contents/:id` | `get_content()` | ✅ 정상 (검수 상세에서 콘텐츠 본문 조회) |

### 2.3 BE Service → Repository 호출 체인

| Service Method | Repository Call | SQL 패턴 | 검증 |
|---|---|---|---|
| `list_approvals()` | `repo.list_requests(org_id, offset, limit, status, ...)` | SELECT + COUNT with RLS | ✅ |
| `get_approval()` | `repo.get_by_id(request_id)` | SELECT with selectinload(histories, requester, content) | ✅ |
| `approve()` | `repo.update(req, {status})` + `repo.add_history()` + `content_repo.update(content, {status: APPROVED})` | UPDATE + INSERT + UPDATE | ✅ |
| `reject()` | `repo.update(req, {status})` + `repo.add_history()` + `content_repo.update(content, {status: REJECTED})` | UPDATE + INSERT + UPDATE | ✅ |
| `count_pending()` | `repo.count_pending(org_id)` | SELECT COUNT WHERE status IN (PENDING_REVIEW, IN_REVIEW) | ✅ |
