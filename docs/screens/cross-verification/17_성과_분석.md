## 6. 쿼리 로직 검증 (5개)

### Q1: analytics_repository.get_performance_by_platform(org_id, start_date, end_date)
- `JOIN Channel ON PublishResult.channel_id == Channel.id`: ✅
- `Channel.organization_id == org_id` 필터 (멀티테넌트): ✅
- `PublishResult.published_at BETWEEN start_date AND end_date`: ✅
- `GROUP BY Channel.platform` + SUM(views, likes, shares, comments): ✅
- `coalesce(SUM(...), 0)` 기본값 처리: ✅

### Q2: analytics_repository.get_engagement_heatmap(org_id, start_date, end_date)
- `JOIN Channel` + org_id 필터: ✅
- `extract('hour', published_at)` → 0~23: ✅
- `extract('isodow', published_at) - 1` → 0~6 (월~일): ✅
- `GROUP BY hour, day_of_week` + AVG(engagement_rate): ✅
- 날짜 범위 필터: ✅

### Q3: analytics_repository.get_trend_data(org_id, start_date, end_date, granularity)
- `date_trunc(granularity, published_at)` 집계 (day/week/month): ✅
- org_id + 날짜 범위 필터: ✅
- `GROUP BY date` + SUM(views) as reach, SUM(likes+shares+comments) as engagement: ✅
- `ORDER BY date ASC`: ✅
- `coalesce` 기본값: ✅

### Q4: analytics_repository.get_top_contents_data(org_id, start_date, end_date, limit=5)
- 3-way JOIN: `Content → PublishResult → Channel`: ✅
- org_id 필터 (`Content.organization_id`): ✅
- `Content.deleted_at.is_(None)` 소프트 삭제 필터: ✅
- `ORDER BY SUM(views) DESC` + `LIMIT 5`: ✅
- rank 번호 Python enumerate 할당: ✅

### Q5: analytics_service.export_performance(org_id, period)
- `get_performance_by_platform()` 재사용: ✅
- CSV 직렬화 (StringIO + csv.writer): ✅
- AM/SA 전용 RBAC: ✅
