## 발견 버그

### BUG-1: /ai/usage 500 에러 (Critical)
- **증상:** `GET /ai/usage` → 500 Internal Server Error
- **원인:** Router에서 `{**stats, "organization_id": ...}` 사용 — `stats`가 `AiUsageResponse` Pydantic 모델이므로 `**` 언패킹 불가
- **수정:** `{**stats, ...}` → `stats.model_dump()` 로 변경
- **검증:** 수정 후 200 정상 반환

## 쿼리 로직 검증 (2개)

### Q1: get_usage_stats
```python
# 전체 합계
select(count(), sum(total_tokens), sum(estimated_cost))
  .where(organization_id == org_id)

# 작업유형별 분석
select(task_type, count(), sum(total_tokens), sum(estimated_cost))
  .where(organization_id == org_id)
  .group_by(task_type)
  .order_by(count().desc())
```
- org_id 필터: ✅
- coalesce 0 기본값: ✅
- Decimal 변환: ✅

### Q2: list_jobs
```python
select(AiJob).where(organization_id == org_id)
  + optional: job_type, status
  .order_by(created_at.desc())
  .offset/limit
```
- org_id 필터: ✅
- 작업유형/상태 선택 필터: ✅
- count 쿼리 동기화: ✅
