# 교차 검증 결과: #10 채널 관리 (ChannelsPage)

> **검증 일자:** 2026-03-11
> **검증 대상:** 설계서 `채널_관리.md` ↔ FE `ChannelsPage.tsx` ↔ BE `channels.py` + `channel_service.py`
> **브라우저:** localhost:5173/channels (1440×900 데스크톱)

---

## 1. 발견 버그 및 수정 사항

### 🟡 중간 (Major)

| # | 구분 | 문제 | 수정 |
|---|------|------|------|
| B1 | BE RBAC | `connect_initiate`와 `disconnect_channel`에 `UserRole.SYSTEM_ADMIN` 누락 → **SA가 채널 연동/해제 시 403 Forbidden** | `require_roles(UserRole.SYSTEM_ADMIN, UserRole.AGENCY_MANAGER)` 추가 |
| B2 | BE Route | `/api-status` 라우트가 `/{channel_id}` 패턴 뒤에 정의됨 → 향후 `GET /{channel_id}` 추가 시 충돌 위험 | `/api-status`를 `/{channel_id}` 패턴 앞으로 이동 |

### 🟢 경미 (Minor)

| # | 구분 | 문제 | 영향 |
|---|------|------|------|
| M1 | FE 타입 | `ChannelHistoryRecord`에 `actor_name` 필드 미포함 | BE는 `actor_name: str | None` 반환하지만 FE 타임라인에서 미사용 → 타입 추가 |

## 2. FE↔BE 쿼리 검증

### 2.1 Hooks → API 매핑

| FE Hook | HTTP 메서드 | BE Endpoint | BE Service Method | 검증 |
|---------|-----------|-------------|-------------------|------|
| `useChannels(page)` | GET | `/channels?page=&limit=20` | `list_channels()` | ✅ 200, 빈 목록 + 페이지네이션 메타 |
| `useApiStatus()` | GET | `/channels/api-status` | `get_api_status()` | ✅ 200, 5개 플랫폼 반환 (60초 자동 갱신) |
| `useChannelHistory(channelId)` | GET | `/channels/:id/history` | `get_channel_history()` | ✅ 404 (유효하지 않은 ID), enabled: !!channelId |
| `useConnectChannel().initiate` | POST | `/channels/connect/initiate` | `initiate_connect()` | ✅ 200, auth_url + state 반환 (RBAC 수정 후) |
| `useConnectChannel().callback` | POST | `/channels/connect/callback` | `handle_callback()` | ✅ 201, 채널 생성/갱신 (역할 제한 없음) |
| `useDisconnectChannel()` | DELETE | `/channels/:id` | `disconnect()` | ✅ 204/404 (RBAC 수정 후) |
| `useRefreshChannelToken()` | POST | `/channels/:id/refresh-token` | `refresh_channel_token()` | ✅ 404 (유효하지 않은 ID) |
