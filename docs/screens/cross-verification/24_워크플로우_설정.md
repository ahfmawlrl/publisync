## 발견 버그

### BUG-1: 라우트 순서 충돌 (Critical)
- **증상:** `GET /approvals/workflows` → 400 (UUID 파싱 에러)
- **원인:** `GET /{approval_id}` (line 100)가 `GET /workflows` (line 137)보다 먼저 등록 → FastAPI가 "workflows"를 UUID로 파싱 시도
- **수정:** `/workflows` 라우트를 `/{approval_id}` 앞으로 이동
- **검증:** 수정 후 AM/AO/CD 모두 GET 200 정상
