## 5. 쿼리 로직 검증

### 5.1 list_users (Repository)
- `deleted_at IS NULL` 소프트 삭제 필터 ✅
- `org_id` 선택적 필터 (UserOrganization JOIN) ✅
- `role` 선택적 필터 ✅
- **`search` 이름/이메일 ILIKE 검색** ✅ (이번 검증에서 추가)
- **`status` 상태 필터** ✅ (이번 검증에서 추가)
- count 쿼리와 data 쿼리 필터 일치 ✅
- `created_at DESC` 정렬 ✅
- offset/limit 페이지네이션 ✅

### 5.2 get_by_id (Repository)
- 단순 PK 조회, deleted_at 체크는 서비스 레이어에서 수행 ✅

### 5.3 create_user (Service)
- 이메일 중복 체크 (`get_by_email`) ✅
- INACTIVE 상태로 생성 ✅
- 임시 비밀번호 생성 + bcrypt 해시 ✅
- UserOrganization 매핑 생성 ✅

### 5.4 update_user (Service/Repository)
- `model_dump(exclude_unset=True)` — 전달된 필드만 업데이트 ✅
- `setattr` 루프로 동적 필드 업데이트 ✅

### 5.5 soft_delete_user (Repository)
- `deleted_at = datetime.now(UTC)` 소프트 삭제 ✅
- 물리 삭제 아님 (데이터 보존) ✅
