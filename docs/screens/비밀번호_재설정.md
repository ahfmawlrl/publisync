# 비밀번호 재설정 (ResetPasswordPage)

> **경로:** `/reset-password` (요청 폼) / `/reset-password?token=xxx` (재설정 폼)
> **소스:** `frontend/src/features/auth/pages/ResetPasswordPage.tsx`
> **접근 권한:** 공개 (비인증 사용자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F08 권한관리 (이메일 기반 비밀번호 재설정)

---

## 1. 화면 개요

비밀번호를 분실한 사용자가 등록된 이메일을 통해 비밀번호를 재설정하는 화면이다.
URL의 `?token=` 쿼리 파라미터 유무에 따라 두 가지 폼을 조건부 렌더링한다.

- **토큰 없음 (`/reset-password`):** 이메일 입력 → 재설정 링크 전송 요청 (Step 1)
- **토큰 있음 (`/reset-password?token=xxx`):** 새 비밀번호 입력 → 비밀번호 변경 (Step 2)

### 핵심 원칙
- **2단계 흐름:** 이메일 인증 기반으로 보안 유지
- **토큰 만료 처리:** 유효하지 않은 토큰에 대해 명확한 오류 메시지 제공
- **완료 후 안내:** 각 단계 완료 시 Result 컴포넌트로 다음 행동 안내

## 6. 연결 흐름 (Connected Flows)

### 6.1 전체 비밀번호 재설정 흐름

```
[로그인 화면] ──("비밀번호 찾기" 클릭)──→ /reset-password
       │
       ↓
[Step 1: RequestForm] ──(이메일 입력)──→ POST /auth/password/reset-request
       │
       ├──(성공)──→ [이메일 전송 확인 Result]
       │                 │
       │                 └──("로그인으로 돌아가기")──→ /login
       │
       └──(실패)──→ "요청 처리에 실패했습니다. 다시 시도하세요." 오류 표시
```

### 6.2 이메일 → 비밀번호 변경 흐름

```
[사용자 이메일 수신]
       │
       └──(재설정 링크 클릭)──→ /reset-password?token=xxx
                                     │
                                     ↓
                              [Step 2: ResetForm]
                                     │
                              ┌──────┴──────┐
                              ↓             ↓
                         (비밀번호 입력)  (토큰 만료/무효)
                              │             │
                              ↓             ↓
                     POST /auth/password/reset  "토큰이 만료되었거나
                              │              유효하지 않습니다" 오류
                              ↓
                     [변경 완료 Result]
                              │
                              └──("로그인")──→ /login
```

## 11. 사용 시나리오

### 시나리오 1: 정상 비밀번호 재설정

1. 로그인 화면에서 "비밀번호 찾기" 클릭 → `/reset-password` 이동
2. 가입된 이메일 입력
3. [재설정 링크 전송] 클릭
4. "이메일을 확인하세요" Result 화면 표시
5. 이메일 수신 → 재설정 링크 클릭 → `/reset-password?token=xxx` 이동
6. 새 비밀번호 입력 (8자 이상)
7. 비밀번호 확인 입력 (동일한 값)
8. [비밀번호 변경] 클릭
9. "비밀번호가 변경되었습니다" Result 화면 표시
10. "로그인" 링크 클릭 → `/login`으로 이동

### 시나리오 2: 만료된 토큰으로 접근

1. 재설정 링크를 늦게 클릭 (토큰 만료)
2. `/reset-password?token=expired_xxx` 이동
3. 새 비밀번호 입력 후 [비밀번호 변경] 클릭
4. "토큰이 만료되었거나 유효하지 않습니다." 오류 메시지 표시
5. 사용자가 `/reset-password`에서 다시 재설정 요청 필요

### 시나리오 3: 비밀번호 불일치 입력

1. 새 비밀번호: "password123" 입력
2. 비밀번호 확인: "password456" 입력

## 13. 보안 관련 사항

| 항목 | 설명 |
|------|------|
| **토큰 전송** | 이메일 본문의 링크에 쿼리 파라미터로 포함 |
| **토큰 만료** | 서버 측에서 유효 시간 관리 (일반적으로 1~24시간) |
| **비밀번호 최소 길이** | 8자 이상 (클라이언트 + 서버 양쪽 검증) |
| **이메일 미존재 처리** | 보안상 존재하지 않는 이메일이라도 동일한 성공 응답 표시 (이메일 열거 공격 방지) |
| **confirm_password** | 서버에 전송하지 않음, 클라이언트 측 일치 검증만 수행 |
