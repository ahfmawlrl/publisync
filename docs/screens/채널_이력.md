# 채널 이력 (ChannelHistoryPage)

> **경로:** `/channels/history`
> **소스:** `frontend/src/features/channels/pages/ChannelHistoryPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F12 채널(계정) 연동

---

## 1. 화면 개요

SA 또는 AM이 연동된 SNS 채널의 이벤트 이력을 조회하는 전용 화면이다.
채널을 선택한 뒤 해당 채널에서 발생한 이벤트(연동, 해제, 토큰 갱신, 토큰 만료, 상태 변경)를
테이블 형태로 시간순 확인한다. 채널 관리 화면(`/channels`)의 Drawer 이력 보기와 달리,
전체 채널에 대해 상세한 필드(처리자, 상세 정보 JSON)를 포함한 목록을 제공하며
페이지네이션을 통해 대량의 이력 데이터를 효율적으로 탐색할 수 있다.

### 핵심 원칙
- **멀티테넌트 격리:** 현재 워크스페이스에 속한 채널의 이력만 조회 가능
- **감사 추적:** 이벤트 발생 일시, 유형, 처리자(actor_id), 상세 정보를 기록하여 변경 이력 추적

## 4. 채널 선택 드롭다운

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Select` |
| **너비** | 320px |
| **위치** | 제목 아래, 테이블 위 (className="mb-4") |
| **플레이스홀더** | "채널을 선택하세요" |
| **allowClear** | `true` -- 선택 해제 가능 |
| **loading** | `channelsLoading` -- 채널 목록 로딩 중 스피너 표시 |
| **데이터 소스** | `useChannels()` 훅의 채널 목록 |

### 4.1 옵션 형식

각 채널은 다음 형식으로 표시된다:

```
{플랫폼 레이블} — {채널명}
```

예시:
- `YouTube — 서울TV`
- `Instagram — @seoul_official`
- `Facebook — 서울시청`
- `X (Twitter) — @seoul_x`
- `네이버 블로그 — 서울시 공식블로그`

### 4.2 선택 동작

| 동작 | 설명 |
|------|------|
| **채널 선택** | `setSelectedChannelId(val)` + `setPage(1)` -- 페이지 1로 초기화 |
| **선택 해제** | `setSelectedChannelId(null)` + `setPage(1)` |

## 6. 연결 흐름 (Connected Flows)

### 6.1 이력 조회 흐름

```
[채널 이력 페이지 진입]
       │
       ↓
GET /channels ──→ 채널 목록 로딩 ──→ Select 옵션 구성
       │
       ↓
[채널 선택 (Select)]
       │
       ├── selectedChannelId 설정
       ├── page = 1 초기화
       │
       ↓
GET /channels/:id/history?page=1&limit=50
       │
       ↓
[이력 테이블 렌더링]
       │
       ├── 이벤트 유형 태그 (색상 구분)
       ├── 발생 일시 (한국어 로케일)
       ├── 채널명 (플랫폼 + 이름)
       ├── 처리자 (actor_id 또는 -)
       └── 상세 정보 (JSON)
```

### 6.2 페이지 이동 흐름

```
[페이지네이션 클릭]
       │
       ↓
setPage(newPage) ──→ useChannelHistory 재요청
                      (queryKey 변경으로 자동 refetch)
       │
       ↓
GET /channels/:id/history?page={newPage}&limit=50
       │
       ↓
[테이블 갱신]
```

### 6.3 채널 관리 페이지와의 관계

```
[채널 관리 (/channels)]                   [채널 이력 (/channels/history)]
       │                                           │
       ├── 간략 이력: Drawer (Timeline)             ├── 상세 이력: 전체 페이지 (Table)
       ├── 단일 채널 대상                           ├── 채널 선택 드롭다운
       ├── 이벤트명 + 일시만 표시                   ├── 일시 + 유형 + 채널명 + 처리자 + 상세
       └── 페이지네이션 없음                        └── 50건 단위 페이지네이션
```

## 8. 데이터 타입

### 8.1 채널 레코드 (ChannelRecord)

```typescript
interface ChannelRecord {
  id: string;                              // 채널 고유 ID (UUID)
  organization_id: string;                 // 소속 기관 ID
  platform: string;                        // 플랫폼 코드 (YOUTUBE | INSTAGRAM | FACEBOOK | X | NAVER_BLOG)
  platform_account_id: string;             // 플랫폼 내 계정 ID
  name: string;                            // 채널 표시명
  status: string;                          // 연동 상태 (ACTIVE | EXPIRING | EXPIRED | DISCONNECTED)
  token_expires_at: string | null;         // 토큰 만료 일시 (ISO 8601)
  metadata: Record<string, unknown> | null; // 추가 메타데이터 (JSONB)
  created_at: string;                      // 생성 일시 (ISO 8601)
}
```

### 8.2 채널 이력 레코드 (ChannelHistoryRecord)

```typescript
interface ChannelHistoryRecord {
  id: string;                                // 이력 고유 ID (UUID)
  channel_id: string;                        // 대상 채널 ID
  event_type: string;                        // 이벤트 유형 (CONNECTED | DISCONNECTED | TOKEN_REFRESHED | TOKEN_EXPIRED | STATUS_CHANGED)
  details: Record<string, unknown> | null;   // 이벤트 상세 정보 (JSONB)
  actor_id: string | null;                   // 처리자 사용자 ID (시스템 이벤트는 null)
  actor_name: string | null;                 // 처리자 이름 (BE 응답에서 추가)
  created_at: string;                        // 발생 일시 (ISO 8601)
}
```

### 8.3 이벤트 유형별 details 구조 (참고)

| 이벤트 | details 예시 |
|--------|-------------|
| `CONNECTED` | `{"scope": "read,write", "platform_account_id": "UC..."}` |
| `DISCONNECTED` | `{"reason": "manual", "disconnected_by": "user_id"}` |
| `TOKEN_REFRESHED` | `{"old_expiry": "2026-03-01T00:00:00Z", "new_expiry": "2026-06-01T00:00:00Z"}` |
| `TOKEN_EXPIRED` | `{"expired_at": "2026-03-07T00:00:00Z"}` |
| `STATUS_CHANGED` | `{"from": "ACTIVE", "to": "EXPIRING"}` |

## 9. 상태 관리

| 영역 | 도구 | 설명 |
|------|------|------|
| **채널 목록** | TanStack Query `useQuery` | `['channels', { page: undefined }]` 키로 전체 채널 조회 |
| **채널 이력** | TanStack Query `useQuery` | `['channels', channelId, 'history', { page }]` 키, `enabled: !!channelId` |
| **선택 채널 ID** | React `useState<string \| null>` | `selectedChannelId` -- 드롭다운 선택 값 |
| **페이지 번호** | React `useState<number>` | `page` -- 테이블 페이지네이션 (기본값: 1) |
| **선택 채널 객체** | 파생 상태 | `channelsData.data.find(ch => ch.id === selectedChannelId)` |

## 10. 오류 처리

| 상황 | 처리 방식 |
|------|----------|
| **채널 목록 로딩 중** | Select 컴포넌트에 loading 스피너 표시 |
| **이력 로딩 중** | Table loading 프로퍼티로 스피너 표시 |
| **채널 미선택** | 테이블 emptyText: "채널을 선택하세요" |
| **이력 없음** | 테이블 emptyText: "이력이 없습니다" |
| **API 오류** | TanStack Query 기본 오류 처리 (error 상태) |
| **details null** | 상세 컬럼에 `-` 표시 |
| **actor_id null** | 처리자 컬럼에 `-` 표시 (시스템 자동 이벤트) |

## 11. 사용 시나리오

### 시나리오 1: 특정 채널의 전체 이력 조회
1. 사이드바 > 채널 관리 > 연동 이력 클릭
2. 채널 선택 드롭다운에서 "YouTube -- 서울TV" 선택
3. 해당 채널의 모든 이벤트가 테이블에 시간순으로 표시
4. "채널 연동" (초록), "토큰 갱신" (파랑) 등 색상 태그로 유형 구분
5. 처리자와 상세 정보 확인

### 시나리오 2: 토큰 만료 이벤트 추적
1. 토큰이 만료된 채널을 드롭다운에서 선택
2. 이력에서 "토큰 만료" (빨강 태그) 이벤트 확인
3. details 컬럼에서 만료 일시 확인
4. 해당 이벤트 전후의 "토큰 갱신" 이벤트 유무 확인
5. actor_id가 `-`이면 시스템 자동 감지, 값이 있으면 수동 작업

### 시나리오 3: 채널 변경 후 다른 채널 조회
1. "YouTube -- 서울TV" 이력 확인 완료
2. 드롭다운에서 "Instagram -- @seoul_official"로 변경
3. 자동으로 페이지 1로 초기화되어 Instagram 이력 표시
4. 다른 이벤트 유형 분포 확인

### 시나리오 4: 대량 이력 페이지 탐색
1. 채널 선택 후 50건 이상의 이력이 있는 경우
2. 하단 페이지네이션에서 "총 120건" 확인
3. 다음 페이지로 이동하여 이전 이력 확인
4. 각 페이지에 50건씩 표시

### 시나리오 5: 채널 선택 해제
1. 드롭다운의 X 버튼으로 채널 선택 해제 (allowClear)
2. 테이블이 빈 상태로 전환 -- "채널을 선택하세요" 안내
