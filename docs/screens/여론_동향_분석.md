# 여론 동향 분석 (SentimentTrendPage)

> **경로:** `/analytics/sentiment`
> **소스:** `frontend/src/features/analytics/pages/SentimentTrendPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 3
> **관련 기능:** F18 여론분석

---

## 1. 화면 개요

기관 SNS 채널에 달린 댓글과 반응의 감성(긍정/중립/부정/위험)을 시계열로 분석하는 화면이다.
부정 급증 알림, 감성 추이 스택 영역 차트, 키워드 빈도 테이블, 감성 알림 상세 테이블을 제공한다.
위험 키워드가 급증하면 상단에 경고 배너가 자동 표시되어, 담당자가 즉각 대응할 수 있도록 한다.

### 핵심 원칙
- **Human-in-the-Loop:** AI 감성 분석 결과는 "참고 자료"이며 최종 판단은 담당자가 한다
- **Fallback 필수:** 감성 데이터 없이도 키워드 빈도만으로 여론 동향 파악 가능
- **멀티테넌트 격리:** 현재 워크스페이스 기관의 댓글/반응 데이터만 분석

## 8. 연결 흐름 (Connected Flows)

### 8.1 여론 분석 진입 흐름

```
[사이드바] ──→ [성과 분석] ──→ [여론 동향 분석 /analytics/sentiment]
                                     │
                                     ├──(기간 변경)──→ 데이터 리패치
                                     │
                                     └──(알림 확인)──→ 부정 급증 키워드 대응
```

### 8.2 알림 대응 흐름

```
[부정 급증 알림 표시]
       │
       ├──→ 키워드 빈도 테이블에서 해당 키워드 확인
       │
       ├──→ 감성 알림 상세에서 변화율/위험도/신뢰도 확인
       │
       └──→ 댓글 관리 화면으로 이동하여 부정 댓글 대응
```

### 8.3 분석 화면 간 연결

```
[성과 분석 /analytics]
       │
       ├──→ [여론 동향 분석 /analytics/sentiment] ← 현재 화면
       │
       ├──→ [성과 예측 /analytics/prediction]
       │
       └──→ [벤치마크 분석 /analytics/benchmark]
```

## 10. 데이터 타입

### 10.1 감성 추이 데이터 (SentimentTrendData)

```typescript
interface SentimentTrendData {
  alerts: SentimentAlert[];
  sentiment_trend: SentimentTrendItem[];
  keyword_cloud: KeywordCloudItem[];
}
```

### 10.2 감성 추이 아이템 (SentimentTrendItem)

```typescript
interface SentimentTrendItem {
  date: string;       // 날짜 (YYYY-MM-DD)
  positive: number;   // 긍정 수치
  neutral: number;    // 중립 수치
  negative: number;   // 부정 수치
  dangerous: number;  // 위험 수치
}
```

### 10.3 감성 알림 (SentimentAlert)

```typescript
interface SentimentAlert {
  keyword: string;      // 알림 대상 키워드
  type: string;         // 유형 (NEGATIVE_SURGE, POSITIVE_SURGE)
  change_rate: number;  // 변화율 (%)
  risk_level: string;   // 위험도 (HIGH, MEDIUM, LOW)
  confidence: string;   // 신뢰도
  timeframe: string;    // 분석 기간
}
```

### 10.4 키워드 클라우드 아이템 (KeywordCloudItem)

```typescript
interface KeywordCloudItem {
  keyword: string;    // 키워드
  count: number;      // 출현 빈도
  sentiment: string;  // 감성 (POSITIVE, NEUTRAL, NEGATIVE, DANGEROUS)
  change: number;     // 변화율 (%)
}
```

## 12. 오류 처리 및 Fallback

| 상황 | 처리 |

## 13. 사용 시나리오

### 시나리오 1: 일상적 여론 모니터링

1. 사이드바 > 성과 분석 > 여론 동향 분석 클릭
2. 기간 "30일" 선택 확인
3. 감성 추이 차트에서 긍정/중립/부정 비율 변화 추이 확인
4. 키워드 빈도 테이블에서 주요 키워드와 감성 분포 확인
5. 변화율 높은 키워드 파악하여 콘텐츠 방향 조정

### 시나리오 2: 부정 급증 대응

1. 화면 접속 시 상단에 경고 배너 확인: "정책" 부정 급증 감지
2. 변화율 +45.2%, 위험도 HIGH 확인
3. 감성 알림 상세 테이블에서 신뢰도 및 기간 상세 확인
4. 키워드 빈도 테이블에서 "정책" 키워드의 출현 빈도 확인
5. 댓글 관리 화면으로 이동하여 관련 부정 댓글 대응

### 시나리오 3: 기간별 감성 비교

1. 기간 "7일" 선택 → 최근 단기 감성 추이 확인
2. 기간 "90일" 전환 → 장기 추세 비교
3. 특정 시점에 부정이 급증한 구간 파악
4. 해당 시점의 게시 콘텐츠와 연관 분석
