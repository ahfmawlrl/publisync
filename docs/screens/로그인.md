# 로그인 (LoginPage)

> **경로:** `/login`
> **소스:** `frontend/src/features/auth/pages/LoginPage.tsx`
> **접근 권한:** 공개 (비인증 사용자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F08 권한관리 (JWT 인증, 5회 실패 잠금, 로그인 유지)

---

## 1. 화면 개요

PubliSync 플랫폼의 진입점으로, 등록된 사용자가 이메일과 비밀번호로 인증하는 화면이다.
JWT HS256 기반 인증(access 30분 / refresh 7일)을 수행하며, 로그인 성공 시 토큰과 사용자 정보를
Zustand persist 스토어에 저장한다.

### 핵심 원칙
- **5회 실패 잠금:** 동일 계정 로그인 5회 연속 실패 시 30분 잠금 (HTTP 423)
- **로그인 유지:** `remember_me` 체크 시 세션 유지 기간 연장
- **공개 접근:** 인증 불필요, 비로그인 사용자만 접근 가능

## 5. 연결 흐름 (Connected Flows)

### 5.1 로그인 성공 흐름

```
[로그인 화면] ──(이메일+비밀번호)──→ POST /auth/login
       │
       ├──(성공)──→ Zustand setAuth(tokens, user)
       │                 │
       │                 ↓
       │           localStorage 'publisync-auth'에 persist
       │                 │
       │                 ↓
       │           message.success('로그인 성공')
       │                 │
       │                 ↓
       │           navigate('/') → [대시보드]
       │
       ├──(401 실패)──→ "이메일 또는 비밀번호가 올바르지 않습니다" 오류 표시
       │
       ├──(423 잠금)──→ "계정이 잠겼습니다. 30분 후 다시 시도하세요." 오류 표시
       │
       └──(기타 오류)──→ "로그인에 실패했습니다. 다시 시도하세요." 오류 표시
```

### 5.2 비밀번호 찾기 연결

```
[로그인 화면] ──("비밀번호 찾기" 클릭)──→ /reset-password
       │
       └──(비밀번호 변경 완료)──→ /login (복귀)
```

### 5.3 초대 수락 후 로그인 불필요

```
[초대 수락 화면] ──(가입 완료)──→ 자동 로그인 → [대시보드]
```

## 7. 데이터 타입

### 7.1 로그인 폼 데이터 (LoginFormData)

```typescript
interface LoginFormData {
  email: string;         // 이메일 (필수, 이메일 형식)
  password: string;      // 비밀번호 (필수)
  remember_me: boolean;  // 로그인 유지 (기본값: false)
}
```

### 7.2 로그인 요청 (LoginRequest)

```typescript
interface LoginRequest {
  email: string;        // 이메일
  password: string;     // 비밀번호
  remember_me?: boolean; // 로그인 유지
}
```

### 7.3 로그인 응답 (LoginResponse)

```typescript
interface LoginResponse {
  tokens: TokenResponse;  // JWT 토큰 쌍
  user: UserResponse;     // 사용자 정보
}

interface TokenResponse {
  access_token: string;   // JWT 액세스 토큰 (30분)
  refresh_token: string;  // JWT 리프레시 토큰 (7일)
  token_type: string;     // "bearer"
}

interface UserResponse {
  id: string;                     // 사용자 UUID
  email: string;                  // 이메일
  name: string;                   // 이름
  role: string;                   // RBAC 역할 (SA|AM|AO|CD)
  status: string;                 // 계정 상태 (ACTIVE|LOCKED 등)
  profile_image_url: string | null; // 프로필 이미지 URL
}
```

### 7.4 API 응답 래퍼 (ApiResponse)

```typescript
interface ApiResponse<T> {
  success: boolean;    // 성공 여부
  data: T;             // 응답 데이터
  error?: {
    code: string;      // 오류 코드
    message: string;   // 오류 메시지
  };
}
```

## 8. 상태 관리

| 영역 | 도구 | 설명 |
|------|------|------|
| **폼 상태** | Ant Design `Form.useForm<LoginFormData>()` | 이메일, 비밀번호, 로그인 유지 값 관리 |
| **로딩 상태** | React `useState<boolean>` | `loading` — API 호출 중 버튼 비활성화 |
| **서버 상태** | TanStack Query `useMutation` | `useLogin()` — 로그인 API 호출 |
| **인증 저장** | Zustand (persist) `useAuthStore` | accessToken, refreshToken, user 정보 저장 |
| **영속 저장** | localStorage `publisync-auth` | Zustand persist로 자동 관리, 새로고침 후에도 유지 |

## 9. 오류 처리

| 상황 | HTTP 상태 | 처리 |
|------|-----------|------|
| **이메일/비밀번호 불일치** | 401 | `message.error('이메일 또는 비밀번호가 올바르지 않습니다.')` — 서버 메시지 우선 표시 |
| **계정 잠금 (5회 실패)** | 423 | `message.error('계정이 잠겼습니다. 30분 후 다시 시도하세요.')` |
| **서버/네트워크 오류** | 500/기타 | `message.error('로그인에 실패했습니다. 다시 시도하세요.')` |
| **이메일 미입력** | (클라이언트) | "이메일을 입력하세요" 필드 하단 유효성 메시지 |
| **이메일 형식 오류** | (클라이언트) | "올바른 이메일 형식이 아닙니다" 필드 하단 유효성 메시지 |
| **비밀번호 미입력** | (클라이언트) | "비밀번호를 입력하세요" 필드 하단 유효성 메시지 |

## 10. 사용 시나리오

### 시나리오 1: 정상 로그인

1. `/login` 화면 진입
2. 이메일 입력 (예: `operator@agency.kr`)
3. 비밀번호 입력
4. [로그인] 버튼 클릭
5. "로그인 성공" 메시지 표시
6. 대시보드(`/`)로 자동 이동

### 시나리오 2: 로그인 유지 체크 후 로그인

1. 이메일, 비밀번호 입력
2. "로그인 유지" 체크박스 선택
3. [로그인] 클릭 → `remember_me: true`로 서버 전송
4. 서버가 연장된 세션의 리프레시 토큰 발급
5. 브라우저 닫고 재방문 시에도 인증 유지

### 시나리오 3: 5회 실패 후 계정 잠금

1. 잘못된 비밀번호로 5회 로그인 시도
2. 5회째 시도 → 서버 응답 HTTP 423
3. "계정이 잠겼습니다. 30분 후 다시 시도하세요." 오류 표시
4. 30분 경과 후 다시 로그인 가능

### 시나리오 4: 비밀번호 분실

1. 비밀번호가 기억나지 않음
2. "비밀번호 찾기" 링크 클릭
3. `/reset-password` 화면으로 이동

## 12. 보안 관련 사항

| 항목 | 설명 |
|------|------|
| **JWT 토큰** | access_token 30분, refresh_token 7일 |
| **계정 잠금** | IP + 계정 기준 5회 실패 → 30분 잠금 |
| **비밀번호 전송** | HTTPS(TLS 1.3) 상에서 JSON body로 전송 |
| **토큰 저장** | Zustand persist → localStorage (`publisync-auth`) |
| **autoComplete** | `off` (폼 레벨), `username`/`current-password` (필드 레벨, 패스워드 매니저 호환) |
| **Redis 블랙리스트** | 로그아웃 시 리프레시 토큰 무효화 |
