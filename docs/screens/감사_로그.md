# 감사 로그 (AuditLogsPage)

> **경로:** `/audit-logs`
> **소스:** `frontend/src/features/audit/pages/AuditLogsPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자), CD (위탁기관 담당자)
> **Phase:** 1-B (MVP 확장)
> **관련 기능:** F14 감사로그

---

## 1. 화면 개요

시스템 관리자(SA), 수탁업체 관리자(AM), 위탁기관 담당자(CD)가 플랫폼 내 모든 사용자 활동 이력을
조회하고 추적하는 화면이다. 콘텐츠 생성/수정/삭제, 채널 연결/해제, 로그인/로그아웃, 승인/반려 등
13가지 액션 유형에 대한 상세 로그를 시간순으로 표시하며,
액션/리소스 유형/날짜 범위별 필터링과 CSV 내보내기 기능을 제공한다.

### 핵심 원칙
- **감사 추적성:** 모든 주요 사용자 활동을 시간, 역할, IP 주소와 함께 기록
- **다차원 필터:** 액션, 리소스 유형, 날짜 범위 조합 필터링
- **데이터 내보내기:** CSV 형식 다운로드로 외부 분석 및 보고서 활용
- **공공기관 규정 준수:** 개인정보보호법 및 감사 요구사항 대응

## 6. 연결 흐름 (Connected Flows)

### 6.1 감사 로그 기록 흐름

```
[사용자 활동 발생]
       │
       ├── 콘텐츠 CRUD ──→ contents 서비스
       ├── 채널 연결/해제 ──→ channels 서비스
       ├── 로그인/로그아웃 ──→ auth 서비스
       ├── 승인/반려 ──→ approvals 서비스
       ├── 사용자 관리 ──→ users 서비스
       ├── 기관 관리 ──→ admin 서비스
       └── 설정 변경 ──→ settings 서비스
              │
              ↓
       [감사 로그 기록]
       (actor_id, actor_role, action, resource_type,
        resource_id, changes, ip_address, user_agent, request_id)
              │
              ↓
       [감사 로그 테이블 (audit_logs)]
              │
              ↓
       [감사 로그 화면에서 조회]
```

### 6.2 CSV 내보내기 흐름

```
[CSV 내보내기 클릭]
       │
       ↓
[날짜 범위 결정]
       │
       ├── 날짜 필터 설정됨 ──→ 해당 범위 사용
       └── 날짜 필터 없음 ──→ 최근 30일 기본 적용
              │
              ↓
GET /audit-logs/export?format=csv&start_date=...&end_date=...
              │
              ↓
[Blob 응답 수신] ──→ [브라우저 다운로드 트리거]
              │
              ↓
[audit_logs_{start}_{end}.csv 파일 저장]
```

### 6.3 연관 화면과의 관계

```
[사용자 관리 (/users)]
       │
       └── 사용자 CRUD 활동 ──→ [감사 로그] (action: CREATE/UPDATE/DELETE, resource: user)

[콘텐츠 관리 (/contents)]
       │
       └── 콘텐츠 CRUD/게시 ──→ [감사 로그] (action: CREATE/UPDATE/DELETE/PUBLISH, resource: content)

[채널 연동 (/channels)]
       │
       └── 채널 연결/해제 ──→ [감사 로그] (action: CONNECT/DISCONNECT, resource: channel)

[승인 관리 (/approvals)]
       │
       └── 승인/반려 처리 ──→ [감사 로그] (action: APPROVE/REJECT, resource: approval)
```

## 8. 데이터 타입

### 8.1 감사 로그 레코드 (AuditLogRecord)

```typescript
interface AuditLogRecord {
  id: string;                              // 로그 고유 ID (UUID)
  organization_id: string;                 // 소속 기관 ID
  actor_id: string | null;                 // 행위자 사용자 ID
  actor_name: string | null;               // 행위자 이름 (BE 응답에서 추가)
  actor_role: string | null;               // 행위자 역할 (SYSTEM_ADMIN 등)
  action: string;                          // 액션 유형 (CREATE/UPDATE/DELETE 등 13종)
  resource_type: string;                   // 리소스 유형 (content/channel/user 등 6종)
  resource_id: string | null;              // 대상 리소스 ID
  changes: Record<string, unknown> | null; // 변경 내역 (JSON)
  ip_address: string | null;              // 요청 IP 주소
  user_agent: string | null;              // 요청 User-Agent
  request_id: string | null;              // X-Request-Id (추적용)
  created_at: string;                      // 발생 시각 (ISO 8601)
}
```

### 8.2 감사 로그 필터 (AuditLogFilters)

```typescript
interface AuditLogFilters {
  page?: number;          // 현재 페이지

  action?: string;        // 액션 유형 필터
  resource_type?: string; // 리소스 유형 필터
  actor_id?: string;      // 행위자 ID 필터
  start_date?: string;    // 시작일 (ISO 8601)
  end_date?: string;      // 종료일 (ISO 8601)
}
```

### 8.3 내보내기 파라미터 (AuditLogExportParams)

```typescript
interface AuditLogExportParams {
  format: 'csv' | 'pdf';  // 내보내기 형식
  start_date: string;     // 시작일 (ISO 8601)
  end_date: string;       // 종료일 (ISO 8601)
  actions?: string;       // 액션 유형 필터 (선택)
```

## 9. 상태 관리

| 영역 | 도구 | 설명 |
|------|------|------|
| **필터** | React `useState` (`filters`) | 페이지/크기/액션/리소스유형/날짜 범위 통합 관리 |
| **날짜 범위** | React `useState` (`dateRange`) | RangePicker 표시용 dayjs 배열 |
| **내보내기** | React `useState` (`exporting`) | CSV 내보내기 로딩 상태 |
| **상세 ID** | React `useState` (`detailId`) | 선택된 로그 ID (null이면 모달 닫힘) |
| **서버 상태 (목록)** | TanStack Query `useQuery` | `useAuditLogs(filters)` — 감사 로그 목록 조회 |
| **서버 상태 (상세)** | TanStack Query `useQuery` | `useAuditLogDetail(detailId)` — 감사 로그 상세 조회 |

### filters 초기값

```typescript
{ page: 1, limit: 20 }
```

## 10. 오류 처리

| 상황 | 처리 |
|------|------|
| **감사 로그 로딩 실패** | 테이블 로딩 상태 유지, TanStack Query 자동 재시도 |

| **권한 부족 (403)** | 인터셉터에서 에러 처리 -- 접근 불가 안내 |

## 11. 사용 시나리오

### 시나리오 1: 특정 기간 콘텐츠 활동 조회
1. 리소스 유형 필터에서 "콘텐츠" 선택
2. 날짜 범위에서 "2026-03-01 ~ 2026-03-08" 선택
3. 해당 기간의 콘텐츠 관련 활동(생성/수정/삭제/게시) 목록 확인
4. 리소스 ID 복사로 특정 콘텐츠 추적

### 시나리오 2: 로그인 이력 감사
1. 액션 필터에서 "로그인" 선택
2. 날짜 범위에서 최근 7일 선택
3. 역할/IP 주소별 로그인 기록 확인
4. 비정상 IP에서의 접근 여부 점검

### 시나리오 3: 승인/반려 이력 추적
1. 액션 필터에서 "승인" 또는 "반려" 선택
2. 리소스 유형 필터에서 "승인" 선택
3. CD 역할 사용자의 승인/반려 처리 이력 확인
4. 리소스 ID로 해당 콘텐츠 원본 조회

### 시나리오 4: 감사 보고서 작성
1. 날짜 범위에서 "2026-02-01 ~ 2026-02-28" 선택
2. [CSV 내보내기] 클릭
3. `audit_logs_2026-02-01_2026-02-28.csv` 파일 다운로드
4. 스프레드시트에서 감사 보고서 작성

### 시나리오 5: 채널 연결/해제 추적
1. 리소스 유형 필터에서 "채널" 선택
2. 연결(CONNECT) / 해제(DISCONNECT) 이력 확인
3. IP 주소로 해당 작업 수행자의 접속 위치 확인
