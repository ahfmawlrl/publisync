## 1. 화면 개요

기관이 운영하는 SNS 채널(YouTube, Instagram, Facebook, X, 네이버 블로그)의 종합 성과를
한눈에 파악하는 분석 대시보드 화면이다. 주요 KPI 카드, 도달/참여 추이 라인 차트,
플랫폼별 성과 비교 바 차트, 상위 콘텐츠 TOP 5 리스트, 게시 시간대별 참여율 히트맵을
제공하며, 기간 선택(7일/30일/90일)에 따라 데이터가 갱신된다.
분석 결과는 PDF로 내보내기 할 수 있다.

### 핵심 원칙
- **워크플로우 내장 AI:** 히트맵 하단에 AI 최적 게시 시간 추천이 인라인으로 표시됨
- **Fallback 필수:** AI 추천 미표시 시에도 히트맵 데이터로 자체 판단 가능
- **멀티테넌트 격리:** 현재 워크스페이스 기관의 성과 데이터만 조회

## 4. KPI 통계 카드 (4개)

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Row` + `Col` (xs=24, sm=12, lg=6) × 4 |
| **카드 구조** | `Card` > `Statistic` (title, value) + `Text` (증감률) |

### 4.1 카드별 상세

| 카드 | Statistic title | 값 계산 | 증감 표시 |
|------|----------------|---------|----------|
| **총 도달** | "총 도달" | `performance.reduce(acc + total_views)` | `+12.3%` (success 색상) |
| **참여율** | "참여율" | `(sum(engagement_rate) / count).toFixed(1)` + `%` suffix | `+0.8%p` (success 색상) |
| **팔로워** | "팔로워" | `performance.reduce(acc + followers)` | `+1,423` (success 색상) |
| **게시물** | "게시물" | `performance.length * 25` + `건` 텍스트 | `+15건` (success 색상) |

**데이터 집계 로직:**
```typescript
const totals = (performance ?? []).reduce(
  (acc, item) => ({
    views: acc.views + item.total_views,
    followers: acc.followers + item.followers,
    contents: acc.contents + 1,
    engagement: acc.engagement + item.engagement_rate,
  }),
  { views: 0, followers: 0, contents: 0, engagement: 0 },
);
```

## 9. 연결 흐름 (Connected Flows)

### 9.1 대시보드 → 성과 분석 흐름

```
[대시보드] ──(성과 분석 메뉴)──→ [성과 분석 /analytics]
                                      │
                                      ├──(기간 변경)──→ 데이터 리패치
                                      │
                                      ├──(PDF 내보내기)──→ CSV 파일 다운로드
                                      │
                                      └──(히트맵 확인)──→ AI 최적 시간 참고
```

### 9.2 분석 하위 메뉴 연결

```
[성과 분석 /analytics] ←── 메인 분석 화면
       │
       ├──→ [여론 동향 분석 /analytics/sentiment] (Phase 3, F18)
       │
       ├──→ [성과 예측 /analytics/prediction] (Phase 3, F20)
       │
       └──→ [벤치마크 분석 /analytics/benchmark] (Phase 4, F23)
```

### 9.3 데이터 갱신 주기

```
[usePerformance] ──(5분 자동 리패치)──→ KPI + 바 차트 갱신
[useEngagementHeatmap] ──(5분 자동 리패치)──→ 히트맵 갱신
```

## 11. 데이터 타입

### 11.1 성과 데이터 (PerformanceData)

```typescript
interface PerformanceData {
  platform: string;        // 플랫폼 코드 (YOUTUBE, INSTAGRAM 등)
  followers: number;       // 팔로워 수
  total_views: number;     // 총 조회수
  total_likes: number;     // 총 좋아요
  total_shares: number;    // 총 공유
  total_comments: number;  // 총 댓글
  engagement_rate: number; // 참여율 (%)
  period: string;          // 조회 기간
}
```

### 11.2 참여율 히트맵 아이템 (EngagementHeatmapItem)

```typescript
interface EngagementHeatmapItem {
  hour: number;         // 시간 (0~23)
  day_of_week: number;  // 요일 (0=월~6=일)
  value: number;        // 참여율 값
}
```

### 11.3 분석 필터 (AnalyticsFilters)

```typescript
interface AnalyticsFilters {
  platform?: string;  // 플랫폼 필터
  period?: string;    // 기간 필터
}
```

| **히트맵 로딩 중** | 히트맵 카드 내 `Spin` (size="large", h-64 중앙) |
| **히트맵 데이터 없음** | `Empty` (description="참여율 데이터가 없습니다") |
| **내보내기 실패** | 에러 무시 (silently fail), 버튼 로딩 해제 |
| **TOP 5 데이터 없음** | "콘텐츠가 없습니다" (List locale emptyText) |

## 14. 사용 시나리오

### 시나리오 1: 월간 성과 확인

1. 사이드바 > 성과 분석 클릭
2. 기간 선택기에서 "30일" 확인 (기본값)
3. KPI 카드에서 총 도달/참여율/팔로워/게시물 확인
4. 도달/참여 추이 라인 차트에서 트렌드 파악
5. 플랫폼별 성과 비교 바 차트에서 채널별 조회수 비교

### 시나리오 2: 최적 게시 시간 확인

1. 성과 분석 화면 하단 히트맵 확인
2. 색상이 진한(빨간) 영역 = 참여율 높은 시간대 확인
3. 히트맵 셀에 마우스 오버하여 상세 참여율 확인
4. AI 추천 "화/목 09:00~10:00" 참고
5. 콘텐츠 작성 시 예약 게시 시간 설정에 활용

### 시나리오 3: 성과 리포트 내보내기

1. 기간 선택기에서 "90일" 선택
2. 데이터 갱신 확인
3. [PDF 내보내기] 버튼 클릭
4. CSV 파일 자동 다운로드
5. 외부 보고서 작성에 활용

### 시나리오 4: 플랫폼별 성과 비교

1. 성과 분석 화면 접속
2. 플랫폼별 성과 비교 바 차트에서 각 플랫폼 조회수 확인
3. 상위 콘텐츠 TOP 5에서 고성과 콘텐츠 파악
4. 플랫폼 태그(YT/IG/FB 등)로 어떤 플랫폼에서 성과가 좋은지 분석
