# 대시보드 (DashboardPage)

> **경로:** `/`
> **소스:** `frontend/src/features/dashboard/pages/DashboardPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F06 대시보드

---

## 1. 화면 개요

로그인 직후 가장 먼저 표시되는 메인 화면으로, 현재 워크스페이스의 핵심 운영 지표를 한눈에 보여준다.
KPI 카드, 플랫폼별 성과 추이 차트, 승인 대기 현황, 댓글 감성 분석, 최근 콘텐츠 목록, 오늘의 게시 일정 등
6개 영역으로 구성되며, 사용자 역할(RBAC)에 따라 배치와 강조 요소가 달라진다.

### 핵심 원칙
- **역할별 맞춤 뷰:** CD는 승인 현황이 최상단에 강조 표시되고, AM은 전체 기관 비교 뷰 안내가 표시됨
- **실시간성:** 모든 데이터가 5분 간격으로 자동 갱신 (refetchInterval: 300,000ms)
- **빈 상태 처리:** 데이터 미존재 시 각 영역에 적절한 안내 메시지 표시

## 11. 역할별 뷰 차이 (RBAC)

### 11.1 CD (위탁기관 담당자)

| 영역 | 동작 |
|------|------|
| **승인 대기 현황** | 최상단에 파란색 테두리 강조 배치 (KPI 카드 위) |
| **Row 2 우측** | 최근 게시 콘텐츠 카드 표시 (승인 카드가 상단으로 이동) |
| **Row 3 우측** | 비어 있음 (최근 콘텐츠가 Row 2로 이동) |
| **AM 안내 배너** | 미표시 |
| **목적** | 승인/반려 업무가 핵심이므로 승인 현황을 가장 먼저 확인 |

### 11.2 AM/SA (수탁업체 관리자 / 시스템 관리자)

| 영역 | 동작 |
|------|------|
| **기관별 현황 비교** | KPI 카드 위에 `Table<OrgSummaryItem>` 표시 — 기관명, 전체 콘텐츠, 게시 완료, 활성 채널, 대기 승인 |
| **승인 대기 현황** | Row 2 우측 표준 위치에 배치 |
| **Row 3 우측** | 최근 게시 콘텐츠 카드 표시 |
| **목적** | 담당 기관들의 전체 운영 현황 비교·파악 |

### 11.3 AO (수탁업체 실무자) / SA (시스템 관리자)

| 영역 | 동작 |
|------|------|
| **AM 안내 배너** | 미표시 |
| **승인 대기 현황** | Row 2 우측 표준 위치에 배치 |
| **Row 3 우측** | 최근 게시 콘텐츠 카드 표시 |
| **목적** | 콘텐츠 작성·게시 중심 운영 현황 파악 |

## 12. 연결 흐름 (Connected Flows)

### 12.1 대시보드 → 상세 화면 네비게이션

```
[대시보드]
    │
    ├──(위험 댓글 카드 클릭)──→ [위험 댓글 목록] (/comments/dangerous)
    │
    ├──(승인 대기 목록 보기 →)──→ [승인 대기 목록] (/approvals)
    │
    └──(기간 필터 변경)──→ 동일 화면 내 데이터 갱신
```

### 12.2 데이터 자동 갱신 흐름

```
[페이지 마운트]
    │
    ├──→ useDashboardSummary() ──→ GET /dashboard/summary
    ├──→ useRecentContents()   ──→ GET /dashboard/recent-contents?limit=10
    ├──→ useTodaySchedule()    ──→ GET /dashboard/today-schedule
    ├──→ useApprovalStatus()   ──→ GET /dashboard/approval-status
    ├──→ useSentimentSummary() ──→ GET /dashboard/sentiment-summary
    └──→ usePlatformTrends()   ──→ GET /dashboard/platform-trends

    (5분 간격 자동 갱신 — refetchInterval: 300,000ms)
```

## 17. 사용 시나리오

### 시나리오 1: CD — 승인 건 확인 후 승인 처리
1. 로그인 직후 대시보드 표시
2. 최상단 강조 카드에서 "검수 대기: 3건" 확인
3. "승인 대기 목록 보기 →" 클릭
4. `/approvals` 화면으로 이동하여 승인/반려 처리

### 시나리오 2: AO — 위험 댓글 즉시 대응
1. 대시보드에서 KPI 카드 "위험 댓글: 5" (빨간색) 확인
2. 위험 댓글 카드 클릭
3. `/comments/dangerous` 화면으로 이동하여 위험 댓글 대응

### 시나리오 3: AM — 기관별 현황 비교 + 플랫폼 성과 분석
1. 대시보드 상단 "기관별 현황 비교" 테이블에서 기관별 콘텐츠·채널·승인 현황 비교
2. 플랫폼별 성과 추이 차트에서 [조회수] 필터 선택
3. 특정 플랫폼의 조회수 추이를 LineChart로 확인
4. 기간 필터를 "최근 30일"로 변경하여 장기 추이 확인

### 시나리오 4: AO — 오늘의 게시 일정 확인
1. 대시보드 하단의 "오늘의 게시 일정" 영역 확인
2. 09:00 [YT] 콘텐츠 A — `● 게시 완료` 확인
3. 14:00 [IG] 콘텐츠 B — `○ 예약됨` 확인, 게시 예정 상태 파악

### 시나리오 5: 전체 감성 현황 파악
1. 댓글 감성 분석 도넛 차트에서 긍정/중립/부정/위험 비율 확인
2. 위험 비율이 높은 경우 위험 댓글 KPI 카드 클릭으로 즉시 대응
