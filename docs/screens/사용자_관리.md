# 사용자 관리 (UsersPage)

> **경로:** `/users`
> **소스:** `frontend/src/features/settings/pages/UsersPage.tsx`
> **접근 권한:** SA (시스템 관리자), AM (수탁업체 관리자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F08 권한관리

---

## 1. 화면 개요

시스템 관리자(SA) 또는 수탁업체 관리자(AM)가 플랫폼에 등록된 사용자를 조회하고, 신규 사용자 추가,
역할 변경, 삭제 등의 사용자 관리 작업을 수행하는 화면이다.
RBAC 4개 역할(SA, AM, AO, CD)을 기반으로 사용자별 권한을 할당하며,
검색 및 필터를 통해 대규모 사용자 목록을 효율적으로 관리할 수 있다.

### 핵심 원칙
- **멀티테넌트 격리:** 사용자는 소속 기관(Organization)에 연결되며, RLS로 데이터 격리
- **RBAC 기반 관리:** 4개 역할 체계로 세분화된 권한 제어
- **검색/필터 지원:** 이름, 이메일, 역할, 상태별 빠른 검색 가능

## 6. 사용자 테이블

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Table` (rowKey="id") |
| **데이터 소스** | `data?.data || []` |
| **페이지 크기** | 20건 |

### 6.1 테이블 컬럼

| 컬럼 | dataIndex | 렌더링 |
|------|-----------|--------|
| **이름** | `name` | 텍스트 |
| **이메일** | `email` | 텍스트 |
| **역할** | `role` | `Tag` (색상: SA=빨강, AM=파랑, AO=녹색, CD=주황) |
| **상태** | `status` | `Tag` (활성=녹색, 비활성=기본, 잠금=빨강, 탈퇴=회색) |
| **최근 로그인** | `last_login_at` | `toLocaleDateString('ko-KR')` 또는 '-' |
| **관리** | - | 수정(EditOutlined) + 삭제(DeleteOutlined) 버튼 |

### 6.2 역할 색상 매핑

| 역할 코드 | 한글명 | Tag 색상 |
|-----------|--------|----------|
| `SYSTEM_ADMIN` | 시스템 관리자 | `red` |
| `AGENCY_MANAGER` | 수탁업체 관리자 | `blue` |
| `AGENCY_OPERATOR` | 수탁업체 실무자 | `green` |
| `CLIENT_DIRECTOR` | 위탁기관 담당자 | `orange` |

### 6.3 상태 색상 매핑

| 상태 코드 | 한글명 | Tag 색상 |
|-----------|--------|----------|
| `ACTIVE` | 활성 | `green` |
| `INACTIVE` | 비활성 | `default` |
| `LOCKED` | 잠금 | `red` |
| `WITHDRAWN` | 탈퇴 | `gray` |

### 6.4 페이지네이션
| 항목 | 내용 |
|------|------|
| **현재 페이지** | `page` 상태 |
| **전체 건수** | `data?.meta?.total || 0` |
| **페이지 크기** | 20 |
| **총 건수 표시** | `총 ${total}명` |

### 6.5 행 액션 버튼

| 버튼 | 아이콘 | 동작 |
|------|--------|------|
| **수정** | `EditOutlined` | `handleEdit(record)` -- 사용자 수정 모달 열기 |
| **삭제** | `DeleteOutlined` (danger) | `Popconfirm` 확인 후 `deleteMutation.mutate(record.id)` |

## 8. 연결 흐름 (Connected Flows)

### 8.1 사용자 CRUD 흐름

```
[사용자 관리 화면]
       │
       ├──(사용자 추가)──→ [추가 모달] ──→ POST /users ──→ 목록 갱신
       │
       ├──(사용자 수정)──→ [수정 모달] ──→ PUT /users/:id ──→ 목록 갱신
       │
       └──(사용자 삭제)──→ [Popconfirm] ──→ DELETE /users/:id ──→ 목록 갱신
```

### 8.2 검색/필터 흐름

```
[검색어 입력 / 역할 선택 / 상태 선택]
       │
       ↓
[page=1 초기화 + queryKey 변경]
       │
       ↓
[TanStack Query 자동 재요청]
       │
       ↓
[GET /users?page=1&search=...&role=...&status=...]
       │
       ↓
[테이블 데이터 갱신]
```

### 8.3 역할 관리와의 연결

```
[사용자 목록 탭] ←──→ [역할 관리 탭] (Phase 1-B 예정)
                           │
                           └──→ 역할별 권한 상세 설정
                                RBAC 매트릭스 관리
```

| 동작 | HTTP | 엔드포인트 | Hook | Phase |

## 10. 데이터 타입

### 10.1 사용자 레코드 (UserRecord)

```typescript
interface UserRecord {
  id: string;                   // 사용자 고유 ID (UUID)
  email: string;                // 이메일 주소
  name: string;                 // 사용자 이름
  role: string;                 // 역할 (SYSTEM_ADMIN | AGENCY_MANAGER | AGENCY_OPERATOR | CLIENT_DIRECTOR)
  status: string;               // 상태 (ACTIVE | INACTIVE | LOCKED | WITHDRAWN)
  last_login_at: string | null; // 최근 로그인 시간 (ISO 8601)
  created_at: string;           // 생성일시 (ISO 8601)
}
```

### 10.2 사용자 생성 요청

```typescript
interface UserCreateData {
  email: string;           // 이메일 (필수, email 형식)
  name: string;            // 이름 (필수)
  role: Role;              // 역할 (필수)
  organization_id: string; // 소속 기관 ID (필수)
}
```

### 10.3 사용자 수정 요청

```typescript
interface UserUpdateData {
  id: string;       // 대상 사용자 ID (필수)
  name?: string;    // 이름 (선택)
  role?: string;    // 역할 (선택)
}
```

### 10.4 API 응답 타입

```typescript
// 목록 조회 응답
PaginatedResponse<UserRecord> {
  data: UserRecord[];
  meta: {
    total: number;
    page: number;
    limit: number;
  };
}

// 단건 응답

  success: boolean;
```

## 13. 사용 시나리오

### 시나리오 1: 신규 사용자 추가
1. [사용자 추가] 버튼 클릭
2. 이름, 이메일, 역할(수탁업체 실무자), 소속 기관 입력
3. [추가] 클릭 -- POST /users 호출
4. 성공 시 "사용자가 생성되었습니다" 메시지 + 모달 닫힘 + 목록 갱신

### 시나리오 2: 사용자 역할 변경
1. 대상 사용자 행의 수정(연필) 아이콘 클릭
2. 수정 모달에서 역할을 "수탁업체 관리자"로 변경
3. [저장] 클릭 -- PUT /users/:id 호출
4. 성공 시 "사용자 정보가 수정되었습니다" 메시지 + 테이블 역할 Tag 갱신

### 시나리오 3: 특정 역할 사용자 검색
1. 역할 필터에서 "위탁기관 담당자" 선택
2. 검색창에 "서울" 입력 후 검색
3. 위탁기관 담당자 중 이름/이메일에 "서울"이 포함된 사용자만 표시

### 시나리오 4: 잠금 상태 사용자 확인
1. 상태 필터에서 "잠금" 선택
2. 로그인 5회 실패로 잠금된 사용자 목록 확인
3. 필요 시 수정 모달에서 상태 변경 또는 삭제

### 시나리오 5: 사용자 삭제
1. 대상 사용자 행의 삭제(휴지통) 아이콘 클릭
2. "사용자를 삭제하시겠습니까?" 확인 팝업 표시
3. [삭제] 클릭 -- DELETE /users/:id 호출
4. 성공 시 "사용자가 삭제되었습니다" 메시지 + 목록에서 제거
