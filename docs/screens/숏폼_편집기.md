# 숏폼 편집기 (ShortformEditorPage)

> **경로:** `/ai/shortform-editor/:assetId`
> **소스:** `frontend/src/features/ai/pages/ShortformEditorPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자)
> **Phase:** 2
> **관련 기능:** F15 AI 숏폼 생성

---

## 1. 화면 개요

수탁업체 실무자(AO) 또는 관리자(AM)가 긴 영상에서 숏폼(쇼츠/릴스) 후보 구간을 AI로 자동 추출하고,
선택/해제 방식으로 최종 클립을 결정하는 화면이다. 좌측에 동영상 미리보기를, 우측에 AI가 추출한 클립 구간
목록을 배치한 2컬럼 레이아웃으로 구성된다. 각 클립의 미리보기 재생, 선택/해제, 총 재생시간 계산,
선택 구간 요약 확인 기능을 제공한다.

### 핵심 원칙
- **Human-in-the-Loop:** AI가 구간을 추출하지만, 최종 선택/해제는 사용자가 수행
- **Fallback 필수:** AI 구간 추출 실패 시 Fallback 안내 표시
- **워크플로우 내장 AI:** 숏폼 편집 화면 안에서 AI 추출을 직접 호출

## 5. 우측: AI 작업 진행 + 클립 목록

### 5.1 AI 작업 진행 상황 (AiJobProgress)

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `AiJobProgress` (자체 컴포넌트) |
| **표시 조건** | `isJobRunning` (jobId가 있을 때) |
| **props** | `jobId`, `title="AI 구간 추출"`, `onComplete`, `onClose` |
| **폴링** | 3초 간격 GET `/ai/jobs/{jobId}` (useJobStatus) |

#### 작업 상태 표시

| 상태 | Tag 색상 | 아이콘 | 라벨 |
|------|---------|--------|------|
| PENDING | default | Clock | 대기 중 |
| PROCESSING | processing | Loader | 처리 중 |
| COMPLETED | success | CheckCircle | 완료 |
| FAILED | error | XCircle | 실패 |

#### 완료 시 동작 (handleJobComplete)

| 항목 | 내용 |
|------|------|
| **결과 파싱** | `result.clips` 배열에서 `ClipSegment[]` 변환 |
| **매핑** | `id` (문자열), `start` (숫자, 초), `end` (숫자, 초), `label` (문자열) 추출 |
| **기본 선택** | 모든 추출 구간 `selected: true` (전체 선택 상태) |
| **성공** | "{N}개 구간이 추출되었습니다" toast |
| **파싱 실패** | "구간 결과를 파싱할 수 없습니다. 수동으로 입력해주세요." warning toast (Fallback) |
| **중복 방지** | `resultConsumedRef`로 결과가 한 번만 처리되도록 보장 |

### 5.2 클립 목록 카드

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Card` (size="small") |
| **제목** | "추출 구간 ({선택 수}/{전체 수}개 선택)" |
| **extra** | "총 {시간}" (선택된 클립 총 재생시간, 클립 있을 때만 표시) |
| **스크롤** | `max-h-[600px] overflow-y-auto` |

#### 빈 상태

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Empty` (PRESENTED_IMAGE_SIMPLE) |
| **메시지** | "추출된 구간이 없습니다. AI 구간 추출 버튼을 눌러주세요." |

#### 클립 항목 (ClipSegment)

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| **선택 체크박스** | `Checkbox` | `selected` 상태 토글 (handleToggleSelect) |
| **구간 레이블** | `Text` (strong, text-sm) | `clip.label` (예: "하이라이트 1") |
| **구간 길이** | `Tag` (color="blue", text-xs) | `formatDuration(start, end)` (예: "30.0초", "1분 15초") |
| **시간 범위** | `Text` (secondary, text-xs) | `formatTime(start) ~ formatTime(end)` (예: "01:20 ~ 01:50") |
| **미리보기 버튼** | `Button` (type="text", size="small", icon=PlayCircleOutlined) | "미리보기" — 해당 구간 시작 시간으로 비디오 seekTo |

## 7. 시간 포맷 헬퍼

### 7.1 formatTime(seconds)

| 입력 | 출력 | 설명 |
|------|------|------|
| 0 | "00:00" | MM:SS 형식 |
| 90 | "01:30" | 1분 30초 |
| 185 | "03:05" | 3분 5초 |

### 7.2 formatDuration(start, end)

| 입력 | 출력 | 설명 |
|------|------|------|
| (0, 30) | "30.0초" | 60초 미만: N.N초 |
| (0, 90) | "1분 30초" | 60초 이상: N분 N초 |
| (10, 10) | "0.0초" | 동일 시간: 0초 |

## 8. 연결 흐름 (Connected Flows)

### 8.1 AI 구간 추출 흐름

```
[AI 구간 추출 버튼 클릭]
       │
       ↓
POST /ai/extract-shortform (media_asset_id) → 202 Accepted
       │
       ↓
[jobId 반환] → AiJobProgress 표시
       │
       ↓
[3초 간격 폴링] GET /ai/jobs/:jobId
       │
       ├──(PENDING)──→ 대기 중 표시
       ├──(PROCESSING)──→ 처리 중 + Progress 바
       ├──(FAILED)──→ 오류 메시지 Alert
       └──(COMPLETED)──→ result.clips 파싱
                              │
                        ┌─────┴──────┐
                        ↓            ↓
                  [파싱 성공]    [파싱 실패]
                  N개 클립 목록   "수동으로 입력해주세요" (Fallback)
                  에 로드
                  (전체 선택 상태)
```

### 8.2 클립 선택/미리보기 흐름

```
[클립 목록]
       │
       ├──(체크박스 클릭)──→ handleToggleSelect → selected 토글
       │                       → 선택 수/총 시간 자동 재계산
       │
       └──(미리보기 클릭)──→ handlePreviewClip
                               │
                               ├──(seekTo 가능)──→ 비디오 해당 시간으로 이동
                               │                    "{MM:SS}부터 재생합니다" toast
                               │
                               └──(seekTo 불가)──→ (동작 없음)
```

### 8.3 구간 확인 흐름

```
[선택한 구간 확인 버튼 클릭]
       │
       ├──(선택된 클립 없음)──→ "선택된 구간이 없습니다" warning toast
       │
       └──(선택된 클립 있음)──→ [요약 모달]
                                   │
                                   ↓
                              총 N개 구간 / 총 재생시간
                              각 구간의 레이블, 시간 범위, 길이 표시
                                   │
                                   ↓
                              [닫기] → 모달 닫기
```

## 10. 데이터 타입

### 10.1 클립 구간 (ClipSegment, 컴포넌트 내부)

```typescript
interface ClipSegment {
  id: string;              // 클립 고유 ID (예: "clip-1")
  start: number;           // 시작 시간 (초)
  end: number;             // 종료 시간 (초)
  label: string;           // 구간 레이블 (예: "하이라이트 1")
  selected: boolean;       // 선택 여부
}
```

### 10.2 숏폼 추출 요청 (ShortformRequest)

```typescript
interface ShortformRequest {
  media_asset_id: string;       // 대상 미디어 에셋 ID
  target_duration?: number;      // 목표 길이 (초, 선택)
  count?: number;                // 추출 구간 수 (선택)
  style?: string;                // 스타일 (선택)
}
```

### 10.3 AI 작업 생성 응답 (AiJobCreateResponse)

```typescript
interface AiJobCreateResponse {
  job_id: string;          // 작업 ID (폴링 키)
  job_type: string;        // 작업 유형 (SHORTFORM)
  status: string;          // 초기 상태
  message: string;         // 안내 메시지
}
```

### 10.4 AI 작업 상태 (AiJobStatus)

```typescript
interface AiJobStatus {
  job_id: string;
  job_type: string;
  status: 'PENDING' | 'PROCESSING' | 'COMPLETED' | 'FAILED';
  progress: number;                       // 0~100
  result: Record<string, unknown> | null; // 완료 시 { clips: [...] }
  error_message: string | null;           // 실패 시 오류 메시지
  started_at: string | null;
  completed_at: string | null;
  created_at: string;
}
```

## 13. 사용 시나리오

### 시나리오 1: AI 숏폼 구간 자동 추출
1. 미디어 라이브러리에서 긴 영상 에셋 선택 → 숏폼 편집기 진입
2. 좌측에서 원본 동영상 미리보기
3. [AI 구간 추출] 클릭 → "AI 구간 추출 작업을 시작했습니다" toast
4. AiJobProgress에서 진행률 확인 (대기 중 → 처리 중 → 완료)
5. 완료 시 "5개 구간이 추출되었습니다" toast → 우측 클립 목록에 전체 선택 상태로 로드

### 시나리오 2: 클립 선택/해제 및 미리보기
1. AI 추출된 클립 목록에서 각 구간의 체크박스로 선택/해제
2. 선택 수와 총 재생시간이 실시간 업데이트
3. 특정 클립의 [미리보기] 클릭 → 좌측 비디오가 해당 시간으로 이동
4. "01:20부터 재생합니다" toast 확인

### 시나리오 3: 선택 구간 최종 확인
1. 필요한 구간만 선택 완료
2. [선택한 구간 확인] 클릭 → 요약 모달 열림
3. 총 3개 구간, 총 1분 30초 확인
4. 각 구간의 레이블, 시간 범위, 길이 확인
5. [닫기] 클릭

### 시나리오 4: AI 실패 시 Fallback
1. [AI 구간 추출] 클릭 → 작업 시작
2. 서버 오류로 작업 FAILED → AiJobProgress에 오류 메시지 표시
3. [닫기] 버튼으로 진행 상황 카드 닫기
4. "구간 결과를 파싱할 수 없습니다. 수동으로 입력해주세요." 안내 확인
