# 답글 템플릿 관리 (ReplyTemplatesPage)

> **경로:** `/comments/reply-templates`
> **소스:** `frontend/src/features/comments/pages/ReplyTemplatesPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자)
> **Phase:** 1-B (MVP 확장)
> **관련 기능:** F04 댓글관리

---

## 1. 화면 개요

댓글 답변 시 자주 사용하는 정형화된 답글을 템플릿으로 등록·관리하는 화면이다.
민원/불만, 정책 문의, 칭찬/감사, 행사/일정, 기타 5개 카테고리로 분류하며,
`{기관명}`, `{연락처}`, `{담당부서}` 등의 변수를 포함하여 기관별로 자동 치환되는
재사용 가능한 답글 템플릿을 관리한다.
각 템플릿의 사용 횟수를 추적하고, 활성/비활성 토글로 사용 여부를 제어한다.

### 핵심 원칙
- **워크플로우 내장:** 댓글 답변 시 템플릿을 바로 선택하여 적용 가능
- **Fallback 필수:** 템플릿 없이도 수동 답글 작성 가능
- **멀티테넌트 격리:** 템플릿은 기관(organization_id) 단위로 격리

### 7.5 AI 템플릿 개선 모달

| 항목 | 내용 |
|------|------|
| **제목** | "AI 템플릿 개선" |
| **너비** | 560px |
| **열기** | 관리 컬럼의 `RobotOutlined` 버튼 클릭 시 |
| **닫기** | "취소" 버튼 또는 X 버튼 |

#### 모달 상태별 표시

| 상태 | 표시 |
|------|------|
| **AI 처리 중** | `Spin` ("AI가 템플릿을 개선하고 있습니다...") — h-32 가운데 정렬 |
| **개선 완료** | 기존 내용 (읽기 전용) + AI 개선안 (`TextArea`, 편집 가능) |
| **결과 없음** | "개선된 내용이 없습니다." |

#### 개선 결과 영역

| 요소 | 설명 |
|------|------|
| **기존 내용** | 원본 템플릿 내용 (`bg-gray-50 p-3`, 읽기 전용) |
| **AI 개선안** | `TextArea` (rows=5, value=`improvedContent`, 수정 가능) |
| **적용 버튼** | "적용" — 개선된 내용을 편집 폼으로 이동 (수정 모달 열기) |
| **취소 버튼** | "취소" — 모달 닫기 |

#### AI 개선 API

| 항목 | 내용 |
|------|------|
| **엔드포인트** | `POST /ai/improve-template` |
| **요청 데이터** | `{ template_content: string, category: string }` |
| **응답 데이터** | `{ suggestions: [{ content: string }] }` |
| **Hook** | inline `useMutation` (cross-feature import 방지를 위해 `apiClient.post` 직접 사용) |
| **오류 처리** | "AI 개선에 실패했습니다" error toast |

#### 적용 흐름

```
[AI 개선 버튼 클릭]
       │
       ↓
[AI 모달 열기] → POST /ai/improve-template → [개선안 수신]
       │
       ↓
[사용자가 개선안 검토/편집]
       │
       ├── [적용] → 수정 모달 열기 (form에 개선된 내용 세팅) → 사용자 확인 후 [저장]
       └── [취소] → 모달 닫기
```

## 10. 데이터 타입

### 10.1 답글 템플릿 레코드 (ReplyTemplateRecord)

```typescript
interface ReplyTemplateRecord {
  id: string;                      // 템플릿 고유 ID
  organization_id: string;         // 기관 ID (RLS)
  category: string;                // 카테고리 (민원·불만/정책 문의/칭찬·감사/행사·일정/기타)
  name: string;                    // 템플릿명
  content: string;                 // 템플릿 내용 (변수 포함 가능)
  variables: string[] | null;      // 사용된 변수 목록
  usage_count: number;             // 사용 횟수
  is_active: boolean;              // 활성 상태
  created_by: string;              // 생성자 ID
  created_at: string;              // 생성 일시
  updated_at: string;              // 수정 일시
}
```

### 10.2 템플릿 생성 요청 (ReplyTemplateCreateData)

```typescript
interface ReplyTemplateCreateData {
  category: string;        // 카테고리 (필수)
  name: string;            // 템플릿명 (필수)
  content: string;         // 내용 (필수)
  variables?: string[];    // 변수 목록 (선택)
}
```

### 10.3 템플릿 수정 요청 (ReplyTemplateUpdateData)

```typescript
interface ReplyTemplateUpdateData {
  category?: string;       // 카테고리
  name?: string;           // 템플릿명
  content?: string;        // 내용
  variables?: string[];    // 변수 목록
  is_active?: boolean;     // 활성 상태
}
```

## 13. 사용 시나리오

### 시나리오 1: 새 템플릿 생성
1. 사이드바 > 댓글 > 답글 템플릿 관리 진입
2. [+ 새 템플릿] 클릭 → 생성 모달 열림
3. 템플릿명: "민원 기본 안내" 입력
4. 카테고리: "민원·불만" 선택
5. 내용: "안녕하세요, {기관명}입니다. 불편을 드려 죄송합니다. {담당부서}에서 확인 후 연락드리겠습니다. 문의: {연락처}" 입력
6. [생성] 클릭 → 목록에 추가, 기본 활성 상태

### 시나리오 2: 기존 템플릿 수정
1. 수정할 템플릿의 ✏️ 버튼 클릭
2. 수정 모달에 기존 값이 채워짐
3. 내용 수정 또는 카테고리 변경
4. 활성 상태를 비활성으로 변경 (Switch OFF)
5. [저장] 클릭 → 변경사항 반영

### 시나리오 3: 카테고리별 템플릿 관리
1. 카테고리 필터에서 "정책 문의" 선택
2. 정책 문의 관련 템플릿만 표시
3. 각 템플릿의 사용 횟수를 확인하여 활용도 파악
4. 사용 횟수가 낮은 템플릿은 내용 개선 또는 비활성 처리

### 시나리오 4: 불필요한 템플릿 삭제
1. 더 이상 사용하지 않는 템플릿의 🗑 버튼 클릭
2. Popconfirm "템플릿을 삭제하시겠습니까?" 표시
3. [삭제] 클릭 → 영구 삭제, 목록 갱신

### 시나리오 5: 템플릿 활용도 분석
1. 전체 템플릿 목록에서 "사용 횟수" 컬럼 확인
2. 가장 많이 사용된 템플릿의 패턴을 파악
3. 신규 카테고리 또는 변형 템플릿 추가 검토
