# 새 콘텐츠 작성 (ContentCreatePage)

> **경로:** `/contents/create`
> **소스:** `frontend/src/features/contents/pages/ContentCreatePage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F01 통합게시, F02 AI메타데이터, F09 승인워크플로우, F17 톤변환, F21 표현검수

---

## 1. 화면 개요

수탁업체 실무자(AO) 또는 관리자(AM)가 다수 SNS 플랫폼(YouTube, Instagram, Facebook, X, 네이버 블로그)에
동시 게시할 콘텐츠를 작성하는 화면이다. AI 어시스턴트가 제목·설명문·해시태그를 제안하고,
작성 완료 후 위탁기관 담당자(CD)에게 검토를 요청하는 승인 워크플로우의 시작점이다.

### 핵심 원칙
- **Human-in-the-Loop:** AI 산출물은 "제안"이며, 최종 결정은 사용자가 한다
- **Fallback 필수:** AI 실패 시에도 수동 입력으로 작업 계속 가능
- **워크플로우 내장 AI:** AI 기능이 별도 메뉴가 아닌 작성 화면 안에서 호출됨

## 5. 좌측: 콘텐츠 편집 영역

### 5.1 미디어 업로드

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `MediaUpload` (`@/shared/components/MediaUpload.tsx`) |
| **Form 필드명** | `media_urls` |
| **라벨** | "미디어 업로드" |
| **업로드 방식** | 드래그 앤 드롭 + 클릭 (react-dropzone 사용) |
| **지원 형식** | JPEG, PNG, GIF, WebP, MP4, WebM, PDF |
| **파일 크기 제한** | 최대 100MB |
| **파일 개수 제한** | 최대 10개 (props로 설정) |
| **업로드 프로세스** | 1) `POST /media/presigned-upload` → Presigned URL 발급 → 2) MinIO 직접 PUT 업로드 |
| **진행률 표시** | 업로드 중 파일별 Progress 바 표시 |
| **미리보기** | 이미지 파일은 40x40 썸네일 표시, 그 외는 "파일" 아이콘 |
| **삭제** | 각 파일 우측 🗑️ 버튼으로 개별 삭제 |

### 5.2 제목 입력

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Input` (maxLength=500, showCount) |
| **Form 필드명** | `title` |
| **라벨** | "제목" |
| **플레이스홀더** | "콘텐츠 제목" |
| **유효성 검사** | 필수 (`required: true`, 메시지: "제목을 입력하세요") |
| **AI 연동** | 필드 하단에 `[🤖 AI 제목 제안]` 링크 버튼 |
| **실시간 미리보기** | `Form.useWatch('title')` — 입력 즉시 우측 미리보기에 반영 |

### 5.3 본문/설명문 입력

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Input.TextArea` (rows=8) |
| **Form 필드명** | `body` |
| **라벨** | "본문/설명문" |
| **플레이스홀더** | "콘텐츠 본문을 작성하세요" |
| **AI 최소 요건** | 10자 이상 입력 시 AI 제안 요청 가능 (미만 시 경고 toast) |
| **실시간 미리보기** | `Form.useWatch('body')` — 입력 즉시 우측 미리보기에 반영 |

### 5.4 해시태그 입력

| 항목 | 내용 |
|------|------|
| **컴포넌트** | `Select` (mode="tags", tokenSeparators={[' ', ',']}) |
| **Form 필드명** | `hashtags` |
| **라벨** | "해시태그" |
| **플레이스홀더** | "#서울시 #정책브리핑 #3월" |
| **입력 방식** | 태그 모드 — 스페이스/쉼표로 자동 분리, 개별 삭제 가능 |
| **AI 연동** | 필드 하단에 `[🤖 AI 해시태그 추천]` 링크 버튼 — AI 제안 선택(onSelect) 시 해시태그 배열 필드에 직접 추가 |
| **실시간 미리보기** | `Form.useWatch('hashtags')` — 입력 즉시 우측 미리보기에 반영 |

#### AI 제안 결과 표시 (AiSuggestionPanel)

각 AI 제안 결과는 `AiSuggestionPanel` 컴포넌트로 일관되게 표시된다.

| 요소 | 설명 |
|------|------|
| **제안 목록** | 각 제안의 `content`(텍스트)와 `score`(신뢰도) 표시 |
| **신뢰도 바** | Progress 바 + 색상 태그 (≥85% 녹색, ≥70% 파란색, ≥50% 노란색, <50% 빨간색) |
| **[사용하기] 버튼** | 클릭 시 해당 제안을 폼 필드에 자동 적용 |
| **모델 정보** | 사용된 AI 모델명 (Tag, blue) |
| **처리 시간** | 처리 소요 시간 (초 단위) |
| **로딩 상태** | "AI가 제안을 생성하고 있습니다..." (Spin + 파란 배경) |
| **오류 상태** | "AI 제안 생성 실패" + "직접 입력하여 계속 작업할 수 있습니다" (Fallback 안내) |
| **면책 문구** | "AI가 생성한 제안입니다. 최종 결정은 사용자가 합니다." |

#### AI 제안 적용 로직

| 기능 | 적용 방식 |
|------|----------|
| **제목 제안** | `form.setFieldValue('title', content)` — 제목 필드 대체 |
| **설명문 제안** | 기존 본문 뒤에 `\n\n` 구분 후 추가 (비어있으면 바로 적용) |
| **해시태그 제안** | `form.setFieldValue('hashtags', [...currentHashtags, ...newTags])` — 해시태그 배열 필드에 직접 추가 |

```
       │                           └──→ [이 화면으로 복귀]
       │
       └──(임시 저장)──→ [콘텐츠 목록] (status: DRAFT)

```

| **파일 업로드 실패** | 파일명 + 오류 메시지 toast |

## 14. 사용 시나리오

### 시나리오 1: 기본 콘텐츠 작성 (수동)
1. 사이드바 > 콘텐츠 > 새 콘텐츠 작성 클릭
2. YouTube, Instagram 체크
3. 미디어 파일 드래그 앤 드롭
4. 제목, 본문, 해시태그 직접 입력
5. 예약 게시일시 선택
6. [저장 (초안)] 클릭 → 콘텐츠 목록으로 이동

### 시나리오 2: AI 활용 콘텐츠 작성
1. 본문을 10자 이상 입력
2. AI 어시스턴트 > [제목 제안 받기] 클릭
3. 3개 제안 중 마음에 드는 것 [사용하기] 클릭 → 제목 자동 입력
4. [해시태그 제안 받기] → 추천 해시태그 적용
5. [AI 콘텐츠 검수] → 맞춤법·민감표현 검사 결과 확인
6. 이슈 있으면 수정 후 [검토 요청] 클릭

### 시나리오 3: 톤 변환 후 검토 요청
1. 본문 작성 완료
2. [AI 톤 변환] → 플랫폼: Instagram, 톤: 캐주얼 선택 → [변환하기]
3. 변환 결과 중 적합한 것 [적용하기] → 본문 대체
4. [검토 요청] → CD에게 알림 발송, 승인 대기 목록에 추가

### 시나리오 4: AI 실패 시 Fallback
1. [AI 제목 제안] 클릭 → AI 서비스 오류
2. "AI 제안 생성 실패" + "직접 입력하여 계속 작업할 수 있습니다" 표시
3. 사용자가 수동으로 제목 입력하여 작성 계속
