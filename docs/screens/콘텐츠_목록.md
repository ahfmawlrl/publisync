# 콘텐츠 목록 (ContentsListPage)

> **경로:** `/contents`
> **소스:** `frontend/src/features/contents/pages/ContentsListPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 1-A (최소 MVP)
> **관련 기능:** F01 통합게시, F09 승인워크플로우

---

## 1. 화면 개요

조직에 속한 모든 콘텐츠를 목록 형태로 조회하고, 상태별 필터링/검색/플랫폼 필터/기간 필터를 통해
원하는 콘텐츠를 빠르게 찾을 수 있는 화면이다. 행 클릭으로 상세 페이지로 이동하고,
개별 또는 일괄 삭제, 검토 요청 등의 액션을 수행할 수 있다.

### 핵심 원칙
- **효율적 탐색:** 상태 탭 + 플랫폼 필터 + 기간 필터 + 검색의 4중 필터로 빠른 콘텐츠 탐색
- **일괄 처리:** 체크박스 선택 후 일괄 삭제 기능으로 대량 관리 지원
- **상태 가시성:** 12가지 상태를 색상 태그로 직관적 표시

- **데스크톱 (>=1024px):** 전체 컬럼 표시, 필터 가로 배치

## 8. 연결 흐름 (Connected Flows)

### 8.1 목록 → 상세/수정/작성 흐름

```
[콘텐츠 목록] ──(+ 새 콘텐츠)──→ [새 콘텐츠 작성] (/contents/create)
       │
       ├──(제목 링크 클릭)──→ [콘텐츠 상세] (/contents/:id)
       │
       ├──(👁 상세 버튼)──→ [콘텐츠 상세] (/contents/:id)
       │
       ├──(✏ 수정 버튼)──→ [콘텐츠 수정] (/contents/:id/edit)
       │                    (DRAFT/REJECTED 상태만)
       │
       ├──(📤 검토 요청)──→ 상태 변경 (DRAFT → PENDING_REVIEW)
       │                    → [승인 대기 목록] (/approvals)
       │
       └──(🗑 삭제)──→ Popconfirm → 삭제 후 목록 갱신
```

### 8.2 필터링 흐름

```
[상태 탭 전환] ──→ API 재호출 (status 파라미터 변경) ──→ 목록 갱신
[플랫폼 필터 변경] ──→ API 재호출 (platform 파라미터 변경) ──→ 목록 갱신
[검색어 입력] ──→ API 재호출 (search 파라미터 변경) ──→ 목록 갱신
    ※ 모든 필터 변경 시 page=1로 초기화
```

### 8.3 일괄 삭제 흐름

```
[체크박스 선택] ──→ [일괄 삭제 버튼 표시]
       │
       └──(클릭)──→ Popconfirm "N개 항목을 삭제하시겠습니까?"
                          │
                    ┌─────┴──────┐
                    ↓            ↓
               [확인: 삭제]  [취소: 무시]
                    │
                    ↓
              POST /contents/bulk-action
                    │
              ┌─────┴──────┐
              ↓            ↓
         [성공: 토스트]  [실패: 토스트]
         + 선택 초기화   + 상태 유지
```

## 10. 데이터 타입

### 10.1 콘텐츠 레코드 (ContentRecord)

```typescript
interface ContentRecord {
  id: string;                                  // 콘텐츠 고유 ID (UUID)
  organization_id: string;                     // 소속 기관 ID
  title: string;                               // 콘텐츠 제목
  body: string | null;                         // 본문/설명문
  status: string;                              // 콘텐츠 상태 (12가지)
  platforms: string[];                         // 게시 대상 플랫폼 배열
  channel_ids: string[];                       // 채널 ID 배열
  scheduled_at: string | null;                 // 예약 게시 일시 (ISO 8601)
  author_id: string;                           // 작성자 ID
  author_name: string | null;                  // 작성자 이름 (테이블 표시용)
  platform_contents: Record<string, unknown> | null;  // 플랫폼별 맞춤 콘텐츠
  metadata: Record<string, unknown> | null;    // 추가 메타데이터
  ai_generated: boolean;                       // AI 제안 사용 여부
  media_urls: string[];                        // 미디어 파일 URL 배열
  created_at: string;                          // 작성일 (ISO 8601)
  updated_at: string;                          // 수정일 (ISO 8601)
}
```

### 10.2 페이지네이션 응답 (PaginatedResponse)

```typescript
interface PaginatedResponse<T> {
  data: T[];                 // 콘텐츠 배열
  meta: {
    total: number;           // 전체 건수
    page: number;            // 현재 페이지
    limit: number;           // 페이지당 건수
  };
}
```

### 10.3 일괄 처리 요청/응답

```typescript
// 요청
interface BulkActionRequest {
  content_ids: string[];     // 대상 콘텐츠 ID 배열
  action: string;            // 액션 종류 ("delete")
}

// 응답
interface BulkActionResponse {
  affected: number;          // 처리된 건수
}
```

| **현재 페이지** | `useState` | `page` | 페이지네이션 페이지 번호 (기본: 1) |
| **상태 탭** | `useState` | `statusTab` | 선택된 상태 필터 탭 (기본: `'all'`) |
| **플랫폼 필터** | `useState` | `platformFilter` | 선택된 플랫폼 (기본: `undefined`) |
| **기간 필터** | `useState` | `periodFilter` | 기간 필터 값 (기본: `'30d'`) |
| **검색어** | `useState` | `searchText` | 검색 입력값 (기본: `undefined`) |
| **선택된 행** | `useState` | `selectedRowKeys` | 체크박스 선택된 행 키 배열 |
| **서버 데이터** | TanStack Query | `useContents()` | 콘텐츠 목록 + 메타 (캐시 키: `['contents', params]`) |
| **삭제 뮤테이션** | TanStack Query | `useDeleteContent()` | 콘텐츠 삭제 mutation |
| **검토 요청 뮤테이션** | TanStack Query | `useRequestReview()` | 검토 요청 mutation |
| **일괄 처리 뮤테이션** | TanStack Query | `useBulkAction()` | 일괄 처리 mutation |

### 캐시 무효화

모든 mutation 성공 시 `queryClient.invalidateQueries({ queryKey: ['contents'] })`를 호출하여
콘텐츠 목록 자동 갱신.

## 13. 사용 시나리오

### 시나리오 1: 상태별 콘텐츠 필터링
1. 사이드바 > 콘텐츠 > 콘텐츠 목록 진입
2. 전체 탭에서 모든 콘텐츠 확인
3. "검토 대기" 탭 클릭 -- 검토 요청된 콘텐츠만 필터링
4. 원하는 콘텐츠 제목 클릭 -- 상세 페이지로 이동

### 시나리오 2: 플랫폼별 콘텐츠 검색
1. 플랫폼 필터에서 "Instagram" 선택
2. 검색 입력란에 "캠페인" 입력 후 Enter
3. Instagram 대상 + "캠페인" 키워드 포함 콘텐츠만 표시
4. 기간을 "최근 7일"로 변경하여 최신 콘텐츠 확인

### 시나리오 3: 초안 콘텐츠 검토 요청
1. "작성 중" 탭 선택
2. 작성 완료된 콘텐츠 행의 검토 요청 (SendOutlined) 버튼 클릭
3. "검토 요청이 완료되었습니다" 확인
4. 해당 콘텐츠 상태가 "검토 대기"로 변경됨

### 시나리오 4: 일괄 삭제
1. 테이블 좌측 체크박스로 삭제 대상 3건 선택
2. "일괄 삭제 (3)" 버튼 표시됨
3. 버튼 클릭 -- "3개 항목을 삭제하시겠습니까?" 확인 팝업
4. "삭제" 클릭 -- "3개 콘텐츠가 삭제되었습니다" 확인
5. 선택 상태 초기화 + 목록 자동 갱신

### 시나리오 5: CD(위탁기관 담당자) 조회
1. CD 역할로 로그인 후 콘텐츠 목록 진입
2. 소속 기관의 콘텐츠만 표시됨 (RLS 격리)
3. 콘텐츠 제목 클릭 -- 상세 조회 가능
4. 수정/삭제 버튼은 권한에 따라 비활성화

## 15. 구현 상세: 메시지 상수 체계

모든 사용자 피드백 메시지는 `@/shared/constants/messages.ts`의 `CONTENT_MESSAGES`에서 관리된다.
인라인 문자열 대신 상수 참조를 사용하여 일관성을 보장한다.

```typescript

export const CONTENT_MESSAGES = {
  CREATE_SUCCESS: '콘텐츠가 작성되었습니다',
  CREATE_ERROR: '콘텐츠 작성에 실패했습니다',
  UPDATE_SUCCESS: '콘텐츠가 수정되었습니다',
  UPDATE_ERROR: '콘텐츠 수정에 실패했습니다',
  SAVE_DRAFT_SUCCESS: '임시 저장이 완료되었습니다',
  SAVE_DRAFT_ERROR: '임시 저장에 실패했습니다',
  REQUEST_REVIEW_SUCCESS: '검토 요청이 완료되었습니다',
  REQUEST_REVIEW_ERROR: '검토 요청에 실패했습니다',
  DELETE_SUCCESS: '콘텐츠가 삭제되었습니다',
  DELETE_ERROR: '콘텐츠 삭제에 실패했습니다',
  DELETE_CONFIRM: '콘텐츠를 삭제하시겠습니까?',
  CANCEL_PUBLISH_SUCCESS: '게시가 취소되었습니다',
  CANCEL_PUBLISH_ERROR: '게시 취소에 실패했습니다',
  RETRY_PUBLISH_SUCCESS: '재게시가 요청되었습니다',
  RETRY_PUBLISH_ERROR: '재게시 요청에 실패했습니다',
```

## 16. 구현 상세: 안전 fallback 함수

상태/플랫폼 매핑에 `getStatusConfig()` / `getPlatformConfig()` 헬퍼를 사용한다.
미정의 값이 들어올 경우 에러 없이 기본값을 반환하여 런타임 안정성을 확보한다.

```typescript
// @/shared/constants/contentStatus.ts
export function getStatusConfig(status: string): StatusConfig {
  return CONTENT_STATUS_CONFIG[status] ?? { color: 'default', text: status };
}

// @/shared/constants/platform.ts
export function getPlatformConfig(platform: string): PlatformConfig {
  return PLATFORM_CONFIG[platform] ?? { label: platform, short: platform, color: 'default' };
}
```

## 17. 구현 상세: 전체 Hooks 목록

`useContents.ts`는 콘텐츠 목록 페이지 외에도 전체 콘텐츠 생명주기에서 사용되는 11개 Hook을 제공한다.

| Hook | 유형 | API | 사용처 |
|------|------|-----|--------|
| `useContents` | Query | `GET /contents` | 목록 조회 |
| `useContent` | Query | `GET /contents/:id` | 상세 조회 |
| `useCreateContent` | Mutation | `POST /contents` | 콘텐츠 생성 |
| `useUpdateContent` | Mutation | `PUT /contents/:id` | 콘텐츠 수정 |
| `useDeleteContent` | Mutation | `DELETE /contents/:id` | 콘텐츠 삭제 |
| `useSaveDraft` | Mutation | `POST /contents/:id/save-draft` | 임시 저장 |
| `useRequestReview` | Mutation | `POST /contents/:id/request-review` | 검토 요청 |
| `usePublishHistory` | Query | `GET /contents/:id/publish-history` | 게시 이력 조회 |
| `useRetryPublish` | Mutation | `POST /contents/:id/retry-publish` | 재게시 |
| `useBulkAction` | Mutation | `POST /contents/bulk-action` | 일괄 처리 |
| `useCancelPublish` | Mutation | `POST /contents/:id/cancel-publish` | 게시 취소 |

모든 Mutation Hook은 성공 시 `queryClient.invalidateQueries({ queryKey: ['contents'] })`를 호출하여
캐시를 자동 무효화한다.
