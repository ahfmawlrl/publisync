# 벤치마크 분석 (BenchmarkPage)

> **경로:** `/analytics/benchmark`
> **소스:** `frontend/src/features/analytics/pages/BenchmarkPage.tsx`
> **접근 권한:** AM (수탁업체 관리자), AO (수탁업체 실무자), CD (위탁기관 담당자)
> **Phase:** 4
> **관련 기능:** F23 벤치마크

---

## 1. 화면 개요

업종(공공기관) 평균 대비 우리 기관의 소셜 미디어 성과를 비교 분석하는 화면이다.
종합 점수, 플랫폼별 벤치마크 테이블(순위/조회수/참여율/백분위), 조회수 비교 바 차트,
성과 레이더 차트, 기관 간 비교 테이블을 제공한다. 기간(7일/30일/90일)에 따라 데이터가
갱신되며, 업종 평균 대비 우위/열위를 색상으로 직관적으로 표시한다.

### 핵심 원칙
- **멀티테넌트 격리:** 기관 비교 시에도 다른 기관의 상세 데이터는 노출하지 않음 (집계 수준만 표시)
- **Fallback 필수:** 벤치마크 데이터 없이도 개별 기관 성과 확인 가능 (성과 분석 화면 연결)

## 9. 연결 흐름 (Connected Flows)

### 9.1 벤치마크 분석 진입 흐름

```
[사이드바] ──→ [성과 분석] ──→ [벤치마크 분석 /analytics/benchmark]
                                     │
                                     ├──(기간 변경)──→ 전체 데이터 리패치
                                     │
                                     └──(기관 비교)──→ 다른 기관 대비 성과 확인
```

### 9.2 분석 화면 간 연결

```
[성과 분석 /analytics]
       │
       ├──→ [여론 동향 분석 /analytics/sentiment]
       │
       ├──→ [성과 예측 /analytics/prediction]
       │
       └──→ [벤치마크 분석 /analytics/benchmark] ← 현재 화면
```

### 9.3 벤치마크 활용 흐름

```
[벤치마크 확인]
       │
       ├──→ 종합 점수 확인 → 전체적인 성과 수준 파악
       │
       ├──→ 플랫폼별 순위 확인 → 약한 플랫폼 개선 전략 수립
       │
       ├──→ 업종 평균 대비 비교 → 바 차트로 직관적 파악
       │
       ├──→ 레이더 차트 → 다차원 성과 균형 확인
       │
       └──→ 기관 비교 → 동종 기관 대비 포지셔닝 확인
```

## 11. 데이터 타입

### 11.1 벤치마크 데이터 (BenchmarkData)

```typescript
interface BenchmarkData {
  industry: string;                      // 업종 ("공공기관")
  period: string;                        // 조회 기간
  platforms: BenchmarkPlatformItem[];    // 플랫폼별 벤치마크
  overall_score: number;                 // 종합 점수 (0~100)
  updated_at: string;                    // 마지막 업데이트 (ISO 8601)
}
```

### 11.2 벤치마크 플랫폼 아이템 (BenchmarkPlatformItem)

```typescript
interface BenchmarkPlatformItem {
  platform: string;               // 플랫폼 코드
  metrics: BenchmarkMetric[];     // 메트릭 배열
  rank: number;                   // 순위
  total_orgs: number;             // 전체 기관 수
}
```

### 11.3 벤치마크 메트릭 (BenchmarkMetric)

```typescript
interface BenchmarkMetric {
  metric: string;           // 메트릭 코드 (total_views, engagement_rate 등)
  org_value: number;        // 우리 기관 값
  industry_average: number; // 업종 평균 값
  percentile: number;       // 백분위 (0~100)
}
```

### 11.4 기관 비교 데이터 (OrgComparisonData)

```typescript
interface OrgComparisonData {
  period: string;                        // 조회 기간
  organizations: OrgComparisonItem[];    // 기관별 비교 데이터
}
```

### 11.5 기관 비교 아이템 (OrgComparisonItem)

```typescript
interface OrgComparisonItem {
  org_id: string;          // 기관 ID
  org_name: string;        // 기관명
  platform: string;        // 플랫폼
  followers: number;       // 팔로워
  total_views: number;     // 총 조회수
  total_likes: number;     // 총 좋아요
  engagement_rate: number; // 참여율 (%)
  post_count: number;      // 게시 수
}
```

| **기관 비교 데이터 없음** | `Empty` ("비교할 기관 데이터가 없습니다.") |

## 14. 사용 시나리오

### 시나리오 1: 종합 벤치마크 확인

1. 사이드바 > 성과 분석 > 벤치마크 분석 클릭
2. 종합 점수 카드에서 78/100 확인 (녹색 = 상위권)
3. 분석 플랫폼 5개, 업종 "공공기관" 확인
4. 플랫폼별 벤치마크 테이블에서 각 플랫폼 순위 및 업종 평균 대비 확인
5. 백분위 Progress 바로 직관적 성과 수준 파악

### 시나리오 2: 약한 플랫폼 식별

1. 벤치마크 테이블에서 참여율이 업종 평균보다 낮은(빨강 표시) 플랫폼 확인
2. 조회수 비교 바 차트에서 업종 평균(회색) 대비 내 기관(파랑) 비교
3. 레이더 차트에서 백분위가 낮은 메트릭 확인
4. 해당 플랫폼의 콘텐츠 전략 개선 방향 도출

### 시나리오 3: 기관 간 비교

1. 기관 비교 테이블로 스크롤
2. 총 조회수/좋아요/참여율/게시수 컬럼 클릭하여 정렬
3. 동종 기관 대비 우리 기관의 포지셔닝 확인
4. 상위 기관의 게시 빈도 및 참여율 참고

### 시나리오 4: 기간별 추이 비교

1. 기간 "7일" 선택 → 최근 단기 벤치마크 확인
2. 기간 "90일" 전환 → 장기 벤치마크 비교
3. 종합 점수 변화 확인
4. 플랫폼별 순위 변동 파악
