## 3. 프론트엔드 기술 스택

### 3.1 빌드·번들링

| 항목 | 선택 | 비교 후보 | 선택 근거 |
|---|---|---|---|
| 빌드 도구 | **Vite 6** | Webpack 5, Turbopack | ESM 네이티브, 콜드 스타트 < 300ms, React 19 공식 지원, HMR 즉시 반영 |
| 패키지 관리 | **pnpm** | npm, yarn | 디스크 효율(하드링크), 엄격한 의존성 관리, 모노레포 workspace 지원 |

### 3.2 상태 관리

| 항목 | 선택 | 비교 후보 | 선택 근거 |
|---|---|---|---|
| 서버 상태 | **TanStack Query v5** | SWR, RTK Query | 자동 캐싱·재시도·폴링, SSE 폴링 fallback 자연 통합, Devtools 내장 |
| 클라이언트 상태 | **Zustand 5** | Redux Toolkit, Jotai | 보일러플레이트 최소, 미들웨어(persist, devtools) 내장, 번들 크기 1.1KB |

> **역할 분리:** API 응답 데이터는 TanStack Query가 캐시하고, UI 전용 상태(사이드바 접힘, 테마, 모달 열림)만 Zustand에 저장한다.

### 3.3 UI 컴포넌트·스타일링

| 항목 | 선택 | 비교 후보 | 선택 근거 |
|---|---|---|---|
| UI 라이브러리 | **Ant Design 5** | MUI, Shadcn/ui | Enterprise 어드민 특화(Table, Form, Tree, DatePicker), 한국어 locale 기본 제공, ConfigProvider로 다크 모드 토큰 일괄 전환 |
| 유틸리티 CSS | **Tailwind CSS 4** | CSS Modules, vanilla-extract | Ant Design의 레이아웃 빈틈(간격, 정렬, 반응형)을 유틸리티 클래스로 보완, JIT로 미사용 CSS 제거 |
| 아이콘 | **Lucide React** | Ant Design Icons, Heroicons | 트리셰이킹, 일관된 스트로크 스타일, 200+ 아이콘 |

> **테마 전환:** Ant Design 5의 `ConfigProvider` + `theme.algorithm`을 사용하여 light/dark 모드를 CSS 변수 없이 런타임 전환한다. Tailwind는 `darkMode: 'selector'` 설정으로 Ant Design의 다크 모드와 동기화한다.

### 5.3 Redis 키 설계 (개요)

### 5.4 저장소 암호화 전략 (Security at Rest)

| 저장소 | 암호화 방식 | 설명 |
|---|---|---|
| **PostgreSQL** | 볼륨 암호화 (LUKS/dm-crypt) | 디스크 레벨 암호화, OS 계층에서 투명 처리. 컬럼 레벨 암호화는 OAuth 토큰 등 민감 필드에 `cryptography (Fernet)` 적용 |
| **Redis** | TLS 전송 암호화 + AOF 볼륨 암호화 | `requirepass` 인증 필수, 인메모리 데이터 특성상 전송 구간 암호화에 집중 |
| **MinIO** | **SSE-S3** (Server-Side Encryption) | MinIO 내장 AES-256 암호화, 버킷 정책으로 자동 적용, 키 관리는 MinIO KES 사용 |

> **원칙:** 전송 구간(TLS 1.3)은 전 구간 필수, 저장 구간은 민감도에 따라 계층별 차등 적용. OAuth 토큰·개인정보 등 고민감 데이터는 애플리케이션 레벨 암호화를 추가 적용한다.

### 6.4 백업·복구 전략

| 대상 | 백업 방식 | 주기 | 보존 기간 | 복구 목표 |
|---|---|---|---|---|
| **PostgreSQL** | `pg_dump` 전체 백업 + WAL 아카이빙 (PITR) | 일 1회 전체 + 연속 WAL | 전체: 30일, WAL: 7일 | RPO < 1시간, RTO < 2시간 |
| **Redis** | RDB 스냅샷 + AOF | RDB 6시간, AOF 매초 | 최근 3일 | RPO < 1분 (AOF), RTO < 10분 |
| **MinIO** | Erasure Coding + mc mirror (원격 복제) | 실시간 (EC) + 일 1회 (미러) | 원본과 동일 | RPO ≈ 0 (EC), RTO < 1시간 |

> **복구 테스트:** 분기 1회 복구 드릴을 실시하여 백업 유효성을 검증한다. `pg_restore` 및 MinIO 미러 복원 절차를 런북으로 문서화한다.

### 6.5 HA 아키텍처 방향 (Phase 2+)

Phase 1에서는 단일 인스턴스로 운영하되, Phase 2 이후 트래픽 증가에 대비한 HA 전환 방향을 사전 정의한다.

| 구성 요소 | Phase 1 (단일) | Phase 2+ (HA) |
|---|---|---|
| **PostgreSQL** | 단일 인스턴스 | Streaming Replication (1 Primary + 1 Standby), 읽기 분산 |
| **Redis** | 단일 인스턴스 | Redis Sentinel (1 Master + 2 Replica + 3 Sentinel) |
| **API 서버** | uvicorn 4 workers | Docker Swarm / K8s, 수평 스케일링 (Nginx upstream) |
| **Celery** | 단일 워커 | 작업 유형별 큐 분리 (ai, publish, report), 워커 수평 확장 |
| **MinIO** | 단일 노드 (Erasure Coding) | 분산 모드 (4노드 이상) |

> **설계 원칙:** Phase 1 코드에서 DB 연결 문자열, Redis URL 등을 환경 변수로 관리하여 Phase 2 전환 시 코드 변경 없이 인프라만 교체할 수 있도록 한다.

## 10. 기술 리스크 및 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| Ant Design 번들 크기 (1MB+) | 초기 로딩 지연 | 트리셰이킹 + dynamic import, 비핵심 컴포넌트 lazy load |
| Celery 메모리 누수 (장기 워커) | 워커 장애 | `--max-tasks-per-child=100` 설정, Flower로 모니터링 |
| ffmpeg 컨테이너 용량 | Docker 이미지 비대 | 멀티스테이지 빌드, ffmpeg-static 바이너리 사용 |
| WeasyPrint 한글 폰트 | PDF 깨짐 | Docker 이미지에 Noto Sans KR 폰트 사전 설치 |
| MinIO 단일 장애점 | 미디어 접근 불가 | Erasure Coding 활성화, 6.4절 백업 전략 참조 |
| litellm 제공자 API 변경 | AI 기능 장애 | litellm 버전 고정, 제공자별 fallback 체인 설정 |
| PostgreSQL 풀텍스트 검색 한계 | 한국어 형태소 분석 부족 | Phase 2에서 Meilisearch 전환, pg_trgm 임시 보완 |
| MinIO AGPL-3.0 라이선스 | 상용 배포 시 소스 공개 의무 발생 가능 | MinIO를 독립 서비스(Docker)로 실행하면 AGPL 전파 없음. S3 호환 API만 사용하므로 클라우드 전환(AWS S3) 시 코드 변경 최소 |
| 개인정보 보호 (공공기관) | 개인정보 유출 시 법적 리스크 | 로그·에러 리포트에 개인정보 마스킹 적용 (structlog 프로세서에서 이메일·전화번호 패턴 자동 치환), Sentry `before_send` 훅으로 민감 필드 제거 |
| Service Worker 브라우저 호환성 | iOS Safari 제한적 지원 | Web Push가 불가한 환경에서는 SSE(기존 알림) + 이메일 폴백으로 커버, `Notification.permission` 사전 체크 |

## 11. 보안·컴플라이언스 체크리스트

공공기관 데이터를 다루는 SaaS 특성상, 보안은 아키텍처 레벨에서 내재화한다.

| 영역 | 적용 기술 | 세부 사항 |
|---|---|---|
| **전송 구간 암호화** | TLS 1.3 (Nginx) | 전 구간 HTTPS 강제, HSTS 헤더 적용 |
| **저장 구간 암호화** | LUKS + MinIO SSE-S3 + Fernet | 5.4절 참조 |
| **OAuth 토큰 보호** | cryptography (Fernet) | 플랫폼 refresh_token DB 저장 시 AES 대칭 암호화, 키 환경 변수 관리 |
| **개인정보 마스킹** | structlog 프로세서 | 로그 출력 시 이메일(`***@***.com`), 전화번호(`***-****-****`) 자동 마스킹 |
| **에러 추적 필터링** | Sentry `before_send` | 요청 바디·헤더에서 `password`, `token`, `authorization` 필드 자동 제거 |
| **접근 제어** | RLS (PostgreSQL) | `organization_id` 기반 행 수준 보안, 멀티테넌트 데이터 격리 |
| **감사 추적** | 감사 로그 (F14) | 월별 파티션, 사용자·IP·변경 내역 기록, 90일 보존 |
| **의존성 취약점** | GitHub Dependabot + pip-audit | PR 자동 생성, CI에서 알려진 취약점 차단 |
