## 1. 시작하기 — 프로그램 실행

### 1-1. 사전 요구사항

| 항목 | 필요 버전 |
|---|---|
| Docker Desktop | 4.x 이상 |
| Node.js | 18 이상 |
| pnpm | 9.x 이상 |
| Python | 3.11 이상 |

### 1-2. 인프라 서비스 실행

프로젝트 루트(`publisync/`) 디렉터리에서 Docker 컨테이너를 시작합니다.

```bash
# 전체 인프라 서비스 실행 (PostgreSQL, Redis, MinIO, Meilisearch, API, Celery 등)
docker compose up -d
```

실행되는 서비스 목록:

| 서비스 | 포트 | 설명 |
|---|---|---|
| PostgreSQL | 5433 | 데이터베이스 |
| Redis | 6379 | 캐시/브로커 |
| MinIO | 9000 (API), 9001 (콘솔) | 파일 저장소 |
| Meilisearch | 7700 | 검색 엔진 |
| API (FastAPI) | 8000 | 백엔드 API 서버 |
| Nginx | 80 | 리버스 프록시 |
| Celery Worker | — | 비동기 작업 처리 |
| Celery Beat | — | 스케줄 작업 |
| Prometheus | 9090 | 메트릭 수집 |
| Grafana | 3001 | 모니터링 대시보드 |
| Loki | 3100 | 로그 수집 |

### 1-3. 서비스 상태 확인

```bash
# 모든 컨테이너 상태 확인
docker compose ps

# 정상 상태: 모든 서비스가 "running" 또는 "healthy" 표시
```

### 1-4. 데이터베이스 초기화 (최초 1회)

```bash
# API 컨테이너 내에서 마이그레이션 실행
docker compose exec api alembic upgrade head
```

> **참고:** 테스트 계정 및 기관 데이터는 Docker 볼륨에 이미 포함되어 있습니다.
> 만약 볼륨을 삭제(`docker compose down -v`)한 경우, DB 시드 데이터를 수동으로 재삽입해야 합니다.

### 1-5. 프론트엔드 개발 서버 실행

```bash
cd frontend
pnpm install     # 최초 1회 또는 의존성 변경 시
pnpm dev         # Vite 개발 서버 → http://localhost:5173
```

### 1-6. 접속 URL

| 환경 | URL | 설명 |
|---|---|---|
| **개발 모드** | http://localhost:5173 | Vite 개발 서버 (HMR 지원) |
| **프로덕션 모드** | http://localhost:80 | Nginx → 정적 파일 + API 프록시 |
| API 문서 | http://localhost:8000/docs | FastAPI Swagger UI |
| MinIO 콘솔 | http://localhost:9001 | 파일 저장소 관리 (minioadmin/minioadmin) |
| Grafana | http://localhost:3001 | 모니터링 (admin/admin) |

## 2. 로그인

### 2-1. 로그인 화면 접속

브라우저에서 `http://localhost:5173`에 접속하면 로그인 화면이 표시됩니다.

```
┌─────────────────────────────────┐
│                                 │
│          PubliSync              │
│   공공기관 소셜 미디어 통합 관리    │
│                                 │
│   ┌───────────────────────┐     │
│   │ ✉ 이메일              │     │
│   └───────────────────────┘     │
│   ┌───────────────────────┐     │
│   │ 🔒 비밀번호            │     │
│   └───────────────────────┘     │
│                                 │
│   ☐ 로그인 유지   비밀번호 찾기   │
│                                 │
│   ┌───────────────────────┐     │
│   │       로그인           │     │
│   └───────────────────────┘     │
│                                 │
└─────────────────────────────────┘
```

### 2-2. 테스트 계정

| 역할 | 이메일 | 비밀번호 | 설명 |
|---|---|---|---|
| **시스템 관리자 (SA)** | admin@publisync.kr | Admin1234! | 전체 시스템 관리 |
| **수탁업체 관리자 (AM)** | manager@digitalsotong.kr | Manager1234! | 기관 설정·리포트·사용자 관리 |
| **수탁업체 실무자 (AO)** | operator@digitalsotong.kr | Operator1234! | 콘텐츠 작성·댓글 대응 |
| **위탁기관 담당자 (CD)** | director@seoul.go.kr | Director1234! | 승인·대시보드 조회 |

### 2-3. 로그인 절차

1. **이메일** 입력란에 테스트 계정 이메일을 입력합니다.
2. **비밀번호** 입력란에 비밀번호를 입력합니다.
3. 필요 시 **"로그인 유지"** 체크박스를 선택합니다.
4. **"로그인"** 버튼을 클릭합니다.

> **참고:** 로그인 5회 실패 시 계정이 30분간 잠김니다 (423 Locked).

### 2-4. 추천 테스트 순서

전체 기능을 경험하려면 **수탁업체 관리자(AM)** 계정으로 먼저 로그인하는 것을 권장합니다.
AM은 콘텐츠 작성·채널 관리·승인·리포트 등 대부분의 기능에 접근 가능합니다.

## 3. 워크스페이스 선택

로그인 후 상단 바(TopBar)에서 **워크스페이스(기관)**를 선택해야 합니다.

```
┌──────────────────────────────────────────────────────────┐
│ [워크스페이스 선택 ▾]  [🔍 검색]          🌙  🔔  👤 이름 │
└──────────────────────────────────────────────────────────┘
```

### 3-1. 워크스페이스 전환 방법

1. 상단 왼쪽의 **"워크스페이스 선택"** 드롭다운을 클릭합니다.
2. 소속된 기관 목록이 표시됩니다:
   - **서울특별시 디지털정책과**
   - **부산광역시 홍보담당관실**
3. 원하는 기관을 선택하면 해당 기관의 데이터로 전환됩니다.

### 3-2. 테스트 기관 정보

| 기관명 | 역할별 접근 |
|---|---|
| 서울특별시 디지털정책과 | SA, AM, AO, CD 모두 접근 가능 |
| 부산광역시 홍보담당관실 | SA, AM만 접근 가능 |

> **중요:** 워크스페이스를 선택하지 않으면 대부분의 기능이 동작하지 않습니다.
> 멀티테넌트 격리(RLS)에 의해 각 기관의 데이터는 완전히 분리됩니다.

### 3-3. 상단 바 기능

| 요소 | 설명 |
|---|---|
| 워크스페이스 드롭다운 | 기관 전환 |
| 검색 바 | 통합 검색 (콘텐츠 제목, 댓글 등) |
| 🌙/☀ 버튼 | 다크 모드/라이트 모드 전환 |
| 🔔 알림 벨 | 알림 센터 바로가기 |
| 프로필 아바타 | 이름·이메일 표시, 로그아웃 |

## 4. 대시보드

**경로:** `/` (메인 화면)
**접근 역할:** SA, AM, AO, CD (전체)

로그인 후 가장 먼저 보이는 화면입니다. 선택한 기관의 핵심 지표를 한눈에 파악합니다.

### 4-1. 화면 구성

```
┌─────────────────────────────────────────────────────────────┐
│ 대시보드                          [최근 7일 ▾] [최근 30일 ▾] │
├──────────┬──────────┬──────────┬──────────┐                 │
│ 총 도달    │ 총 게시물  │ 총 댓글    │⚠ 위험 댓글│                 │
│ 45,231   │    127   │   892    │    12    │                 │
│ +2.1%    │ +15건    │ +8.3%   │ 즉시확인 │ ← 클릭 가능      │
└──────────┴──────────┴──────────┴──────────┘                 │
│                                                             │
│  ┌─── 댓글 감성 분석 ───┐  ┌──── 플랫폼별 성과 ────────┐    │
│  │     (도넛 차트)       │  │     (막대 차트)            │    │
│  │  긍정 40%  중립 35%  │  │  조회수 / 좋아요 / 공유    │    │
│  │  부정 20%  위험 5%   │  │  YouTube / Insta / FB ...  │    │
│  └──────────────────────┘  └────────────────────────────┘    │
│                                                             │
│  ┌─ 최근 콘텐츠 ───────┐  ┌─ 오늘 일정 ───────────────┐    │
│  │ 제목 | 상태 | 플랫폼 │  │ 09:00 YT 정책브리핑 ✅    │    │
│  └─────────────────────┘  │ 14:00 IG 봄축제 ⏳         │    │
│                            └───────────────────────────┘    │
│  ┌─ 승인 현황 ─────────────────────────────────────────┐    │
│  │ 승인 대기: 3건  검토 중: 1건  승인됨: 5건  반려: 1건  │    │
│  │                         [승인 대기 목록 보기 →]      │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                             │
│  데이터 최신성: YouTube 2분 전 · Instagram 5분 전 · ...     │
└─────────────────────────────────────────────────────────────┘
```

### 4-2. KPI 카드 (4개)

| 카드 | 설명 |
|---|---|
| **총 도달** | 모든 플랫폼의 합산 조회수 (API `views_growth` 값 표시) |
| **총 게시물** | 해당 기간 내 게시된 콘텐츠 수 (API `contents_growth` 값 표시) |
| **총 댓글** | 모든 플랫폼의 합산 댓글 수 (API `comments_growth` 값 표시) |
| **⚠ 위험 댓글** | 위험으로 분류된 미처리 댓글 수 (빨간색, 클릭 시 `/comments/dangerous`로 이동) |

### 4-3. AM/SA 역할: 기관별 현황 비교

AM 또는 SA 역할로 로그인하면, KPI 카드 아래에 **기관별 현황 비교 테이블**이 표시됩니다.
담당하는 모든 기관의 핵심 지표를 한눈에 비교할 수 있습니다.

| 컬럼 | 설명 |
|---|---|
| **기관명** | 기관 이름 |
| **전체 콘텐츠** | 해당 기관의 총 콘텐츠 수 |
| **게시 완료** | 게시 완료(PUBLISHED) 건수 |
| **활성 채널** | 연동된 활성 채널 수 |
| **대기 승인** | 검수 대기 중인 콘텐츠 수 |

### 4-4. 기간 선택

상단 우측에서 **최근 7일 / 최근 30일** 기간을 선택하여 KPI 및 차트 데이터를 필터링합니다.

### 4-5. 차트

- **댓글 감성 분석 (도넛 차트):** 수집된 댓글의 감성 분포 (긍정/중립/부정/위험)
- **플랫폼별 성과 (막대 차트):** 각 플랫폼(YouTube, Instagram 등)별 조회수·좋아요·공유 비교

### 4-6. 하단 리스트

- **최근 콘텐츠:** 최근 작성/게시된 콘텐츠 목록과 상태 태그, 플랫폼 약어(YT·IG·FB) 표시
- **오늘 일정:** 당일 예약 게시 예정인 콘텐츠를 시간순으로 표시 (플랫폼 태그 + 상태 아이콘)
- **승인 현황:** 검토 대기/검토 중/승인됨/반려됨 건수 요약 + **"승인 대기 목록 보기 →"** 링크

### 4-7. 데이터 최신성 표시

화면 하단에 각 플랫폼별 데이터의 마지막 동기화 시간이 표시됩니다 (예: "YouTube 2분 전 · Instagram 5분 전").

## 5. 콘텐츠 관리

**경로:** `/contents`
**접근 역할:** AM, AO

### 5-1. 콘텐츠 목록

```
┌───────────────────────────────────────────────────────────┐
│ 콘텐츠 목록                        [+ 콘텐츠 작성]        │
├───────────────────────────────────────────────────────────┤
│ [전체] [작성 중] [검수 대기] [승인됨] [게시 완료] [반려]   │  ← 탭 바
├───────────────────────────────────────────────────────────┤
│ [플랫폼 ▾]  [기간 ▾]  [🔍 제목 검색]         [일괄 삭제]  │
├───────────────────────────────────────────────────────────┤
│ ☐ │ 상태     │ 제목          │ 플랫폼   │ 작성자 │ 날짜   │ 관리│
│───┼──────────┼───────────────┼──────────┼────────┼────────┼─────│
│ ☐ │ 초안     │ 시민참여 안내  │ YT       │ 박실무 │ 3/5    │👁✏📤🗑│
│ ☐ │ 게시완료 │ 봄축제 홍보   │ IG·FB    │ 이매니 │ 3/4    │👁✏  │
│ ☐ │ 검수대기 │ 교통안전 캠페인│ X·Blog   │ 박실무 │ 3/3    │👁   │
├───────────────────────────────────────────────────────────┤
│ 선택: 2건                                       1 / 5 페이지│
└───────────────────────────────────────────────────────────┘
```

> **RBAC:** 위탁기관 담당자(CD) 역할에서는 **"+ 콘텐츠 작성"** 버튼이 표시되지 않습니다. CD는 콘텐츠 조회·승인만 가능합니다.

**상태 탭 바:** 상단 탭을 클릭하여 상태별 필터링 (기존 드롭다운 방식 → 탭 방식으로 변경)
- **전체** | **작성 중** (DRAFT) | **검수 대기** (PENDING_REVIEW) | **승인됨** (APPROVED) | **게시 완료** (PUBLISHED) | **반려** (REJECTED)

**테이블 컬럼 순서:** 상태 → 제목 → 플랫폼 → 작성자 → 날짜 → 관리

**플랫폼 약어 표시:** YouTube → **YT**, Instagram → **IG**, Facebook → **FB**, X → **X**, 네이버 블로그 → **Blog** (복수 플랫폼은 `·`로 구분)

**추가 필터:**
- **플랫폼:** YouTube, Instagram, Facebook, X, 네이버 블로그
- **기간:** 최근 7일 / 최근 30일 / 최근 90일 / 전체

**행 액션 버튼:**
| 아이콘 | 기능 | 조건 |
|---|---|---|
| 👁 상세 | 콘텐츠 상세 보기 | 항상 |
| ✏ 수정 | 콘텐츠 수정 | 초안/반려됨 상태만 |
| 📤 검토 요청 | 승인 요청 전송 | 초안 상태만 |
| 🗑 삭제 | 콘텐츠 삭제 | 확인 팝업 표시 |

**일괄 작업:** 체크박스로 여러 콘텐츠를 선택 후 "일괄 삭제" 가능. 하단에 선택 건수 표시.

### 5-2. 콘텐츠 작성

**경로:** `/contents/create`

콘텐츠 목록에서 **"+ 콘텐츠 작성"** 버튼을 클릭합니다.

```
┌─────────── 새 콘텐츠 작성 ──────────────────────────────────┐
│ [← 뒤로]  새 콘텐츠 작성            [임시 저장] [검토 요청]  │
├─────────────────────────────────────────────────────────────┤

#### 작성 절차

1. **게시 플랫폼** 상단에서 선택 (복수 선택 가능, 체크박스)
2. **미디어 업로드** — 이미지·영상·문서 파일 업로드 (최대 10개, 드래그 앤 드롭)
3. **제목** 입력 (필수, 500자 이내)
4. **본문** 작성 (자유 텍스트)
5. **해시태그** 입력 — 태그 형식 입력 (스페이스 또는 쉼표로 구분, AI 추천 가능)
6. **예약 게시일시** — 특정 날짜/시간에 자동 게시하려면 설정
7. **"저장 (초안)"** 클릭 → 초안 상태로 저장

#### 오른쪽 패널 구성

화면 오른쪽에는 3개의 카드가 순서대로 배치됩니다:

**① 플랫폼별 미리보기** — 선택한 플랫폼(YouTube / Instagram / Facebook / X / 네이버 블로그)에 맞는 콘텐츠 미리보기를 탭으로 전환하며 확인합니다. 제목·본문·해시태그가 실시간으로 반영됩니다.

**② AI 어시스턴트** — 본문을 10자 이상 작성한 뒤 AI 기능을 사용할 수 있습니다.

| 버튼 | 기능 | 사용 방법 |
|---|---|---|
| **🤖 제목 제안 받기** | AI가 본문을 분석하여 3개의 제목 후보를 생성 | 원하는 제목 클릭 → 자동 적용 |
| **🤖 설명문 제안 받기** | AI가 본문 기반 2개의 설명문 생성 | 원하는 설명문 클릭 → 본문 끝에 추가 |
| **🤖 해시태그 제안 받기** | AI가 관련 해시태그 5개 생성 | 원하는 해시태그 클릭 → 기존 태그에 자동 추가 |
| **🔄 AI 톤 변환** | 본문을 다른 톤/플랫폼에 맞게 변환 | 플랫폼+톤 선택 후 "적용하기" |
| **✅ AI 콘텐츠 검수** | 맞춤법, 민감 표현, 편향성 검사 | 발견된 이슈와 수정 제안 표시 |

**③ 영상 콘텐츠 추가 기능** — 자막 자동 생성(F03), 숏폼 생성(F15), 효과음/이모지 추천 버튼이 표시됩니다 (Phase 2에서 활성화).

#### 워크플로우 진행바

화면 하단에 현재 콘텐츠의 상태 흐름이 표시됩니다:
```
● 작성 중 → ○ 검토 요청 → ○ 검수 중 → ○ 게시
```
현재 단계가 파란색 원(●)으로, 미래 단계가 빈 원(○)으로 표시됩니다.

#### 상단 액션 버튼

- **임시 저장** — 현재 작성 중인 내용을 초안으로 저장
- **검토 요청** — 승인 워크플로우 시작 (검수 대기 상태로 전환)

> **Human-in-the-Loop:** 모든 AI 제안은 사용자가 확인 후 "적용하기"를 클릭해야만 반영됩니다.

### 5-3. 콘텐츠 상세/수정

- **상세 보기** (`/contents/:id`): 콘텐츠 전문, 상태 이력, 게시 결과 확인
- **수정** (`/contents/:id/edit`): 초안 또는 반려된 콘텐츠만 수정 가능

### 5-4. 검토 요청 (승인 프로세스 시작)

콘텐츠 목록에서 초안 상태의 콘텐츠 행에 있는 **📤 (검토 요청)** 아이콘을 클릭하면 승인 워크플로우가 시작됩니다.

콘텐츠 상태 전이:
```
초안(DRAFT) → 검토 대기(PENDING_REVIEW) → 검토 중(IN_REVIEW) → 승인됨(APPROVED) → 예약/게시
                                                              ↘ 반려됨(REJECTED) → 수정 후 재요청
```

### 6-3. 승인/반려 절차

1. **카드 목록**에서 검토할 콘텐츠의 **"상세 보기"** 클릭
2. **분할 패널** 왼쪽에서 콘텐츠 내용을, 오른쪽에서 AI 검수 결과와 승인 이력을 확인
3. 오른쪽 하단에 **의견**을 입력 (선택사항)
4. **✅ 승인** — 콘텐츠를 승인합니다
5. **❌ 반려** — 반려 사유를 입력하여 반려합니다

### 7-2. AI 답글 생성

댓글 상세 패널에서 **🤖 AI 답글** 버튼을 클릭하면:
1. AI가 댓글의 맥락을 분석합니다
2. 적절한 답글 후보 2~3개를 제안합니다
3. 원하는 답글을 선택하여 편집 후 전송할 수 있습니다

### 7-3. 답글 템플릿 AI 개선

답글 템플릿 관리 화면(`/comments/templates`)에서 각 템플릿의 **"🤖 AI 개선"** 버튼을 클릭하면, AI가 템플릿 내용을 분석하여 개선된 버전을 제안합니다. 제안된 내용을 검토 후 적용하면 편집 모드로 전환되어 저장할 수 있습니다.

3. **"AI 생성"** 클릭 → 비동기 처리 시작 (202 Accepted)
4. 생성 진행 상태가 표시됨 (AiJob 폴링)
5. 완료 시 리포트 목록에 "초안" 상태로 등록

### 15-3. 리포트 구성 섹션

AI가 생성하는 리포트에는 다음 섹션이 포함됩니다:
- **요약 (Summary):** 전체 기간 성과 요약
- **플랫폼별 성과 (Platform Performance):** 각 플랫폼의 주요 지표
- **상위 콘텐츠 (Top Contents):** 가장 성과 좋은 콘텐츠 TOP 5
- **댓글 분석 (Comment Analysis):** 감성 분포 및 주요 키워드
- **AI 제안 (AI Suggestions):** 개선 방향 제안

### 15-4. 리포트 편집 및 확정

- **편집 (✏):** 초안 상태의 리포트만 수정 가능 (Tiptap 에디터)
- **확정 (✅):** 리포트를 최종 확정 → 더 이상 수정 불가
- **PDF 다운로드 (📥):** 리포트를 PDF 파일로 다운로드 (WeasyPrint 생성)

## 17. 감사 로그

**경로:** `/audit-logs`
**접근 역할:** SA, AM

### 17-1. 감사 로그 화면

```
┌───────────────────────────────────────────────────────────┐
│ 감사 로그                                                  │
├───────────────────────────────────────────────────────────┤
│ [작업유형 ▾] [리소스 ▾] [📅 기간] [🔍 검색]  [📥 내보내기]│

│ 시간              │ 작업    │ 처리자    │ 리소스    │ 상세│

│ 3/5 10:30:15     │ CREATE  │ 이매니저   │ 콘텐츠    │ 👁 │  ← 행 클릭 시 상세 모달

- **CSV 내보내기:** 감사 로그를 CSV 파일로 다운로드

### 18-2. 사용자 초대

1. **"+ 사용자 초대"** 버튼 클릭

   - **이메일:** 초대할 사용자의 이메일

   - **기관:** 할당할 기관 선택

- **역할 변경:** AM / AO / CD 중 선택
- **상태 변경:** 활성(ACTIVE) / 비활성(INACTIVE) / 잠금(LOCKED) 중 선택 — 잠긴 계정 해제 시 활용

### 유용한 팁

4. **AI 기능:** 모든 AI 기능은 "제안"이며 자동 적용되지 않습니다. 반드시 사용자가 확인 후 적용합니다.
5. **데이터 격리:** 워크스페이스(기관) 간 데이터는 PostgreSQL RLS로 완전히 분리되어 있어, 다른 기관의 데이터에 접근할 수 없습니다.

## 문제 해결

### 자주 발생하는 문제

| 증상 | 원인 | 해결 |
|---|---|---|
| 로그인 후 빈 화면 | 워크스페이스 미선택 | 상단 드롭다운에서 기관 선택 |
| API 요청 403 에러 | 역할 권한 부족 | 해당 기능의 접근 역할 확인 |
| 콘텐츠 수정 불가 | 초안/반려 상태가 아님 | 현재 상태가 DRAFT 또는 REJECTED인지 확인 |
| 파일 업로드 실패 | MinIO 미실행 | `docker compose ps`로 minio 상태 확인 |
| 알림 미수신 | SSE 연결 끊김 | 페이지 새로고침으로 SSE 재연결 |
| AI 기능 에러 | AI API 키 미설정 | `.env` 파일의 AI 관련 환경변수 확인 |
| 계정 잠금 | 5회 로그인 실패 | 30분 대기 후 재시도 |

### 서비스 재시작

```bash
# 전체 재시작
docker compose down && docker compose up -d

# 특정 서비스만 재시작
docker compose restart api
docker compose restart nginx
docker compose restart celery-worker
```
