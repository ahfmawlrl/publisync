## 1. 시작하기 — 프로그램 실행

### 1-1. 사전 요구사항

| 항목 | 필요 버전 |
|---|---|
| Docker Desktop | 4.x 이상 |
| Node.js | 18 이상 |
| pnpm | 9.x 이상 |
| Python | 3.11 이상 |

### 1-2. 인프라 서비스 실행

프로젝트 루트(`publisync/`) 디렉터리에서 Docker 컨테이너를 시작합니다.

```bash
# 전체 인프라 서비스 실행 (PostgreSQL, Redis, MinIO, Meilisearch, API, Celery 등)
docker compose up -d
```

실행되는 서비스 목록:

| 서비스 | 포트 | 설명 |
|---|---|---|
| PostgreSQL | 5433 | 데이터베이스 |
| Redis | 6379 | 캐시/브로커 |
| MinIO | 9000 (API), 9001 (콘솔) | 파일 저장소 |
| Meilisearch | 7700 | 검색 엔진 |
| API (FastAPI) | 8000 | 백엔드 API 서버 |
| Nginx | 80 | 리버스 프록시 |
| Celery Worker | — | 비동기 작업 처리 |
| Celery Beat | — | 스케줄 작업 |
| Prometheus | 9090 | 메트릭 수집 |
| Grafana | 3001 | 모니터링 대시보드 |
| Loki | 3100 | 로그 수집 |

### 1-3. 서비스 상태 확인

```bash
# 모든 컨테이너 상태 확인
docker compose ps

# 정상 상태: 모든 서비스가 "running" 또는 "healthy" 표시
```

### 1-4. 데이터베이스 초기화 (최초 1회)

```bash
# 1) DB 마이그레이션 실행
docker compose exec api alembic upgrade head

# 2) 테스트 계정 및 기관 시드 데이터 생성
cd backend
python -m scripts.seed
```

시드 스크립트가 생성하는 데이터:
- **수탁업체:** 디지털소통
- **기관 2개:** 서울특별시 디지털정책과, 부산광역시 홍보담당관실
- **테스트 계정 4개:** SA / AM / AO / CD (2-2. 테스트 계정 참조)

> **참고:** 시드 데이터를 재생성하려면 `python -m scripts.seed --reset`을 실행하세요.
> 볼륨을 삭제(`docker compose down -v`)한 경우, 마이그레이션과 시드를 다시 실행해야 합니다.

### 1-5. 로컬 개발 환경 실행 (Docker 없이)

Docker 없이 로컬에서 직접 실행할 수도 있습니다.

```bash
# 1) 인프라만 Docker로 실행
docker compose up -d postgres redis minio

# 2) 백엔드 의존성 설치 (.venv 활성화 상태)
cd backend
pip install -r requirements.txt

# 3) DB 마이그레이션 + 시드
alembic upgrade head
python -m scripts.seed

# 4) 백엔드 서버 실행
uvicorn app.main:app --reload --port 8000

# 5) (별도 터미널) 프론트엔드 실행
cd frontend
pnpm install
pnpm dev
```

### 1-6. 프론트엔드 개발 서버 실행

```bash
cd frontend
pnpm install     # 최초 1회 또는 의존성 변경 시
pnpm dev         # Vite 개발 서버 → http://localhost:5173
```

### 1-7. 접속 URL

| 환경 | URL | 설명 |
|---|---|---|
| **개발 모드** | http://localhost:5173 | Vite 개발 서버 (HMR 지원) |
| **프로덕션 모드** | http://localhost:80 | Nginx → 정적 파일 + API 프록시 |
| API 문서 | http://localhost:8000/docs | FastAPI Swagger UI |
| MinIO 콘솔 | http://localhost:9001 | 파일 저장소 관리 (minioadmin/minioadmin) |
| Grafana | http://localhost:3001 | 모니터링 (admin/admin) |

## 2. 로그인

### 2-1. 로그인 화면 접속

브라우저에서 `http://localhost:5173`에 접속하면 로그인 화면이 표시됩니다.

```
┌─────────────────────────────────┐
│                                 │
│          PubliSync              │
│   공공기관 소셜 미디어 통합 관리    │
│                                 │
│   ┌───────────────────────┐     │
│   │ ✉ 이메일              │     │
│   └───────────────────────┘     │
│   ┌───────────────────────┐     │
│   │ 🔒 비밀번호            │     │
│   └───────────────────────┘     │
│                                 │
│   ☐ 로그인 유지   비밀번호 찾기   │
│                                 │
│   ┌───────────────────────┐     │
│   │       로그인           │     │
│   └───────────────────────┘     │
│                                 │
└─────────────────────────────────┘
```

### 2-2. 테스트 계정

| 역할 | 이메일 | 비밀번호 | 설명 |
|---|---|---|---|
| **시스템 관리자 (SA)** | admin@publisync.kr | demo1234! | 전체 시스템 관리 |
| **수탁업체 관리자 (AM)** | manager@digitalsotong.kr | demo1234! | 기관 설정·리포트·사용자 관리 |
| **수탁업체 실무자 (AO)** | operator@digitalsotong.kr | demo1234! | 콘텐츠 작성·댓글 대응 |
| **위탁기관 담당자 (CD)** | director@seoul.go.kr | demo1234! | 승인·대시보드 조회 |

> **주의:** 위 테스트 계정은 시드 스크립트(`python -m scripts.seed`)를 실행해야 생성됩니다.
> DB 마이그레이션만 실행하면 SA 계정만 존재하며 비밀번호가 `admin123!`입니다.
> 시드 스크립트를 실행하면 모든 계정의 비밀번호가 `demo1234!`로 통일됩니다.

### 2-3. 로그인 절차

1. **이메일** 입력란에 테스트 계정 이메일을 입력합니다.
2. **비밀번호** 입력란에 비밀번호를 입력합니다.
3. 필요 시 **"로그인 유지"** 체크박스를 선택합니다.
4. **"로그인"** 버튼을 클릭합니다.

> **참고:** 로그인 5회 실패 시 계정이 30분간 잠김니다 (423 Locked).

### 2-4. 추천 테스트 순서

전체 기능을 경험하려면 **수탁업체 관리자(AM)** 계정으로 먼저 로그인하는 것을 권장합니다.
AM은 콘텐츠 작성·채널 관리·승인·리포트 등 대부분의 기능에 접근 가능합니다.

## 3. 워크스페이스 선택

로그인 후 상단 바(TopBar)에서 **워크스페이스(기관)**를 선택해야 합니다.

```
┌──────────────────────────────────────────────────────────┐
│ [워크스페이스 선택 ▾]  [🔍 검색]          🌙  🔔  👤 이름 │
└──────────────────────────────────────────────────────────┘
```

### 3-1. 워크스페이스 전환 방법

1. 상단 왼쪽의 **"워크스페이스 선택"** 드롭다운을 클릭합니다.
2. 소속된 기관 목록이 표시됩니다:
   - **서울특별시 디지털정책과**
   - **부산광역시 홍보담당관실**
3. 원하는 기관을 선택하면 해당 기관의 데이터로 전환됩니다.

### 3-2. 테스트 기관 정보

| 기관명 | 역할별 접근 |
|---|---|
| 서울특별시 디지털정책과 | SA, AM, AO, CD 모두 접근 가능 |
| 부산광역시 홍보담당관실 | SA, AM만 접근 가능 |

> **중요:** 워크스페이스를 선택하지 않으면 대부분의 기능이 동작하지 않습니다.
> 멀티테넌트 격리(RLS)에 의해 각 기관의 데이터는 완전히 분리됩니다.

### 3-3. 상단 바 기능

| 요소 | 설명 |
|---|---|
| 워크스페이스 드롭다운 | 기관 전환 |
| 검색 바 | 통합 검색 (콘텐츠 제목, 댓글 등) |
| 🌙/☀ 버튼 | 다크 모드/라이트 모드 전환 |
| 🔔 알림 벨 | 알림 센터 바로가기 |
| 프로필 아바타 | 이름·이메일 표시, 로그아웃 |

## 4. 대시보드

**경로:** `/` (메인 화면)
**접근 역할:** SA, AM, AO, CD (전체)

로그인 후 가장 먼저 보이는 화면입니다. 선택한 기관의 핵심 지표를 한눈에 파악합니다.

### 4-1. 화면 구성

```
┌─────────────────────────────────────────────────────────────┐
│ 대시보드                          [최근 7일 ▾] [최근 30일 ▾] │
├──────────┬──────────┬──────────┬──────────┐                 │
│ 총 도달    │ 총 게시물  │ 총 댓글    │⚠ 위험 댓글│                 │
│ 45,231   │    127   │   892    │    12    │                 │
│ +2.1%    │ +15건    │ +8.3%   │ 즉시확인 │ ← 클릭 가능      │
└──────────┴──────────┴──────────┴──────────┘                 │
│                                                             │
│  ┌─── 댓글 감성 분석 ───┐  ┌──── 플랫폼별 성과 ────────┐    │
│  │     (도넛 차트)       │  │     (막대 차트)            │    │
│  │  긍정 40%  중립 35%  │  │  조회수 / 좋아요 / 공유    │    │
│  │  부정 20%  위험 5%   │  │  YouTube / Insta / FB ...  │    │
│  └──────────────────────┘  └────────────────────────────┘    │
│                                                             │
│  ┌─ 최근 콘텐츠 ───────┐  ┌─ 오늘 일정 ───────────────┐    │
│  │ 제목 | 상태 | 플랫폼 │  │ 09:00 YT 정책브리핑 ✅    │    │
│  └─────────────────────┘  │ 14:00 IG 봄축제 ⏳         │    │
│                            └───────────────────────────┘    │
│  ┌─ 승인 현황 ─────────────────────────────────────────┐    │
│  │ 승인 대기: 3건  검토 중: 1건  승인됨: 5건  반려: 1건  │    │
│  │                         [승인 대기 목록 보기 →]      │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                             │
│  데이터 최신성: YouTube 2분 전 · Instagram 5분 전 · ...     │
└─────────────────────────────────────────────────────────────┘
```

### 4-2. KPI 카드 (4개)

| 카드 | 설명 |
|---|---|
| **총 도달** | 모든 플랫폼의 합산 조회수 (API `views_growth` 값 표시) |
| **총 게시물** | 해당 기간 내 게시된 콘텐츠 수 (API `contents_growth` 값 표시) |
| **총 댓글** | 모든 플랫폼의 합산 댓글 수 (API `comments_growth` 값 표시) |
| **⚠ 위험 댓글** | 위험으로 분류된 미처리 댓글 수 (빨간색, 클릭 시 `/comments/dangerous`로 이동) |

### 4-3. AM/SA 역할: 기관별 현황 비교

AM 또는 SA 역할로 로그인하면, KPI 카드 아래에 **기관별 현황 비교 테이블**이 표시됩니다.
담당하는 모든 기관의 핵심 지표를 한눈에 비교할 수 있습니다.

| 컬럼 | 설명 |
|---|---|
| **기관명** | 기관 이름 |
| **전체 콘텐츠** | 해당 기관의 총 콘텐츠 수 |
| **게시 완료** | 게시 완료(PUBLISHED) 건수 |
| **활성 채널** | 연동된 활성 채널 수 |
| **대기 승인** | 검수 대기 중인 콘텐츠 수 |

### 4-4. 기간 선택

상단 우측에서 **최근 7일 / 최근 30일** 기간을 선택하여 KPI 및 차트 데이터를 필터링합니다.

### 4-5. 차트

- **댓글 감성 분석 (도넛 차트):** 수집된 댓글의 감성 분포 (긍정/중립/부정/위험)
- **플랫폼별 성과 (막대 차트):** 각 플랫폼(YouTube, Instagram 등)별 조회수·좋아요·공유 비교

### 4-6. 하단 리스트

- **최근 콘텐츠:** 최근 작성/게시된 콘텐츠 목록과 상태 태그, 플랫폼 약어(YT·IG·FB) 표시
- **오늘 일정:** 당일 예약 게시 예정인 콘텐츠를 시간순으로 표시 (플랫폼 태그 + 상태 아이콘)
- **승인 현황:** 검토 대기/검토 중/승인됨/반려됨 건수 요약 + **"승인 대기 목록 보기 →"** 링크

### 4-7. 데이터 최신성 표시

화면 하단에 각 플랫폼별 데이터의 마지막 동기화 시간이 표시됩니다 (예: "YouTube 2분 전 · Instagram 5분 전").

```
├───────────────────────────────────────────────────────────┤
│ ☐ │ 상태     │ 제목          │ 플랫폼   │ 작성자 │ 날짜   │ 관리│
│───┼──────────┼───────────────┼──────────┼────────┼────────┼─────│
│ ☐ │ 초안     │ 시민참여 안내  │ YT       │ 박실무 │ 3/5    │👁✏📤🗑│
│ ☐ │ 게시완료 │ 봄축제 홍보   │ IG·FB    │ 이매니 │ 3/4    │👁✏  │
│ ☐ │ 검수대기 │ 교통안전 캠페인│ X·Blog   │ 박실무 │ 3/3    │👁   │
├───────────────────────────────────────────────────────────┤
│ 선택: 2건                                       1 / 5 페이지│
└───────────────────────────────────────────────────────────┘
```

> **RBAC:** 위탁기관 담당자(CD) 역할에서는 **"+ 콘텐츠 작성"** 버튼이 표시되지 않습니다. CD는 콘텐츠 조회·승인만 가능합니다.

**상태 탭 바:** 상단 탭을 클릭하여 상태별 필터링 (기존 드롭다운 방식 → 탭 방식으로 변경)
- **전체** | **작성 중** (DRAFT) | **검수 대기** (PENDING_REVIEW) | **승인됨** (APPROVED) | **게시 완료** (PUBLISHED) | **반려** (REJECTED)

**테이블 컬럼 순서:** 상태 → 제목 → 플랫폼 → 작성자 → 날짜 → 관리

**플랫폼 약어 표시:** YouTube → **YT**, Instagram → **IG**, Facebook → **FB**, X → **X**, 네이버 블로그 → **Blog** (복수 플랫폼은 `·`로 구분)

**추가 필터:**
- **플랫폼:** YouTube, Instagram, Facebook, X, 네이버 블로그
- **기간:** 최근 7일 / 최근 30일 / 최근 90일 / 전체

**행 액션 버튼:**
| 아이콘 | 기능 | 조건 |
|---|---|---|
| 👁 상세 | 콘텐츠 상세 보기 | 항상 |
| ✏ 수정 | 콘텐츠 수정 | 초안/반려됨 상태만 |
| 📤 검토 요청 | 승인 요청 전송 | 초안 상태만 |
| 🗑 삭제 | 콘텐츠 삭제 | 확인 팝업 표시 |

**일괄 작업:** 체크박스로 여러 콘텐츠를 선택 후 "일괄 삭제" 가능. 하단에 선택 건수 표시.

#### 작성 절차

1. **게시 플랫폼** 상단에서 선택 (복수 선택 가능, 체크박스)
2. **미디어 업로드** — 이미지·영상·문서 파일 업로드 (최대 10개, 드래그 앤 드롭)
3. **제목** 입력 (필수, 500자 이내)
4. **본문** 작성 (자유 텍스트)
5. **해시태그** 입력 — 태그 형식 입력 (스페이스 또는 쉼표로 구분, AI 추천 가능)
6. **예약 게시일시** — 특정 날짜/시간에 자동 게시하려면 설정
7. **"저장 (초안)"** 클릭 → 초안 상태로 저장

#### 오른쪽 패널 구성

화면 오른쪽에는 3개의 카드가 순서대로 배치됩니다:

**① 플랫폼별 미리보기** — 선택한 플랫폼(YouTube / Instagram / Facebook / X / 네이버 블로그)에 맞는 콘텐츠 미리보기를 탭으로 전환하며 확인합니다. 제목·본문·해시태그가 실시간으로 반영됩니다.

**② AI 어시스턴트** — 본문을 10자 이상 작성한 뒤 AI 기능을 사용할 수 있습니다.

| 버튼 | 기능 | 사용 방법 |
|---|---|---|
| **🤖 제목 제안 받기** | AI가 본문을 분석하여 3개의 제목 후보를 생성 | 원하는 제목 클릭 → 자동 적용 |
| **🤖 설명문 제안 받기** | AI가 본문 기반 2개의 설명문 생성 | 원하는 설명문 클릭 → 본문 끝에 추가 |
| **🤖 해시태그 제안 받기** | AI가 관련 해시태그 5개 생성 | 원하는 해시태그 클릭 → 기존 태그에 자동 추가 |
| **🔄 AI 톤 변환** | 본문을 다른 톤/플랫폼에 맞게 변환 | 플랫폼+톤 선택 후 "적용하기" |
| **✅ AI 콘텐츠 검수** | 맞춤법, 민감 표현, 편향성 검사 | 발견된 이슈와 수정 제안 표시 |

**③ 영상 콘텐츠 추가 기능** — 자막 자동 생성(F03), 숏폼 생성(F15), 효과음/이모지 추천 버튼이 표시됩니다 (Phase 2에서 활성화).

#### 워크플로우 진행바

화면 하단에 현재 콘텐츠의 상태 흐름이 표시됩니다:
```
● 작성 중 → ○ 검토 요청 → ○ 검수 중 → ○ 게시
```
현재 단계가 파란색 원(●)으로, 미래 단계가 빈 원(○)으로 표시됩니다.

#### 상단 액션 버튼

- **임시 저장** — 현재 작성 중인 내용을 초안으로 저장
- **검토 요청** — 승인 워크플로우 시작 (검수 대기 상태로 전환)

### 5-4. 검토 요청 (승인 프로세스 시작)

콘텐츠 목록에서 초안 상태의 콘텐츠 행에 있는 **📤 (검토 요청)** 아이콘을 클릭하면 승인 워크플로우가 시작됩니다.

콘텐츠 상태 전이:
```
초안(DRAFT) → 검토 대기(PENDING_REVIEW) → 검토 중(IN_REVIEW) → 승인됨(APPROVED) → 예약/게시
                                                              ↘ 반려됨(REJECTED) → 수정 후 재요청
```

### 7-2. AI 답글 생성

### 7-3. 답글 템플릿 AI 개선

## 부록 B: Docker 없이 로컬 환경 실행

Docker Desktop 없이 Windows에서 직접 인프라를 설치·실행하는 방법입니다.
테스트·개발 목적으로 최소 구성(PostgreSQL + Redis)만 있으면 대부분의 기능이 동작합니다.

### B-1. 사전 요구사항

| 항목 | 필요 버전 | 용도 |
|---|---|---|
| Node.js | 18 이상 | 프론트엔드 빌드 |
| pnpm | 9.x 이상 | 프론트엔드 패키지 관리 |
| Python | 3.11 이상 | 백엔드 실행 |
| PostgreSQL | 16.x | 데이터베이스 |
| Redis | 7.x (또는 Memurai) | 캐시·브로커 |
| MinIO *(선택)* | 최신 | 파일 저장소 (없어도 기본 기능 동작) |

### B-2. PostgreSQL 설치 (Windows)

#### 방법 A: 공식 인스톨러

1. https://www.postgresql.org/download/windows/ 에서 PostgreSQL 16 다운로드
2. 설치 시 **포트: 5432**, **슈퍼유저 비밀번호** 설정
3. 설치 완료 후 DB 및 유저 생성:

```powershell
# psql 접속 (설치 시 설정한 슈퍼유저 비밀번호 입력)
psql -U postgres

# DB 및 유저 생성
CREATE USER publisync WITH PASSWORD 'publisync';
CREATE DATABASE publisync OWNER publisync;
GRANT ALL PRIVILEGES ON DATABASE publisync TO publisync;
\q
```

## 부록 C: 실전 시나리오 — 콘텐츠 작성부터 게시까지

이 시나리오는 미디어 라이브러리의 동영상을 활용하여 콘텐츠를 작성하고, 승인 과정을 거쳐 YouTube와 X(Twitter)에 게시하는 전체 과정을 안내합니다.

### C-1. 사전 준비

| 항목 | 확인 사항 |
|---|---|
| MinIO 실행 | 프로젝트 루트에서 `./minio.exe server ./minio-data --console-address ":9001"` |
| 백엔드 실행 | `cd backend && uvicorn app.main:app --reload --port 8000` |
| 프론트엔드 실행 | `cd frontend && pnpm dev` |
| 로그인 계정 | 수탁업체 실무자(AO): `operator@digitalsotong.kr` / `demo1234!` |
| 승인 계정 | 위탁기관 담당자(CD): `director@seoul.go.kr` / `demo1234!` |
| 채널 연동 | 채널 관리(`/channels`)에서 YouTube, X가 "연동됨" 상태인지 확인 |

### C-2. Step 1 — 미디어 라이브러리에서 동영상 확인

1. AO 계정으로 로그인합니다.
2. 좌측 메뉴에서 **미디어 라이브러리**(`/media`)를 클릭합니다.
3. 업로드된 동영상 파일(예: `팔봉_10번홀_티샷.mp4`)이 그리드에 표시되는지 확인합니다.
4. 눈(👁) 아이콘을 클릭하면 **미디어 상세** 모달에서 동영상 미리보기와 메타데이터(해상도, 재생 시간 등)를 확인할 수 있습니다.

> **팁:** 미디어 라이브러리에 파일이 없다면 "업로드" 버튼이나 드래그 앤 드롭으로 먼저 업로드하세요.

### C-3. Step 2 — 새 콘텐츠 작성

1. 좌측 메뉴에서 **새 콘텐츠 작성**(`/contents/create`)을 클릭합니다.

2. **플랫폼 선택** — 상단에서 **YouTube**와 **X**를 체크합니다.

3. **채널 선택** — 연동된 채널을 선택합니다 (예: "모름지기" YouTube 채널, "X Account").

4. **미디어 첨부** — 두 가지 방법 중 선택:
   - **직접 업로드:** 드래그 앤 드롭 영역에 파일을 끌어다 놓기
   - **미디어 라이브러리에서 선택:** "미디어 라이브러리에서 선택" 버튼 클릭 → 그리드에서 원하는 파일을 클릭(체크 표시) → "선택 완료" 클릭

5. **제목 입력** — 예: `골프 코스 투어 - 팔봉 10번홀 티샷`

6. **본문 작성** — 예: `아름다운 풍경과 함께하는 팔봉 골프장 10번홀 티샷 영상입니다.`

7. **해시태그** — 예: `#골프`, `#팔봉`, `#티샷`

> **참고:** YouTube는 동영상 필수, X는 텍스트만으로도 게시 가능합니다. X는 280자 제한이 있어 초과 시 자동으로 잘립니다.

### C-4. Step 3 — 검토 요청

1. 콘텐츠 작성이 완료되면 상단의 **"검토 요청"** 버튼을 클릭합니다.
2. 확인 메시지가 표시되면 **"확인"**을 클릭합니다.
3. 콘텐츠 상태가 **초안(DRAFT) → 검수 대기(PENDING_REVIEW)**로 전환됩니다.
4. 자동으로 콘텐츠 상세 페이지로 이동합니다.

> 이 시점에서 위탁기관 담당자(CD)에게 승인 요청 알림이 전송됩니다.

### C-5. Step 4 — 승인 (CD 계정)

1. 현재 AO 계정에서 **로그아웃**합니다 (우측 상단 프로필 → 로그아웃).
2. **위탁기관 담당자(CD)** 계정으로 로그인합니다: `director@seoul.go.kr` / `demo1234!`
3. 좌측 메뉴에서 **승인 대기**(`/approvals`)를 클릭합니다.
4. "내가 검수할 항목" 탭에서 방금 요청된 콘텐츠 카드를 찾습니다.
5. 카드를 클릭하면 **분할 패널**이 열립니다:
   - **왼쪽:** 콘텐츠 미리보기 (제목, 본문, 미디어, 플랫폼)
   - **오른쪽:** 승인 이력 + 의견 입력란
6. 의견란에 `"확인 완료"` 등을 입력한 후 **"승인"** 버튼을 클릭합니다.
7. 콘텐츠 상태가 **검수 대기 → 승인됨(APPROVED)**으로 전환됩니다.

### C-6. Step 5 — 게시 (AO 계정)

1. CD 계정에서 **로그아웃**합니다.
2. **수탁업체 실무자(AO)** 계정으로 다시 로그인합니다: `operator@digitalsotong.kr` / `demo1234!`
3. **콘텐츠 목록**(`/contents`)에서 "승인됨" 탭을 클릭합니다.
4. 해당 콘텐츠의 **상세 보기(👁)** 아이콘을 클릭합니다.
5. 콘텐츠 상세 페이지에서 **"게시"** 버튼이 활성화되어 있습니다.
6. **"게시"** 버튼을 클릭하고 확인 팝업에서 **"확인"**을 누릅니다.
7. 시스템이 선택된 채널(YouTube, X)에 순서대로 게시를 시도합니다.

### C-7. Step 6 — 게시 결과 확인

1. 콘텐츠 상세 페이지에서 **"게시 이력"** 탭을 클릭합니다.
2. 채널별 게시 결과가 타임라인으로 표시됩니다:

| 상태 | 의미 | 표시 |
|---|---|---|
| ✅ SUCCESS | 게시 성공 | 플랫폼 URL 링크 표시 |
| ❌ FAILED | 게시 실패 | 에러 메시지 표시 |
| ⚠️ PARTIAL | 일부 성공 | 성공/실패 채널별 상세 |

3. 최종 콘텐츠 상태:
   - **모두 성공** → `PUBLISHED` (게시 완료)
   - **일부 성공** → `PARTIALLY_PUBLISHED` (부분 게시) → "재시도" 버튼으로 실패 채널 재게시
   - **모두 실패** → `PUBLISH_FAILED` (게시 실패) → "재시도" 버튼 표시

```
│ │  │  업로드 드롭존 / VideoPlayer / 이미지 갤러리 │           │  │
│ │  └────────────────────────────────────────────┘           │  │
│ │  ┌────────────────────────────────────────────┐           │  │
│ │  │  WaveformViewer (영상 있을 때만)            │           │  │
│ │  └────────────────────────────────────────────┘           │  │
│ │  [파일 추가] [교체]                                       │  │
│ └───────────────────────────────────────────────────────────┘  │
│                                                                │
│ ┌─ 도구 바 ───────────────────────────────────────────────┐    │
│ │ [자막] [효과음] [이모지] [숏폼 추출] [AI 어시스턴트]     │    │
│ └──────────────────────────────────────────────────────────┘   │
│                                                                │
│ ┌─ 활성 패널 (도구 바 선택에 따라 전환) ──────────────────┐    │
│ │  자막 편집 / 효과음 관리 / 이모지 관리 / 숏폼 추출 / AI │    │
│ └──────────────────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────────────────┘
```

1. AO 계정으로 로그인합니다.
2. 좌측 메뉴에서 **콘텐츠** → **+ 새 콘텐츠** 버튼을 클릭합니다.
3. 화면 중앙의 **업로드 드롭존**에 영상 파일을 드래그하거나, **미디어 라이브러리에서 선택** 버튼을 클릭합니다.
4. 영상 업로드가 완료되면 자동으로 **VideoPlayer + WaveformViewer**로 전환됩니다.
5. 헤더의 **[콘텐츠 정보]** 버튼을 클릭하여 제목·본문·해시태그를 입력합니다.
6. **[임시 저장]**을 클릭하여 초안을 저장합니다.

> **핵심 변경:** 이전 버전에서는 텍스트 입력이 메인이었지만, v2.0에서는 영상이 메인이고 텍스트는 모달로 분리되었습니다.

### D-3. Step 2 — AI 자막 생성

자막 편집은 콘텐츠 에디터 안에서 바로 가능합니다.

1. 도구 바에서 **[자막]** 을 선택합니다.
2. 자막 패널이 표시됩니다:

```
┌─ 자막 편집 ──────────────────────────────────────────────────────┐
│ [AI 자막 생성] [서버에 저장] [🎬 자막 합성 영상] [SRT 내보내기] [+ 추가] │
│ ┌─ 진행 상태 (AI 작업 중일 때) ──────────────────────┐           │
│ │ ■ 처리 중  72%  ████████░░░░                       │           │
│ └────────────────────────────────────────────────────┘           │
│ ┌─ 자막 목록 ────────────────────────────────────────┐           │
│ │ #1  00:00:01,000 ~ 00:00:03,500                    │           │
│ │     [안녕하세요 서울시에서 전합니다___________][삭제] │           │
│ │ #2  00:00:03,500 ~ 00:00:06,200                    │           │
│ │     [오늘 소개할 정책은___________________] [삭제]   │           │
│ └────────────────────────────────────────────────────┘           │
└──────────────────────────────────────────────────────────────────┘
```

3. **"AI 자막 생성"** 버튼을 클릭합니다.
4. AI 작업이 완료되면 자막 목록이 자동으로 채워집니다.
5. 상단 VideoPlayer에서 영상을 재생하며 자막 시간/텍스트를 확인·수정합니다.

| 동작 | 방법 |
|---|---|
| 시작/종료 시간 수정 | 각 자막의 시간 입력란 직접 편집 (HH:MM:SS,mmm) |
| 자막 텍스트 수정 | 텍스트 영역 직접 편집 |
| 자막 추가 | **[+ 추가]** 버튼 (이전 자막 종료 시점부터 자동 배치) |
| 자막 삭제 | 각 항목의 **[삭제]** 버튼 (자동 재번호) |
| 서버 저장 | **[서버에 저장]** 버튼 |
| 자막 합성 영상 | **[🎬 자막 합성 영상]** 버튼 → 자막이 영상에 하드코딩된 새 영상 생성 |
| SRT 내보내기 | **[SRT 내보내기]** 버튼 → .srt 파일 다운로드 |

6. **자막 합성 영상 생성** — 자막 편집이 완료되면 **[🎬 자막 합성 영상]** 버튼을 클릭합니다.
   - 현재 자막이 자동으로 서버에 저장된 후, ffmpeg로 자막이 영상에 합성(burn-in)됩니다.
   - 완료 후 **미디어 라이브러리**에서 `{원본파일명}_subtitled.mp4` 파일을 확인할 수 있습니다.
   - 저장 위치: `{조직ID}/media/subtitle-burnin/` (MinIO 또는 로컬 storage)
   - 재생: 미디어 라이브러리에서 선택 후 내장 VideoPlayer로 재생하거나, 다운로드하여 로컬 플레이어(VLC 등)로 재생

> **Human-in-the-Loop:** AI가 생성한 자막은 반드시 사용자가 시간·텍스트를 검토한 뒤 저장합니다.

### D-4. Step 3 — 효과음 추가 (신규 기능)

도구 바에서 **[효과음]** 을 선택합니다.

```
┌─ 효과음 관리 ────────────────────────────────────────────┐
│ [AI 효과음 추천] [수동 추가]                              │
│                                                           │
│ AI 효과음 추천:                                           │
│ ┌─────────────────────────────────────────────────────┐   │
│ │ "박수 효과음" 92%  [사용하기]                        │   │
│ │ "전환 효과음" 85%  [사용하기]                        │   │
│ └─────────────────────────────────────────────────────┘   │
│                                                           │
│ 효과음 목록:                                              │
│ [박수 효과음] [0초] ~ [2초]  [삭제]                        │
│ [전환 효과음] [5초] ~ [2초]  [삭제]                        │
└──────────────────────────────────────────────────────────┘
```

1. **"AI 효과음 추천"** 버튼을 클릭하면 AI가 콘텐츠에 맞는 효과음을 추천합니다.
2. 추천 결과에서 **[사용하기]**를 클릭하면 효과음 목록에 추가됩니다.
3. 각 효과음의 **삽입 시점(초)**과 **재생 길이(초)**를 지정합니다.
4. **[수동 추가]**로 직접 효과음 항목을 추가할 수도 있습니다.

### D-5. Step 4 — 이모지 오버레이 추가 (신규 기능)

도구 바에서 **[이모지]** 를 선택합니다.

1. **"AI 이모지 추천"** 버튼을 클릭하면 AI가 콘텐츠에 맞는 이모지를 추천합니다.
2. 각 이모지에 대해 **삽입 시점**, **표시 시간**, **위치**(좌상/우상/중앙/좌하/우하)를 지정합니다.
3. **[수동 추가]**로 직접 이모지를 추가할 수도 있습니다.

> **데이터 저장:** 효과음/이모지 데이터는 콘텐츠의 metadata 필드에 자동 저장됩니다. 별도 저장 버튼 불필요.

### D-6. Step 5 — AI 숏폼 클립 추출

도구 바에서 **[숏폼 추출]** 을 선택합니다.

1. **"AI 구간 추출"** 버튼을 클릭합니다.
2. AI가 영상을 분석하여 숏폼에 적합한 구간을 자동 추출합니다.
3. 각 클립에 대해:
   - **체크박스:** 사용할 구간 선택/해제
   - **[미리보기]:** 클릭하면 상단 VideoPlayer가 해당 시점으로 이동
   - **시간·라벨:** 시작~종료 시간과 구간 길이 표시
4. **[선택한 구간 확인]** 클릭 → 요약 모달에서 최종 확인 후 **[확정]**을 누릅니다.
5. **숏폼 영상 렌더링** — 구간 확정 후 **[숏폼 영상 렌더링]** 버튼을 클릭합니다.
   - 선택된 구간별로 ffmpeg가 개별 영상 클립을 생성합니다.
   - (선택) "자막 포함" 체크 시 저장된 자막이 클립에 함께 합성됩니다.
   - 완료 후 **미디어 라이브러리**에서 `{원본파일명}_clip1.mp4`, `_clip2.mp4`, ... 파일을 확인합니다.
   - 저장 위치: `{조직ID}/media/shortform/` (MinIO 또는 로컬 storage)
   - 재생: 미디어 라이브러리에서 선택 후 내장 VideoPlayer로 재생하거나, 다운로드 후 로컬 재생
   - 활용: YouTube Shorts, Instagram Reels, TikTok 등에 직접 게시 가능

> **팁:** 숏폼은 15~60초 구간이 권장됩니다. YouTube Shorts, Instagram Reels 등에 활용됩니다.

### D-7. Step 6 — AI 어시스턴트 (톤 변환·검수·번역·썸네일)

도구 바에서 **[AI 어시스턴트]** 를 선택하면 4개 AI 도구가 탭으로 표시됩니다:

| 탭 | 기능 | 설명 |
|---|---|---|
| **톤 변환** | 플랫폼별 톤 변경 | 플랫폼 + 톤(공식적/캐주얼/친근한/전문적) 선택 → 변환 |
| **콘텐츠 검수** | 맞춤법·민감표현·편향성 검사 | HIGH/MEDIUM/LOW 심각도 이슈 표시 |
| **번역** | 4개 언어 번역 | en, zh, ja, vi 선택 → 번역 결과 표시 |
| **썸네일** | AI 썸네일 생성 | 스타일(4종) + 비율(3종) 선택 → 비동기 생성 |

> **공공기관 필수:** 게시 전 반드시 **콘텐츠 검수** 탭에서 AI 검수를 실행하는 것을 권장합니다.

### D-8. Step 7 — 콘텐츠 정보 입력 (모달)

헤더의 **[콘텐츠 정보]** 버튼을 클릭하면 모달이 열립니다:

- **제목:** 필수 입력. AI 제안 드롭다운으로 제목/설명문/해시태그 AI 제안 가능
- **본문/설명문:** 콘텐츠 본문
- **해시태그:** 태그 입력. AI 해시태그 추천 결과 자동 병합
- **예약 게시:** 선택적 날짜/시간 설정

> **AI 버그 수정:** v2.0에서는 제목 또는 본문 중 하나만 있어도 AI 제안이 동작합니다 (기존: 본문 10자 이상 필수).

### D-9. 전체 워크플로우 요약

```
원본 영상 수신
    │
    ▼
① 새 콘텐츠 작성 + 영상 업로드 (/contents/create)
    │  → 영상 드래그 앤 드롭 → 자동 VideoPlayer 전환
    │
    ├──▶ ② 도구 바 [자막] → AI 자막 생성 → 편집 → 저장/SRT/자막 합성 영상
    │
    ├──▶ ③ 도구 바 [효과음] → AI 추천 → 시점·길이 지정
    │
    ├──▶ ④ 도구 바 [이모지] → AI 추천 → 시점·위치 지정
    │
    ├──▶ ⑤ 도구 바 [숏폼 추출] → AI 구간 추출 → 선택 → 확정 → 숏폼 렌더링
    │
    ├──▶ ⑥ 도구 바 [AI 어시스턴트] → 톤 변환 / 검수 / 번역 / 썸네일
    │
    ├──▶ ⑦ 헤더 [콘텐츠 정보] → 제목·본문·해시태그·예약 게시
    │
    ├──▶ ⑧ 헤더 [플랫폼 설정] → 플랫폼 선택·채널 배정·변형본
    │
    ▼
⑨ [검수 요청] → 승인 → 게시 (부록 C 참조)
```

### D-10. 문제 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| AI 자막 생성 버튼 클릭 후 무반응 | Celery 워커 미실행 | `celery -A app.tasks worker -Q ai --loglevel=info` 실행 |
| "처리 중" 상태에서 멈춤 | AI API 키 미설정 또는 만료 | `.env`에서 `OPENAI_API_KEY` 등 확인 |
| 영상이 재생되지 않음 | 미디어 파일이 스토리지에 없음 | 미디어 라이브러리에서 파일 업로드 상태 확인 |
| 도구 바 [자막]/[숏폼] 클릭 시 "영상을 업로드하세요" | source_media_id 미설정 | 콘텐츠를 저장한 뒤 편집 모드에서 진입 |
| 효과음/이모지가 저장 안 됨 | 콘텐츠 미저장 | 임시 저장 클릭 후 확인 |
| 자막 합성/숏폼 렌더링 "처리 중" 멈춤 | ffmpeg 미설치 | 서버에 ffmpeg 7.0+ 설치 확인 (`ffmpeg -version`) |
| 합성 영상이 미디어 라이브러리에 안 보임 | 워크스페이스 캐시 | 페이지 새로고침 또는 미디어 라이브러리 목록 재조회 |
| 검수에서 이슈가 0개 | 콘텐츠 텍스트가 너무 짧음 | 5자 이상 텍스트 작성 후 재실행 |

> **팁:** 미디어 라이브러리에서도 독립적으로 자막 편집기(`/ai/subtitle-editor/{assetId}`)와 숏폼 편집기(`/ai/shortform-editor/{assetId}`)에 진입할 수 있습니다.

## 부록 E: 실전 시나리오 — 콘텐츠 수정 및 플랫폼별 변형본 관리

기존 콘텐츠를 수정하거나, 하나의 원본에서 플랫폼별 최적화된 변형본을 관리하는 워크플로우입니다.

### E-1. 사전 준비

| 항목 | 확인 사항 |
|---|---|
| 로그인 계정 | AO: `operator@digitalsotong.kr` / `demo1234!` |
| 기존 콘텐츠 | 초안(DRAFT) 또는 반려됨(REJECTED) 상태의 콘텐츠 |
| 채널 연동 | 게시 대상 채널이 "연동됨" 상태 |

### E-2. Step 1 — 콘텐츠 수정 진입

1. 좌측 메뉴에서 **콘텐츠**(`/contents`)를 클릭합니다.
2. 수정할 콘텐츠의 **수정** 아이콘을 클릭합니다.
3. 콘텐츠 편집기(`/contents/{id}/edit`)가 열립니다.
4. 기존 미디어가 있으면 자동으로 VideoPlayer(영상) 또는 이미지 갤러리가 표시됩니다.

> **주의:** 수정은 **초안(DRAFT)** 또는 **반려됨(REJECTED)** 상태에서만 가능합니다.

### E-3. Step 2 — 콘텐츠 정보 수정 (모달)

헤더의 **[콘텐츠 정보]** 버튼을 클릭하여 모달에서 수정합니다:

1. **제목·본문·해시태그:** 공통 내용을 수정합니다.
2. **AI 제안 활용:** 드롭다운에서 AI 제목/설명문/해시태그 제안을 받을 수 있습니다.
3. **예약 게시:** 필요 시 날짜/시간 설정 또는 해제
4. **[적용]** 을 클릭하여 변경 내용을 반영합니다.

### E-4. Step 3 — 플랫폼 설정 (모달)

헤더의 **[플랫폼 설정]** 버튼을 클릭하여 모달에서 설정합니다:

1. **플랫폼 선택:** 체크박스에서 게시 대상 플랫폼을 변경합니다.
2. **게시 모드:**
   - **전체 동일:** 모든 플랫폼에 동일한 콘텐츠 게시. 채널 선택만 설정.
   - **플랫폼별 맞춤:** 토글 활성화 시, 각 플랫폼 탭에서 변형본(VariantEditor)을 편집합니다.

```
┌─── 플랫폼 설정 ─────────────────────────────────────────┐
│ 게시 플랫폼: ☑ YouTube  ☑ Instagram  ☐ X  ☐ Facebook   │
│                                                          │
│ 게시 모드: [전체 동일 ⟷ 플랫폼별 맞춤]                    │
│                                                          │
│ [YouTube] [Instagram]                          ← 탭      │
│ ┌─ YouTube 변형본 ────────────────────────────────────┐   │
│ │ 채널: [모름지기 ▾]                                  │   │
│ │ 제목: [플랫폼별 제목] (비우면 공통 제목 사용)       │   │
│ │ 본문: [플랫폼별 본문] (비우면 공통 본문 사용)       │   │
│ │ 해시태그: [...]                                     │   │
│ └─────────────────────────────────────────────────────┘   │
│                                          [적용]           │
└──────────────────────────────────────────────────────────┘
```

3. 각 변형본에서:
   - **제목/본문/해시태그 오버라이드:** 값을 입력하면 해당 플랫폼만 다른 내용 사용
   - **채널 선택:** 해당 플랫폼의 연동된 채널 지정

### E-5. Step 4 — 미디어 교체 및 영상 도구 활용

1. 메인 미디어 영역에서 **[파일 추가]** 또는 **[교체]** 버튼으로 미디어를 변경합니다.
2. 영상이 변경되면 자동으로 VideoPlayer가 갱신됩니다.
3. 도구 바에서 자막/효과음/이모지/숏폼 도구를 활용하여 영상 부가 정보를 수정합니다.

### E-6. Step 5 — AI 활용으로 변형본 최적화

도구 바에서 **[AI 어시스턴트]**를 선택하여 AI를 활용합니다:

| 플랫폼 | 추천 AI 전략 |
|---|---|
| **YouTube** | AI 제목 제안(SEO 최적화) + AI 해시태그(검색 키워드) |
| **Instagram** | AI 톤 변환("캐주얼") + AI 해시태그(인기 태그) |
| **X** | AI 톤 변환(짧은 임팩트) — 280자 제한 유의 |
| **전체** | AI 콘텐츠 검수 실행(맞춤법·민감표현·편향성 확인) |

### E-7. Step 6 — 저장 및 검토 요청

1. 모든 수정이 완료되면 **[임시 저장]**을 클릭합니다.
2. **[검수 요청]**을 클릭합니다.
3. 확인 모달에서 플랫폼별 현황이 표시됩니다:
   - 플랫폼별 맞춤 여부 (커스터마이즈됨 / 공통 정보 사용)
   - 각 변형본의 채널 배정 상태
4. **[검수 요청]**을 클릭하면 검수 대기(PENDING_REVIEW) 상태로 전환됩니다.

### E-8. 반려 후 재수정 워크플로우

콘텐츠가 반려된 경우:

1. 콘텐츠 목록에서 **"반려됨"** 탭을 클릭합니다.
2. 해당 콘텐츠의 **수정** 아이콘을 클릭합니다.
3. 반려 사유를 확인합니다 (콘텐츠 상세의 승인 이력 탭).
4. 반려 사유에 따라 내용을 수정합니다.
5. 수정 완료 후 다시 **[검수 요청]**을 클릭합니다.

```
DRAFT → PENDING_REVIEW → REJECTED → (수정) → PENDING_REVIEW → APPROVED → 게시
```

### E-9. 문제 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 수정 버튼이 없음 | APPROVED/PUBLISHED 상태 | 초안 또는 반려 상태에서만 수정 가능 |
| 변형본 탭이 안 보임 | [플랫폼 설정] 모달에서 맞춤 모드 비활성 | 토글을 켠 뒤 플랫폼 2개 이상 선택 |
| 채널 목록이 비어 있음 | 채널 미연동 또는 상태 INACTIVE | 채널 관리에서 해당 플랫폼 채널 연동 |
| 저장 시 500 오류 | 백엔드 variants 로딩 오류 | 페이지 새로고침 후 재시도 |
| 검토 요청 실패 | 필수 필드 미입력 | [콘텐츠 정보] 모달에서 제목 입력 확인 |
| 제목 미입력 시 저장 불가 | 제목 필수 | [콘텐츠 정보] 모달이 자동으로 열림 |
