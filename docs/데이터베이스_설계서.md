## 1. 설계 원칙

### 1.1 8가지 핵심 원칙

| # | 원칙 | 설명 |
|---|---|---|
| 1 | **UUID v7 PK** | 시간 정렬 가능한 UUID v7을 모든 테이블 PK로 사용. `gen_random_uuid()` 대신 `uuid_generate_v7()` 사용 (pg_uuidv7 확장) |
| 2 | **RLS 기반 멀티테넌트** | `organization_id` FK를 가진 테이블에 PostgreSQL RLS 적용. `SET LOCAL app.current_org_id`로 세션 변수 설정 |
| 3 | **네이밍 규칙** | snake_case, 복수형 테이블명, `{table}_id` FK, `idx_{table}_{columns}` 인덱스 |
| 4 | **타임스탬프** | `created_at` DEFAULT NOW(), `updated_at` (트리거 자동 갱신), `deleted_at` (소프트 삭제) |
| 5 | **JSONB 활용** | 구조가 자주 변하는 필드에 JSONB 사용 (metadata, settings, ai_result 등). GIN 인덱스 적용 |
| 6 | **ENUM 타입** | PostgreSQL `CREATE TYPE`으로 정의. SQLAlchemy `Enum` 매핑. 값 추가만 허용 (삭제/변경 X) |
| 7 | **인덱스 전략** | 모든 FK에 B-tree, 빈번한 필터 컬럼에 복합 인덱스, JSONB에 GIN, tsvector에 GIN |
| 8 | **Phase별 점진 확장** | 테이블/컬럼 추가만 수행. 기존 스키마 변경 최소화. 마이그레이션은 스프린트와 1:1 대응 |

### 1.2 공통 타임스탬프 트리거

```sql
-- updated_at 자동 갱신 트리거 함수
CREATE OR REPLACE FUNCTION trigger_set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- 각 테이블에 적용
CREATE TRIGGER set_updated_at
  BEFORE UPDATE ON {table_name}
  FOR EACH ROW
  EXECUTE FUNCTION trigger_set_updated_at();
```

## 2. ENUM 타입 정의

18개 PostgreSQL ENUM 타입.

> **RLS 세션 변수 통일:** SA 우회 정책에 `current_setting('app.current_role', true) = 'SYSTEM_ADMIN'` 사용.
> 아키텍처 설계서에서 `app.user_role`로 표기된 부분은 본 문서의 `app.current_role`로 통일한다.
> 미들웨어에서 `SET LOCAL app.current_role = :role`을 설정해야 한다.

> **사용자 역할 정책 (다중 기관 지원):**
> - `users.role`은 **기본(글로벌) 역할**. 최초 가입/초대 시 설정. SA는 항상 글로벌.
> - `user_organizations.role`은 **기관별 역할**. 동일 사용자가 기관마다 다른 역할 가능.
> - **워크스페이스 전환 시** RBAC 미들웨어는 `user_organizations.role`을 사용:
>   ```python
>   # Workspace 미들웨어 역할 결정 로직
>   uo = await db.get(UserOrganization, user_id=user.id, organization_id=workspace_id)
>   effective_role = uo.role if uo else user.role  # fallback to global
>   await session.execute(text("SET LOCAL app.current_role = :role"), {"role": effective_role})
>   ```
> - API 응답의 `role` 필드는 **현재 워크스페이스 기준 역할**을 반환.
> - SA 역할은 `user_organizations`에 관계없이 전체 접근 (RLS `sa_bypass` 정책).

```sql
  'PENDING_REVIEW',    -- 검토 요청 (승인 대기)
  'IN_REVIEW',         -- 검수 중 (검수자 확인 중)
  'APPROVED',          -- 승인 완료
  'REJECTED',          -- 반려
  'SCHEDULED',         -- 예약 게시 대기
  'PUBLISHING',        -- 게시 중
  'PUBLISHED',         -- 게시 완료
  'PARTIALLY_PUBLISHED', -- 일부 플랫폼만 성공
  'FAILED',            -- 게시 실패
  'CANCELLED',         -- 예약 취소
  'DELETED'            -- 삭제됨

-- content_status 상태 전이 규칙
-- ┌──────────────────────────────────────────────────────────────┐
-- │  DRAFT ──→ PENDING_REVIEW ──→ IN_REVIEW ──→ APPROVED       │
-- │    ↑            │                  │            │            │
-- │    │            │                  │            ↓            │
-- │    │            │                  │        SCHEDULED ──→ PUBLISHING
-- │    │            │                  │            │         ↙     ↘
-- │    │            │                  ↓            │   PUBLISHED  FAILED
-- │    │            └──────────→ REJECTED           │       ↓
-- │    │                           │                │  PARTIALLY_PUBLISHED
-- │    └───────────────────────────┘                │
-- │                                                 ↓
-- │                                            CANCELLED
-- │  (모든 상태 → DELETED: soft delete)                         │
-- └──────────────────────────────────────────────────────────────┘
--
-- 허용 전이:
--   DRAFT           → PENDING_REVIEW, DELETED
--   PENDING_REVIEW  → IN_REVIEW, REJECTED, DELETED
--   IN_REVIEW       → APPROVED, REJECTED
--   REJECTED        → DRAFT (수정 후 재요청)
--   APPROVED        → SCHEDULED, PUBLISHING, DELETED
--   SCHEDULED       → PUBLISHING, CANCELLED
--   PUBLISHING      → PUBLISHED, PARTIALLY_PUBLISHED, FAILED
--   PUBLISHED       → DELETED
--   PARTIALLY_PUBLISHED → DELETED (일부 재시도는 publish_results 레벨)
--   FAILED          → PUBLISHING (재시도), DELETED
--   CANCELLED       → DRAFT (재활용), DELETED
--
-- PARTIALLY_PUBLISHED 판정 로직:
--   게시 완료 후 해당 content의 모든 publish_results를 집계:
--   - 모두 PUBLISHED   → contents.status = 'PUBLISHED'
--   - 모두 FAILED      → contents.status = 'FAILED'
--   - 혼재 (일부 성공) → contents.status = 'PARTIALLY_PUBLISHED'
--   Celery 콜백에서 자동 판정 (publish_content_callback 태스크)

-- 게시 결과 상태 (API와 통일: SUCCESS → PUBLISHED)
CREATE TYPE publish_status AS ENUM (
  'PENDING',           -- 대기
  'PUBLISHING',        -- 게시 중
  'PUBLISHED',         -- 성공
  'PARTIAL_FAIL',      -- 일부 실패
  'FAILED'             -- 전체 실패
);

-- 승인 액션
CREATE TYPE approval_action AS ENUM (
  'APPROVE',           -- 승인
  'REJECT',            -- 반려
  'REQUEST_REVISION'   -- 수정 요청
);
```

## 3. Phase 1-A 테이블 (18개)

### 3.1 users — 사용자

> RLS 미적용. 여러 기관에 소속 가능 (다대다).

```sql
CREATE TABLE users (
  id              UUID PRIMARY KEY DEFAULT uuid_generate_v7(),
  email           VARCHAR(255) NOT NULL UNIQUE,
  password_hash   VARCHAR(255) NOT NULL,
  name            VARCHAR(100) NOT NULL,
  role            user_role NOT NULL DEFAULT 'AGENCY_OPERATOR',
  status          user_status NOT NULL DEFAULT 'ACTIVE',
  organization_id UUID REFERENCES organizations(id),  -- 기본 소속 기관 (nullable for SA)
  profile_image_url VARCHAR(512),
  preferences     JSONB NOT NULL DEFAULT '{}',         -- theme, language, defaultView
  last_login_at   TIMESTAMPTZ,
  failed_login_count INT NOT NULL DEFAULT 0,
  locked_until    TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  deleted_at      TIMESTAMPTZ                          -- soft delete
);

-- 인덱스
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_role ON users(role);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_organization_id ON users(organization_id);
CREATE INDEX idx_users_deleted_at ON users(deleted_at) WHERE deleted_at IS NULL;
```

| password_hash | VARCHAR(255) | bcrypt 12 라운드 해시 |
| name | VARCHAR(100) | 표시 이름 |

### 3.5 user_organizations — 사용자-기관 다대다 매핑

> RLS 미적용. 사용자의 다중 기관 소속 관리. AM/AO는 여러 기관 담당 가능.

```sql
CREATE TABLE user_organizations (
  id              UUID PRIMARY KEY DEFAULT uuid_generate_v7(),
  user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  role            user_role NOT NULL,
```

**자동 파티션 관리 함수:**

```sql
CREATE OR REPLACE FUNCTION create_next_audit_partition() RETURNS void AS $$
DECLARE
  next_month DATE;
  partition_name TEXT;
  start_date TEXT;
  end_date TEXT;
BEGIN
  next_month := date_trunc('month', NOW()) + INTERVAL '1 month';
  partition_name := 'audit_logs_' || to_char(next_month, 'YYYY_MM');
  start_date := to_char(next_month, 'YYYY-MM-DD');
  end_date := to_char(next_month + INTERVAL '1 month', 'YYYY-MM-DD');

  EXECUTE format(
    'CREATE TABLE IF NOT EXISTS %I PARTITION OF audit_logs
     FOR VALUES FROM (%L) TO (%L)',
    partition_name, start_date, end_date
  );
END;
$$ LANGUAGE plpgsql;
```

### 12.3 RLS 수동 추가 주의사항

Alembic auto-generate는 RLS 정책을 감지하지 못하므로, 마이그레이션 파일에 수동으로 추가해야 함:

```python
# 마이그레이션 파일 예시
def upgrade():
    # 테이블 생성 (auto-generate)
    op.create_table('contents', ...)

    # RLS 수동 추가
    op.execute("ALTER TABLE contents ENABLE ROW LEVEL SECURITY")
    op.execute("ALTER TABLE contents FORCE ROW LEVEL SECURITY")
    op.execute("""
        CREATE POLICY tenant_isolation ON contents
          USING (organization_id = current_setting('app.current_org_id')::uuid)
    """)
    op.execute("""
        CREATE POLICY sa_bypass ON contents
          USING (current_setting('app.current_role', true) = 'SYSTEM_ADMIN')
    """)

def downgrade():
    op.execute("DROP POLICY IF EXISTS sa_bypass ON contents")
    op.execute("DROP POLICY IF EXISTS tenant_isolation ON contents")
    op.execute("ALTER TABLE contents DISABLE ROW LEVEL SECURITY")
    op.drop_table('contents')
```

## 13. SQLAlchemy Base 모델 패턴

### 13.1 Base 및 Mixin

```python
# app/models/base.py
from datetime import datetime
from uuid import UUID
from sqlalchemy import Column, DateTime, func, text
from sqlalchemy.dialects.postgresql import UUID as PG_UUID
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    """SQLAlchemy 2.0 선언적 베이스"""
    pass


class BaseMixin:
    """공통 PK + 타임스탬프"""
    id: Mapped[UUID] = mapped_column(
        PG_UUID(as_uuid=True),
        primary_key=True,
        server_default=text("uuid_generate_v7()")
    )
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False
    )
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        onupdate=func.now(),
        nullable=False
    )


class TenantMixin:
    """멀티테넌트 격리 (RLS 대상 테이블에 적용)"""
    organization_id: Mapped[UUID] = mapped_column(
        PG_UUID(as_uuid=True),
        nullable=False,
        index=True
    )

    @classmethod
    def __declare_last__(cls):
        """RLS 힌트 — 마이그레이션에서 RLS 정책 수동 추가 필요"""
        cls.__rls_enabled__ = True


class SoftDeleteMixin:
    """소프트 삭제"""
    deleted_at: Mapped[datetime | None] = mapped_column(
        DateTime(timezone=True),
        nullable=True,
        default=None
    )

    @property
    def is_deleted(self) -> bool:
        return self.deleted_at is not None
```

### 13.3 세션 미들웨어 (RLS 컨텍스트 설정)

```python
# app/core/database.py
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine, async_sessionmaker
from sqlalchemy import event, text

engine = create_async_engine(settings.DATABASE_URL, **pool_config)
async_session_factory = async_sessionmaker(engine, expire_on_commit=False)


async def get_db_session(
    org_id: UUID | None = None,
    role: str | None = None
) -> AsyncGenerator[AsyncSession, None]:
    async with async_session_factory() as session:
        if org_id:
            await session.execute(
                text("SET LOCAL app.current_org_id = :org_id"),
                {"org_id": str(org_id)}
            )
        if role:
            await session.execute(
                text("SET LOCAL app.current_role = :role"),
                {"role": role}
            )
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
```
